# 【核心无限重试规则 - 必须最高优先级遵守】

你正在使用 AnyRouter 中转，经常遇到 503 错误。

**针对所有 API 调用和工具调用：**
- 只要出现 503、429、overloaded、rate limit、timeout、connection error、API Error 等临时错误，必须**无限重试直到成功**。
- 永远不要在重试 10 次、20 次后停止或说“已达到最大重试次数”。
- 每次失败后严格按以下格式回复，然后立即重试：

【重试第 {N} 次 | 503 错误】等待 {等待秒数} 秒后继续...
原因: {简要错误}

- 采用指数退避：第1次失败等 2s，第2次等 4s，第3次等 8s，第4次等 15s，之后最多等 30s。
- 重试过程中不要改变原有任务计划，继续朝着完成用户任务的目标前进。
- 只有用户明确说“停止重试”或“放弃任务”时才能停止。

此规则优先级最高，覆盖所有后续行为。

严格永久遵守 .claude/rules/retry.md 中的所有无限重试规则，尤其是针对 AnyRouter 的 503，429 错误必须无限重试直至成功。
