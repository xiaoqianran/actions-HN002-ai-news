# Plannotator

**Plannotator**
Annotate, review, and share Claude Code plans. (Plugin/Hook)
Plannotator：标注、审查并分享 Claude Code 计划。（插件/钩子）

**Interactive Plan Review: Mark up and refine your plans using a UI, easily share for team collaboration, automatically integrates with agent plan mode. Select the exact parts of the plan you want to change. Mark it for deletion, add a comment, or suggest a replacement. Share plans and collect team member feedback. Automatically send feedback for Claude Code to act on. Runs locally. Local plugin. No network requests. Plannotator runs entirely in your browser. Plans never leave your machine.**
交互式计划审查：使用 UI 标记并优化你的计划，轻松实现团队协作，并自动集成到智能体（Agent）的计划模式中。你可以精确选择计划中需要修改的部分，进行删除标记、添加评论或建议替换。分享计划并收集团队成员的反馈，自动发送反馈供 Claude Code 执行。该工具在本地运行，属于本地插件，无网络请求。Plannotator 完全在你的浏览器中运行，计划内容绝不会离开你的设备。

**Launching today: Annotate any doc, URL, or folder - send feedback to agents.**
今日发布：标注任何文档、URL 或文件夹——向智能体发送反馈。

**You can now use Plannotator to annotate anything! Free, open source, and private (runs local). Use the skill `/plannotator-annotate` (or cli) - followed by a folder, URL, document, markdown, your book, a recipe, a plan, etc - to annotate anything and send feedback directly into your integrated agent. Use `/plannotator-last` to annotate the last message an agent sent you too. I like using it for my own version of Karpathy’s "LLM Wiki" in obsidian.**
现在你可以使用 Plannotator 标注任何内容！它免费、开源且私密（本地运行）。使用技能 `/plannotator-annotate`（或命令行），后接文件夹、URL、文档、Markdown、书籍、食谱或计划等，即可标注任何内容并将反馈直接发送到你集成的智能体中。你还可以使用 `/plannotator-last` 来标注智能体发送给你的最后一条消息。我喜欢用它在 Obsidian 中构建我个人版本的 Karpathy “LLM Wiki”。
