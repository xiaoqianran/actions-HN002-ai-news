# An open letter asking NHS England to keep its code open
# 致英国国家医疗服务体系（NHS England）的一封公开信：请保持代码开源

Code paid for with public money should be open to the public. This principle is enshrined in the UK Government Design Principles and the NHS Service Standard. It is now being walked back. We are signing this to restate the case.
由公共资金资助的代码应当向公众开放。这一原则被写入了《英国政府设计原则》和《NHS 服务标准》之中。然而，这一原则目前正遭到背弃。我们签署这份公开信，旨在重申这一立场。

### Statement
### 声明

We disagree with the NHS technical leadership’s decision to hide the source code of all of their repositories. Making code open source requires more work than keeping it closed. That hard work is the point. It requires a higher bar of quality. It requires processes to proactively find, fix, and monitor for vulnerabilities. It requires identifying risk, and putting barriers in place to contain any damage when things go wrong.
我们不同意 NHS 技术领导层隐藏其所有代码仓库源代码的决定。与闭源相比，开源代码需要付出更多努力，而这种努力正是其核心价值所在。它要求更高的质量标准，需要建立主动发现、修复和监控漏洞的流程，并要求识别风险，在出现问题时设置屏障以控制损害。

But it works like the human immune system: being exposed to threats hardens the attack surface. Closed source allows that work to be skipped. It substitutes obscurity for depth, and obscurity buys you precious little when a sufficiently motivated attacker is involved.
但这就像人类的免疫系统一样：暴露在威胁之下反而能增强攻击面的防御能力。闭源则允许人们跳过这些工作。它用“隐蔽性”取代了“深度”，但当面对动机明确的攻击者时，这种隐蔽性几乎起不到任何保护作用。

### Warning
### 警告

We call on NHS England to withdraw the SDLC-8 red line and reaffirm its commitment to the NHS Service Standard Principle 12: “Make new source code open.”
我们呼吁 NHS England 撤销 SDLC-8 红线政策，并重申其对《NHS 服务标准》第 12 条原则的承诺：“使新源代码保持开源。”
