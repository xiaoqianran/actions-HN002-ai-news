# Progressive Web Components

I’ve worked with web components for nearly a decade and built various enterprise-scale design systems with them. While I love what they offer on paper, the same pain points keep coming back: Layout shifts, flash of unstyled content, poor server-side rendering support, too much reliance on client side JavaScript, doesn't play well with frameworks like React Server Components, accessibility issues, and so on…
我从事 Web Components 开发已有近十年，并利用它们构建了各种企业级设计系统。虽然我非常认可它们在理论上的优势，但同样的痛点总是反复出现：布局偏移（Layout shifts）、未样式化内容闪烁（FOUC）、服务器端渲染（SSR）支持不佳、过度依赖客户端 JavaScript、与 React Server Components 等框架兼容性差、可访问性问题等等……

Despite all of this, I still think web components are a great foundation for a design system. No other approach gives you true cross-framework portability built on what the web platform already provides. The problem isn’t necessarily the model itself, it’s how we’ve been building them.
尽管如此，我仍然认为 Web Components 是构建设计系统的绝佳基础。没有其他方案能像它一样，基于 Web 平台原生能力提供真正的跨框架可移植性。问题不在于模型本身，而在于我们构建它们的方式。

This is how I ended up creating Elena, a library that I’m open sourcing today. Elena starts from HTML and CSS, and stays grounded in web standards and what the web platform natively provides.
这就是我创建 Elena 的初衷，今天我将其开源。Elena 从 HTML 和 CSS 出发，始终立足于 Web 标准以及 Web 平台原生提供的能力。

### What is a Progressive Web Component?
### 什么是渐进式 Web 组件（Progressive Web Component）？

A Progressive Web Component is a native Custom Element designed in two layers: a base layer of HTML and CSS that renders immediately, without JavaScript, and an enhancement layer of JavaScript that adds reactivity, event handling, and more advanced templating.
渐进式 Web 组件是一种原生自定义元素，采用双层设计：基础层由 HTML 和 CSS 组成，无需 JavaScript 即可立即渲染；增强层由 JavaScript 组成，用于添加响应式、事件处理和更高级的模板功能。

There are three types of Progressive Web Components:
渐进式 Web 组件分为三种类型：

*   **Composite Components** that wrap and enhance the HTML composed inside them. All of their HTML and CSS lives in the Light DOM. You could also call these HTML Web Components.
    **组合组件（Composite Components）**：包裹并增强其内部组合的 HTML。其所有 HTML 和 CSS 都位于 Light DOM 中。你也可以称之为 HTML Web Components。
*   **Primitive Components** that are self-contained and render their own HTML. All of their CSS lives in the Light DOM together with the base HTML required for rendering the initial state.
    **基础组件（Primitive Components）**：自包含并渲染其自身的 HTML。其所有 CSS 与渲染初始状态所需的基础 HTML 一起位于 Light DOM 中。
*   **Declarative Components** that are a hybrid of these and utilize Declarative Shadow DOM.
    **声明式组件（Declarative Components）**：上述两者的混合体，并利用了声明式 Shadow DOM。

Note: Elena doesn’t force this taxonomy. They’re all just web components, and you choose how to build yours. But since “Progressive Web Components” is a design philosophy rather than a library feature, understanding the distinction between these approaches helps when deciding what fits your use case.
注意：Elena 并不强制要求这种分类。它们本质上都是 Web Components，你可以自由选择构建方式。但由于“渐进式 Web 组件”是一种设计理念而非库功能，理解这些方法之间的区别有助于你决定哪种方式最适合你的应用场景。

### So what is Elena, anyway?
### 那么，Elena 到底是什么？

Elena is a simple, tiny library (2.6kB) for helping you to build Progressive Web Components. Unlike most web component libraries, Elena doesn’t force JavaScript for everything. You can load HTML and CSS first, then use JavaScript to progressively add interactivity.
Elena 是一个简单且轻量（2.6kB）的库，旨在帮助你构建渐进式 Web 组件。与大多数 Web 组件库不同，Elena 不会强制要求所有内容都依赖 JavaScript。你可以先加载 HTML 和 CSS，然后通过 JavaScript 逐步添加交互性。

I built Elena for teams creating component libraries and design systems. If you need web components that work across multiple frameworks, render HTML and CSS before JavaScript loads, and sidestep common issues like accessibility problems, server-side rendering limitations, and layout shifts, Elena is built for exactly that.
我为那些构建组件库和设计系统的团队开发了 Elena。如果你需要跨框架工作的 Web 组件，希望在 JavaScript 加载前渲染 HTML 和 CSS，并避开可访问性问题、SSR 限制和布局偏移等常见痛点，那么 Elena 正是为此而生。

It handles the cross-framework complexity (prop/attribute syncing, event delegation, framework compatibility) so you can focus on building web components rather than plumbing.
它处理了跨框架的复杂性（属性同步、事件委托、框架兼容性），让你能够专注于构建组件本身，而不是处理底层琐事。

Since Progressive Web Components is a design philosophy rather than a library feature, Elena also allows you to build regular web components. The full standard custom element lifecycle and features such as open or closed Shadow DOM, `<template>`, `<slot>` and even Declarative Shadow DOM are all supported out of the box.
由于“渐进式 Web 组件”是一种设计理念而非库功能，Elena 也允许你构建常规的 Web 组件。它开箱即用地支持完整的标准自定义元素生命周期，以及开放/封闭 Shadow DOM、`<template>`、`<slot>` 甚至声明式 Shadow DOM 等特性。

### What is Elena’s approach to SSR?
### Elena 的 SSR 方法是什么？

Elena’s approach to server-side rendering is simple and straightforward. Since Progressive Web Components are primarily HTML and CSS, you don’t need any special logic on the server to render them.
Elena 的服务器端渲染方法简单直接。由于渐进式 Web 组件主要由 HTML 和 CSS 构成，你不需要在服务器端使用任何特殊逻辑来渲染它们。

Components without a `render()` method are fully SSR-compatible by default, while components with `render()` provide partial support and complete hydration on the client side.
没有 `render()` 方法的组件默认完全兼容 SSR，而带有 `render()` 的组件则提供部分支持，并在客户端进行完整的水合（Hydration）。

The “partial support” bit for the latter means that you can render the initial state without JavaScript, but JS is needed for the interactivity (unless you also use the provided `@elenajs/ssr` tool).
后者提到的“部分支持”意味着你可以在没有 JavaScript 的情况下渲染初始状态，但交互功能仍需要 JS（除非你使用了提供的 `@elenajs/ssr` 工具）。

Elena also supports Declarative Shadow DOM for cases where you may need stronger isolation, but still want the component to render server-side.
Elena 还支持声明式 Shadow DOM，适用于那些需要更强隔离性，但仍希望在服务器端渲染组件的场景。

### 发布候选版本今日上线！

**Feature highlights include:**
**功能亮点包括：**

*   🔋 **Extremely lightweight**: 2.6kB minified and compressed, simple and tiny by design.
    **极度轻量**：压缩后仅 2.6kB，设计简洁小巧。
*   📈 **Progressively enhanced**: Renders HTML and CSS first, then hydrates with JavaScript.
    **渐进增强**：优先渲染 HTML 和 CSS，随后通过 JavaScript 进行水合。

    **默认可访问**：基于语义化 HTML 基础，无 Shadow DOM 障碍。
*   🌍 **Standards based**: Built entirely on native custom elements and web standards.
    **基于标准**：完全构建于原生自定义元素和 Web 标准之上。
*   ⚡ **Reactive updates**: Prop and state changes trigger efficient, batched re-renders.
    **响应式更新**：属性和状态变更会触发高效的批量重新渲染。

    **作用域样式**：简单整洁的 CSS 封装，无需复杂的变通方案。

    **SSR 友好**：开箱即用，如有需要可使用可选的服务器端工具。
*   🧩 **Zero dependencies**: No runtime dependencies, runs entirely on the web platform.
    **零依赖**：无运行时依赖，完全运行在 Web 平台上。

    **零锁定**：适用于所有主流框架，或者完全不使用框架。

### Included packages
### 包含的包

Elena is divided into 13 npm packages published under the `@elenajs` scope. These are the main packages intended for development:
Elena 被拆分为 13 个 npm 包，发布在 `@elenajs` 命名空间下。以下是用于开发的主要包：

*   `@elenajs/core`: Elena core runtime. The thing you’d use 99% of the time, other packages are just supporting tooling.
    `@elenajs/core`：Elena 核心运行时。99% 的情况下你只需要用到它，其他包仅作为辅助工具。
*   `@elenajs/bundler`: Bundler for Elena web component libraries.
    `@elenajs/bundler`：用于 Elena Web 组件库的打包工具。
*   `@elenajs/cli`: CLI for scaffolding Elena web components.
    `@elenajs/cli`：用于搭建 Elena Web 组件脚手架的命令行工具。
*   `@elenajs/ssr`: Server-side rendering tools for Elena.
    `@elenajs/ssr`：Elena 的服务器端渲染工具。
*   `@elenajs/mcp`: MCP server to support working with Elena.
    `@elenajs/mcp`：支持 Elena 开发的 MCP 服务器。
*   `@elenajs/components`: Example Progressive Web Components.
    `@elenajs/components`：渐进式 Web 组件示例。
