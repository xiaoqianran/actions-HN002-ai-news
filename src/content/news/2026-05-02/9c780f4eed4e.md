### The calm before the ransom: What you see is not all there is
### 勒索前的平静：你所看到的并非全部

What I want you to take away from this article: Just because everything is calm in your environment, it does not mean that your system is safe. Who knows, the login for your critical infrastructure may already be on sale and just waiting for someone interested in attacking you to buy it and come after you. From the article: ...the absence of a visible incident is just silence, and silence can mean several things. The company with an immaculate record may indeed have top-notch defenses. But it may also have avoided the attention of anyone ill-intentioned. In its 2025 Data Breach Investigations Report, Verizon put a number on how wide the gap between perceived security and actual exposure can get: it found that 54% of ransomware victims had their domains appear in at least one infostealer log or illicit marketplace posting before the attack.

我希望你从这篇文章中领悟到：仅仅因为你的环境一切平静，并不意味着你的系统是安全的。谁知道呢，你关键基础设施的登录凭证可能已经在售，正等着有兴趣攻击你的人买下它并对你下手。文章指出：……没有明显的安全事件仅仅是沉默，而沉默可能意味着多种情况。一家拥有完美记录的公司确实可能拥有顶级的防御措施，但也可能只是尚未引起恶意攻击者的注意。Verizon 在其《2025 年数据泄露调查报告》中量化了感知安全与实际暴露之间的巨大差距：报告发现，54% 的勒索软件受害者在遭受攻击前，其域名至少出现在一个信息窃取日志或非法市场帖子中。

### Malicious AI Prompt Injection Attacks Increasing, but Sophistication Still Low: Google
### 恶意 AI 提示词注入攻击呈上升趋势，但复杂程度仍较低：谷歌

The title of the article is perfectly phrased to reflect what the article is trying to convey. I'll add the following: it's only a matter of time before that sophistication changes from "low" to "high". What happens if Mythos is released publicly? I don't know. You tell me. From the article: The most important, however, from a security standpoint are the malicious prompt injection attempts. The researchers uncovered two types of such attacks: exfiltration and destruction. Some websites contained prompts instructing AI to collect data, including IPs and credentials, and send it to an attacker-specified email address.

这篇文章的标题非常准确地传达了其核心内容。我要补充一点：这种复杂程度从“低”变“高”只是时间问题。如果 Mythos 被公开发布会怎样？我不知道，你告诉我。文章指出：然而，从安全角度来看，最重要的是恶意提示词注入尝试。研究人员发现了两种此类攻击：数据外泄和破坏。一些网站包含的提示词会指示 AI 收集包括 IP 和凭据在内的数据，并将其发送到攻击者指定的电子邮件地址。

### Critical GitHub Vulnerability Exposed Millions of Repositories
### GitHub 关键漏洞暴露数百万个存储库

I was stunned when I saw the article's title. Surprised when I read it. Happy when I learned that they [the researchers] informed GitHub and they patched it the same day, March 4, 2026. However, at the time of writing, the article noted that 88 percent of Enterprise Server instances have not been updated to a patched version. From the article: By exploiting an injection flaw in GitHub’s internal protocol, any authenticated user could execute arbitrary commands on GitHub’s backend servers with a single git push command – using nothing but a standard git client. In the case of GitHub Enterprise Server, an attacker can exploit the vulnerability to fully compromise the server and gain access to all repositories and internal secrets.

看到这篇文章的标题时我惊呆了。阅读内容时我感到惊讶。得知他们（研究人员）通知了 GitHub 并且 GitHub 在 2026 年 3 月 4 日当天就修复了该漏洞时，我感到欣慰。然而，在撰写本文时，文章指出 88% 的企业服务器实例尚未更新到修复版本。文章指出：通过利用 GitHub 内部协议中的注入漏洞，任何经过身份验证的用户只需一个 git push 命令，仅使用标准的 git 客户端，就能在 GitHub 后端服务器上执行任意命令。对于 GitHub 企业服务器，攻击者可以利用该漏洞完全控制服务器，并获取所有存储库和内部机密的访问权限。

### Popular WordPress redirect plugin hid dormant backdoor for years
### 热门 WordPress 重定向插件隐藏休眠后门多年

I really don't know what to say about this. Undoubtedly, this is a breach of trust of whoever was using that plug-in on their website. From the article: The real danger for impacted websites, though, comes from the updating mechanism itself, which enabled arbitrary code execution on demand. That mechanism is still present on sites using the plugin, but dormant because the malicious external command-and-control subdomain does not resolve. The domain is active, though.

对此我真的不知道该说什么。毫无疑问，这辜负了所有在网站上使用该插件的用户对开发者的信任。文章指出：受影响网站的真正危险来自于更新机制本身，它允许按需执行任意代码。该机制在仍在使用该插件的网站上依然存在，但处于休眠状态，因为恶意的外部命令与控制子域名无法解析。不过，该域名目前仍处于活跃状态。

### Critical cPanel Authentication Vulnerability Identified — Update Your Server Immediately
### cPanel 关键身份验证漏洞被发现——请立即更新服务器

相信我，你真的应该更新！情况如下：……网络托管和域名注册公司 Namecheap 披露，该漏洞“涉及一种身份验证登录利用，可能允许未经授权访问控制面板”。该身份验证绕过漏洞已被分配 CVE 编号 CVE-2026-41940，CVSS 评分为 9.8 分（满分 10 分）。cPanel 在其公告更新中表示，补丁也已推送到 WP Squared 136.1.7 版本。

### The most severe Linux threat to surface in years catches the world flat-footed
### 多年来最严重的 Linux 威胁出现，令全球措手不及

Given the significant role that Linux plays in the world right now, this does not look good. For added context, the name of the vulnerability is CopyFail. More details: The critical flaw, tracked as CVE-2026-31431 and the name CopyFail, is a local privilege escalation, a vulnerability class that allows unprivileged users to elevate themselves to administrators. CopyFail is particularly severe because it can be exploited with a single piece of exploit code. With that, an attacker can, among other things, hack multi-tenant systems, break out of containers based on Kubernetes or other frameworks, and create malicious pull requests that pipe the exploit code through CI/CD workflows.

考虑到 Linux 目前在世界范围内的重要作用，这可不是什么好消息。补充背景：该漏洞名为 CopyFail。更多细节：这个关键缺陷被追踪为 CVE-2026-31431，名为 CopyFail，属于本地提权漏洞，这类漏洞允许无特权用户将自己提升为管理员。CopyFail 特别严重，因为它只需一段利用代码即可被利用。利用这一点，攻击者可以入侵多租户系统、从基于 Kubernetes 或其他框架的容器中逃逸，并创建恶意拉取请求（Pull Request），将利用代码通过 CI/CD 工作流进行传播。

### New Python Backdoor Uses Tunneling Service to Steal Browser and Cloud Credentials
### 新型 Python 后门利用隧道服务窃取浏览器和云凭据

When I read articles like this, genuinely, I am not surprised. Why? Attackers always, I repeat, always try to find a way to compromise a system. From the article: What makes the attack chain noteworthy is that the core Python implant is embedded directly inside the dropper script, from where it's extracted, reconstructed, and executed. This reduces the need for repeatedly having to reach out to external infrastructure and minimizes the forensic footprint. Once launched, the malware establishes communication with "bore[.]pub," a Rust-based tunneling service, allowing the operator to issue commands that facilitate remote command execution and extensive surveillance.

当我读到这类文章时，我真的不感到惊讶。为什么？因为攻击者总是——我再说一遍，总是——试图寻找破坏系统的方法。文章指出：该攻击链值得注意的地方在于，核心 Python 植入程序直接嵌入在释放脚本（dropper script）中，并从中提取、重构和执行。这减少了重复连接外部基础设施的需求，并最大限度地减少了取证痕迹。一旦启动，该恶意软件就会与基于 Rust 的隧道服务“bore[.]pub”建立通信，允许操作员下达指令，从而实现远程命令执行和广泛的监控。
