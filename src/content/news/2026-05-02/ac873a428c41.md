Before You Touch Any Code — Read These Five Rules
I spent about half a day debugging issues that all traced back to violating one of these. Saving you the pain:

在动代码之前——请阅读这五条规则
我花了半天时间调试问题，最后发现全都是因为违反了其中一条规则。为了让你少走弯路：

Rule 1 — MEILISEARCH_URL is the only thing that changes between environments. Your NestJS code never hardcodes a host. It always reads from process.env.MEILISEARCH_URL. The .env file is what changes per machine, per deployment topology. Code stays identical everywhere.

规则 1 — MEILISEARCH_URL 是不同环境间唯一需要改变的配置。你的 NestJS 代码绝不能硬编码主机地址。它应该始终从 process.env.MEILISEARCH_URL 读取。不同机器、不同部署拓扑结构只需修改 .env 文件。代码在任何地方都保持一致。

Rule 2 — Never use the Master Key in your application code. The master key is a one-time bootstrap tool. Use it once to generate scoped API keys, then your app only ever uses those scoped keys. The master key never appears in search.service.ts or any NestJS code.

规则 2 — 永远不要在应用代码中使用 Master Key。Master Key 是一次性的引导工具。用它生成受限的 API Key 后，你的应用就只应使用这些受限 Key。Master Key 不应出现在 search.service.ts 或任何 NestJS 代码中。

Rule 3 — Always pin the MeiliSearch Docker image version. Never use :latest. It silently upgrades between incompatible versions and corrupts your database on restart. Always pin: getmeili/meilisearch:v1.37.0.

规则 3 — 始终锁定 MeiliSearch 的 Docker 镜像版本。永远不要使用 :latest。它会在不兼容的版本间静默升级，导致重启时数据库损坏。请务必锁定版本，例如：getmeili/meilisearch:v1.37.0。

Rule 4 — MEILI_ENV=production is not optional. Development mode accepts unauthenticated requests — a security hole. Production mode enforces API key auth on every request. Always set this regardless of environment.

规则 4 — MEILI_ENV=production 不是可选项。开发模式接受未经身份验证的请求——这是一个安全漏洞。生产模式强制要求每个请求都进行 API Key 验证。无论什么环境，请务必设置此项。

Rule 5 — Port 7700 must always be reachable from wherever NestJS runs. MeiliSearch always runs in Docker. NestJS might run in Docker, raw on the same machine, or on a completely different server. The only requirement: port 7700 on the MeiliSearch host must be reachable from wherever NestJS is.

规则 5 — 无论 NestJS 在哪里运行，7700 端口必须始终可达。MeiliSearch 始终运行在 Docker 中。NestJS 可能运行在 Docker 中、同一台机器的宿主机上，或者完全不同的服务器上。唯一的要求是：NestJS 所在位置必须能访问到 MeiliSearch 主机的 7700 端口。
