# OpenWarp

**Model Routing & BYOP (Bring Your Own Provider)**
Native support for 6 API protocols via a GenAI adaptation layer, allowing free combinations of Base URL, API Key, and Model. Native routing for 6 API protocols—no more "forcing" OpenAI compatibility. Supports multi-turn reasoning feedback: DeepSeek `reasoning_content`, Claude `thinking`, and Gemini.
模型路由 BYOP 自定义提供商，通过 GenAI 适配层原生支持 6 种 API 协议，Base URL / API Key / Model 自由组合。6 种 API 协议原生路由，不再「OpenAI 兼容硬塞」。支持推理思考多轮回传：DeepSeek `reasoning_content` / Claude `thinking` / Gemini。

**System Prompt Template**
Utilizes a `minijinja` template system for system prompts, configurable via `~/.config/openwarp.toml`.
使用 `minijinja` 模板系统渲染系统提示词，配置路径为 `~/.config/openwarp.toml`。

**Privacy-First Local Storage**
Privacy is the priority. Cloud Agent and Computer Use are disabled by default; no data is uploaded to the cloud, and credentials are saved strictly locally. 100% local credential storage, zero telemetry collection, and no outbound credential transmission.
隐私优先，本地存储。默认关闭 Cloud Agent / Computer Use，不上传云端，凭证仅本地保存。100% 本地凭证存储，不收集遥测，凭证零外发。

**Minijinja Template Engine**
Powerful template rendering based on `minijinja` to dynamically render instructions based on context.
基于 `minijinja` 的强大模板，根据上下文动态渲染指令。

**Multilingual Interface**
Native support for Chinese and English UI, with the community invited to extend support for more languages in the future.
原生中文与英文 UI，后续社区可继续扩展更多语种。

**Preserving the Warp Experience**
Continuously merged with the upstream Warp codebase, fully preserving Blocks, AI Commands, Workflows, and Keymaps.
基于 Warp 上游持续合并，完整保留块（Blocks）、AI 命令、工作流（Workflows）及键位（Keymaps）。

**Fully Open Source**
Adheres to the same open-source licensing as upstream Warp, utilizing dual AGPL/MIT licensing with all code publicly available.
开源协议与 Warp 上游一致，采用 AGPL / MIT 双许可，代码全部公开。
