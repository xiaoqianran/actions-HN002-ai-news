# simstudioai / sim

The open-source platform to build AI agents and run your agentic workforce. Connect 1,000+ integrations and LLMs to orchestrate agentic workflows.
这是一个用于构建 AI 智能体并运行智能体工作流的开源平台。通过连接 1,000 多种集成工具和大型语言模型（LLM），你可以轻松编排智能体工作流。

Build Workflows with Ease: Design agent workflows visually on a canvas—connect agents, tools, and blocks, then run them instantly.
轻松构建工作流：在画布上可视化设计智能体工作流——连接智能体、工具和功能块，即可立即运行。

Supercharge with Copilot: Leverage Copilot to generate nodes, fix errors, and iterate on flows directly from natural language.
利用 Copilot 提效：借助 Copilot，你可以直接通过自然语言生成节点、修复错误并迭代工作流。

Integrate Vector Databases: Upload documents to a vector store and let agents answer questions grounded in your specific content.
集成向量数据库：上传文档至向量存储，让智能体基于你的特定内容回答问题。

### Quickstart
### 快速开始

Cloud-hosted: sim.ai
云端托管版：sim.ai

Self-hosted: NPM Package
自托管版：NPM 包
```bash
npx simstudio → http://localhost:3000
```
Note: Docker must be installed and running on your machine.
注意：你的机器上必须安装并运行 Docker。

Options:
选项：
- `-p, --port <port>`: Port to run Sim on (default 3000) / 运行 Sim 的端口（默认 3000）
- `--no-pull`: Skip pulling latest Docker images / 跳过拉取最新的 Docker 镜像

Self-hosted: Docker Compose
自托管版：Docker Compose
```bash
git clone https://github.com/simstudioai/sim.git && cd sim
docker compose -f docker-compose.prod.yml up -d
```
Open http://localhost:3000
访问 http://localhost:3000

Sim also supports local models via Ollama and vLLM — see the Docker self-hosting docs for setup details.
Sim 还支持通过 Ollama 和 vLLM 使用本地模型——具体设置请参阅 Docker 自托管文档。

Self-hosted: Manual Setup
自托管版：手动安装

Requirements: Bun, Node.js v20+, PostgreSQL 12+ with pgvector
要求：Bun, Node.js v20+，以及带有 pgvector 的 PostgreSQL 12+。

Clone and install:
克隆并安装：
```bash
git clone https://github.com/simstudioai/sim.git
cd sim
bun install
bun run prepare # Set up pre-commit hooks / 设置预提交钩子
```

Set up PostgreSQL with pgvector:
设置带有 pgvector 的 PostgreSQL：
```bash
docker run --name simstudio-db -e POSTGRES_PASSWORD=your_password -e POSTGRES_DB=simstudio -p 5432:5432 -d pgvector/pgvector:pg17
```
Or install manually via the pgvector guide.
或者通过 pgvector 指南手动安装。

Configure environment:
配置环境变量：
```bash
cp apps/sim/.env.example apps/sim/.env
# Create your secrets / 创建密钥
perl -i -pe "s/your_encryption_key/$(openssl rand -hex 32)/" apps/sim/.env
perl -i -pe "s/your_internal_api_secret/$(openssl rand -hex 32)/" apps/sim/.env
perl -i -pe "s/your_api_encryption_key/$(openssl rand -hex 32)/" apps/sim/.env

# DB configs for migration / 迁移数据库配置
cp packages/db/.env.example packages/db/.env
# Edit both .env files to set DATABASE_URL="postgresql://postgres:your_password@localhost:5432/simstudio"
# 编辑两个 .env 文件以设置 DATABASE_URL
```

Run migrations:
运行迁移：
```bash
cd packages/db && bun run db:migrate
```

Start development servers:
启动开发服务器：
```bash
bun run dev:full # Starts Next.js app and realtime socket server / 启动 Next.js 应用和实时 socket 服务器
```
Or run separately: `bun run dev` (Next.js) and `cd apps/sim && bun run dev:sockets` (realtime).
或者分开运行：`bun run dev` (Next.js) 和 `cd apps/sim && bun run dev:sockets` (实时服务)。

### Copilot API Keys
### Copilot API 密钥

Copilot is a Sim-managed service. To use Copilot on a self-hosted instance:
Copilot 是由 Sim 管理的服务。若要在自托管实例中使用 Copilot：
1. Go to https://sim.ai → Settings → Copilot and generate a Copilot API key
   前往 https://sim.ai → Settings → Copilot 并生成 Copilot API 密钥。
2. Set `COPILOT_API_KEY` environment variable in your self-hosted `apps/sim/.env` file to that value.
   将自托管 `apps/sim/.env` 文件中的 `COPILOT_API_KEY` 环境变量设置为该值。

### Tech Stack
### 技术栈

*   **Framework:** Next.js (App Router)
*   **Runtime:** Bun
*   **Database:** PostgreSQL with Drizzle ORM
*   **Authentication:** Better Auth
*   **UI:** Shadcn, Tailwind CSS
*   **Streaming Markdown:** Streamdown
*   **State Management:** Zustand, TanStack Query
*   **Flow Editor:** ReactFlow
*   **Docs:** Fumadocs
*   **Monorepo:** Turborepo
*   **Realtime:** Socket.io
*   **Background Jobs:** Trigger.dev
*   **Remote Code Execution:** E2B
*   **Isolated Code Execution:** isolated-vm
