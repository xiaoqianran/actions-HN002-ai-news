# Beyond Lists: Using Python Deque for Real-Time Sliding Windows
# 超越列表：使用 Python Deque 实现实时滑动窗口

Programming Beyond Lists: Using Python Deque for Real-Time Sliding Windows. Use deque to build high-performance history buffers and rolling averages.
超越列表编程：使用 Python Deque 实现实时滑动窗口。利用 deque 构建高性能历史缓冲区和滚动平均值。

Python double-ended queue, or simply deque, is an uncommon type of collection. If we search around the internet, we will find a lot of information about lists, dictionaries, and tuples, but little on deque. Deque (you can also pronounce it “deck”) is an interesting and useful type of collection in Python. What makes it different than other objects is that it will hold only the number of items that you want or fewer, never more. A double-ended queue will hold just up to the number of items that you determine. Never more.
Python 的双端队列（double-ended queue），简称 deque，是一种不太常见的集合类型。如果我们上网搜索，会发现大量关于列表（list）、字典（dict）和元组（tuple）的信息，但关于 deque 的却很少。Deque（你也可以读作“deck”）是 Python 中一种有趣且实用的集合类型。它与其他对象的不同之处在于，它只会保留你指定数量或更少的项目，绝不会超过这个数量。双端队列只会容纳你所设定的项目数量，绝不多于此。

### What the Heck is a Deck?
### 到底什么是 Deck？

So it works just as a deck using the FIFO system (First In, First Out). Once the deck is full, if you append another item, it will drop the first element on the left and add the new one to the right. Let’s see some basic examples to understand this collection. First, import it from collections: `from collections import deque`.
它就像一副牌一样，使用 FIFO（先进先出）系统工作。一旦队列满了，如果你再添加一个项目，它会丢弃左侧的第一个元素，并将新元素添加到右侧。让我们通过一些基本示例来了解这个集合。首先，从 collections 模块导入它：`from collections import deque`。

### Creating a deque
### 创建 deque

Next, we will create a simple deck and add a maximum length of 3 items to it.
接下来，我们将创建一个简单的队列，并为其设置 3 个项目的最大长度。

```python
# Create a new deque with 3 (or less) elements
# 创建一个包含 3 个（或更少）元素的新 deque
my_deck = deque(maxlen = 3)

# Adding one element
# 添加一个元素
my_deck.append(1)
my_deck.append(2)
my_deck.append(3)

# View my_deck
# 查看 my_deck
# deque([1, 2, 3])
```

Nice. Once our deck is full, observe what happens when I try to add another value to it. The first item on the left (1) gets dropped, opening space to the new item appended to the right (extra).
很好。一旦我们的队列满了，观察当我尝试向其中添加另一个值时会发生什么。左侧的第一个项目 (1) 会被丢弃，从而为右侧新添加的项目 (extra) 腾出空间。

```python
# It drops the first element and adds the new one at the end.
# 它会丢弃第一个元素并在末尾添加新元素。
my_deck.append('extra')
deque([2, 3, 'extra'])
```

### Append Items to the Left
### 向左侧添加项目

Now, remember that the collection is named as double-ended queue; thus, you can also append or extend elements to the left. In that case, naturally, it will drop the element on the far right.
现在，请记住该集合被称为双端队列；因此，你也可以向左侧添加或扩展元素。在这种情况下，它自然会丢弃最右侧的元素。

```python
# Append to left
# 向左添加
my_deck.appendleft('left')
# [OUT]: deque(['left', 2, 3])

# Extend to left
# 向左扩展
my_deck.extendleft(['d', 'd'])
#[OUT]: deque(['d', 'd', 'left'])
```

### Rotate Items
### 旋转项目

You can also rotate the elements, moving them one (or more) positions to the right or left.
你还可以旋转元素，将它们向右或向左移动一个（或多个）位置。

```python
# Create a new deque with 3 (or less) elements
my_deck = deque(maxlen = 3)
# Adding one element
my_deck.extend([1,2,3]) # deque([1, 2, 3])

# Rotating the elements by one position to the right
# 将元素向右旋转一个位置
my_deck.rotate() # deque([3, 1, 2])

# Rotate to the left
# 向左旋转
my_deck.rotate(-1) # deque([1, 2, 3])
```

Besides the rotation, it is easy to completely reverse the deck.
除了旋转之外，还可以轻松地完全反转队列。

```python
# New deck
my_deck.extend([1, 2, 3]) #[OUT]: deque([1, 2, 3])

# Reverse deck
# 反转队列
my_deck.reverse() # [OUT]: deque([3, 2, 1])
```

### Removing Items
### 移除项目

In a deck, you can remove an element from the left, right, or by name.
在队列中，你可以从左侧、右侧移除元素，或按名称移除。

```python
# Remove item from the left
# 从左侧移除项目
my_deck.popleft()

# Remove item from the right
# 从右侧移除项目
my_deck.pop()

# Remove item by name
# 按名称移除项目
my_deck.remove('a')
```

You can also remove all items and clear your deck.
你也可以移除所有项目并清空你的队列。

```python
my_deck.clear() #[OUT]: deque([])
```

### Applications
### 应用场景

#### 1. The “Recent Search History” (Memory Management)
#### 1. “最近搜索历史”（内存管理）

While lists grow indefinitely, deque has a `maxlen` parameter. This is perfect for features like “Recently Viewed” or “Recent Search History”, where you only want to keep the last N items without manually deleting the old ones.
虽然列表会无限增长，但 deque 具有 `maxlen` 参数。这非常适合“最近浏览”或“最近搜索历史”等功能，你只需保留最后 N 个项目，而无需手动删除旧项目。

```python
# Keep only the last 3 user searches
# 仅保留最后 3 次用户搜索
search_history = deque(maxlen=3)
search_history.append("Python tutorials")
search_history.append("Machine Learning")
search_history.append("Data Science")
search_history.append("Deep Learning")
# "Python tutorials" is automatically removed
# "Python tutorials" 会被自动移除
print(list(search_history)) 
# Output: ['Machine Learning', 'Data Science', 'Deep Learning']
```

#### 2. Live Data Stream & Moving Averages
#### 2. 实时数据流与移动平均值

In data science or IoT, you often need to calculate a moving average of a stream (like temperature sensors or stock prices). Using a deque allows you to maintain a “sliding window” of data efficiently.
在数据科学或物联网领域，你经常需要计算数据流（如温度传感器或股票价格）的移动平均值。使用 deque 可以让你高效地维护一个数据的“滑动窗口”。

```python
def moving_average(stream, window_size=5):
    window = deque(maxlen=window_size)
    for val in stream:
        window.append(val)
        if len(window) == window_size:
            yield sum(window) / window_size
```

#### 3. Multithreaded Task Queues (Thread Safety)
#### 3. 多线程任务队列（线程安全）

One of the “hidden” benefits of deque in CPython is that `.append()` and `.popleft()` are thread-safe. This makes it an excellent choice for a simple producer-consumer pattern where one thread adds tasks, and another executes them.
CPython 中 deque 的一个“隐藏”优势是 `.append()` 和 `.popleft()` 是线程安全的。这使其成为简单的生产者-消费者模式的绝佳选择，其中一个线程添加任务，另一个线程执行任务。
