# Sparse Prefix Caching for Hybrid and Recurrent LLM Serving
# 用于混合与循环大语言模型推理的稀疏前缀缓存

**Abstract:** Prefix caching is a key latency optimization for autoregressive LLM serving, yet existing systems assume dense per-token key/value reuse. State-space models change the structure of the problem: a recurrent layer can resume from a single stored state rather than requiring the entire token history. This asymmetry opens a new design point between no reuse and dense caching: store exact recurrent states at a sparse set of checkpoint positions and, on a cache hit, resume from the deepest stored checkpoint and recompute the remaining suffix exactly.

**摘要：** 前缀缓存是自回归大语言模型（LLM）推理中一项关键的延迟优化技术，但现有系统通常假设对每个 token 进行密集的键/值（KV）重用。状态空间模型（SSM）改变了这一问题的结构：循环层可以从单个存储的状态恢复，而无需整个 token 历史记录。这种不对称性在“无重用”和“密集缓存”之间开辟了一个新的设计点：在稀疏的检查点位置存储精确的循环状态；当缓存命中时，从最深处的存储检查点恢复，并精确地重新计算剩余的后缀。

We formalize sparse prefix caching as checkpoint placement under a distribution over overlap depths, yielding an exact O(NM) dynamic program. For use cases where requests share a non-trivial prefix (e.g. asking different questions about a single long document), we show that our method consistently improves the Pareto frontier traced by standard heuristics on real-world data.

我们将稀疏前缀缓存形式化为基于重叠深度分布的检查点放置问题，并得出了一个精确的 O(NM) 动态规划算法。针对请求共享非平凡前缀的使用场景（例如针对同一篇长文档提出不同问题），我们证明了该方法在真实数据上能够持续改善由标准启发式算法所追踪的帕累托前沿（Pareto frontier）。

Across QuALITY and System Prompts, distribution-aware placement dominates every fixed-budget baseline on the measured layer-group Pareto frontier and matches or outperforms the strongest heuristic (block caching) while typically using substantially fewer checkpoints, with the largest gains at low checkpoint budgets where the overlap distribution is most non-uniform.

在 QuALITY 和系统提示词（System Prompts）数据集的测试中，这种感知分布的放置策略在测量的层组帕累托前沿上优于所有固定预算的基准方法，并能匹配或超越最强的启发式算法（块缓存），同时通常使用显著更少的检查点。在检查点预算较低且重叠分布最不均匀的情况下，该方法的增益最为显著。

The method is most relevant when many requests share a substantial but not identical prefix within a retained cache entry. It preserves exact outputs, does not change the recurrent computation itself or require new recurrent update kernels, applies to recurrent/SSM layers whose hidden state can be extracted and restored exactly, and for hybrid models can be combined with existing KV-cache compression techniques.

当许多请求在保留的缓存条目中共享一个实质性但不完全相同的前缀时，该方法最为适用。它能保持输出的精确性，不会改变循环计算本身，也不需要新的循环更新内核；它适用于隐藏状态可以被精确提取和恢复的循环/SSM 层，并且对于混合模型，它可以与现有的 KV 缓存压缩技术相结合。
