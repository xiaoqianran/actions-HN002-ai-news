# iOS, macOS, and iPadOS 26.5 updates arrive with encrypted RCS messaging and more
# iOS、macOS 和 iPadOS 26.5 更新发布，带来加密 RCS 消息及更多功能

Apple has released version 26.5 of all of its operating systems today: iOS 26.5, iPadOS 26.5, macOS 26.5, watchOS 26.5, tvOS 26.5, visionOS 26.5, and version 26.5 of the HomePod software (whew). None of these are particularly momentous updates, which is pretty normal this late in their lifecycle, but they add a small batch of new features alongside the pile of patches outlined on Apple’s security vulnerabilities page.

苹果公司今日发布了旗下所有操作系统的 26.5 版本：iOS 26.5、iPadOS 26.5、macOS 26.5、watchOS 26.5、tvOS 26.5、visionOS 26.5 以及 HomePod 软件的 26.5 版本（呼，阵容庞大）。这些更新并非重大版本迭代，这在系统生命周期的后期阶段属于正常现象，但它们在修复苹果安全漏洞页面所列出的诸多问题的同时，也增加了一小批新功能。

This is Apple’s first release to support end-to-end encryption for the RCS messaging standard, for example, which, when enabled, can give green-bubble messages some of the same security and privacy advantages that iMessage users have long enjoyed. Encrypted RCS messaging has a “beta” label in this release, and Apple says it’s limited to a subset of supported cellular carriers. Expanded support “will roll out over time.” Encrypted chats will show up with a padlock icon in the Messages app; if you don’t see a padlock, the message isn’t encrypted, even if you’re using RCS.

例如，这是苹果首次发布支持 RCS 消息标准端到端加密的版本。启用后，绿色气泡消息（非 iMessage 消息）也能获得 iMessage 用户长期以来享有的部分安全和隐私优势。在此次发布中，加密 RCS 消息带有“测试版”标签，苹果表示该功能目前仅限于部分支持的移动运营商。更广泛的支持将“随时间推移逐步推出”。加密聊天在“信息”应用中会显示一个挂锁图标；如果你没有看到挂锁，即使你正在使用 RCS，该消息也未加密。

Other additions in the 26.5 releases are new Pride-themed wallpapers and some of the initial work needed to support ads in the Apple Maps app. There are also a handful of smaller platform-specific additions and bug fixes, which you can find on Apple’s release notes pages.

26.5 版本中的其他新增功能包括全新的“彩虹骄傲”主题壁纸，以及为支持苹果地图应用内广告所做的部分初步工作。此外，还有一些针对特定平台的较小更新和错误修复，你可以在苹果的发布说明页面上查看详情。

Apple has been testing several changes to third-party wearable support in the EU to comply with local regulations, but those features haven’t yet been included in the public versions of those iOS updates.

苹果一直在欧盟地区测试针对第三方可穿戴设备支持的几项变更，以符合当地法规，但这些功能尚未包含在 iOS 更新的公开版本中。

These are likely to be the last major updates we get for these versions of the operating system before the next-generation versions are unveiled at Apple’s Worldwide Developers Conference next month. It’s hard to say what to expect from the 27 releases, except that we’re still waiting on an updated AI-backed version of Siri that was first mentioned as part of the iOS 18 cycle in 2024, was very strongly implied to be part of the iOS 26 cycle last year, and still has not been demoed in any Apple presentations or in any software betas.

这些很可能是我们在下个月苹果全球开发者大会（WWDC）发布下一代操作系统之前，所能看到的这些版本中的最后一次重大更新。很难说 27 系列版本会有什么值得期待的内容，唯一确定的是，我们仍在等待 AI 加持的 Siri 更新版。该功能最早在 2024 年的 iOS 18 周期中被提及，去年曾被强烈暗示将包含在 iOS 26 周期中，但至今仍未在任何苹果发布会或软件测试版中进行演示。

Apple and Google announced that Apple would use Google’s Gemini language models to power the new Siri, but aside from vague promises to ship the feature this calendar year, Apple has kept mostly quiet about it. Recent reporting from Bloomberg also suggests that users will be able to choose the AI models they want to use for Apple Intelligence’s writing and image-generation tools in iOS 27, iPadOS 27, and macOS 27.

苹果和谷歌曾宣布，苹果将使用谷歌的 Gemini 语言模型来驱动新版 Siri，但除了模糊承诺在今年内推出该功能外，苹果对此一直保持沉默。彭博社最近的报道还指出，用户在 iOS 27、iPadOS 27 和 macOS 27 中，将能够选择自己想要使用的 AI 模型来执行 Apple Intelligence 的写作和图像生成工具。
