# RateQuant: Optimal Mixed-Precision KV Cache Quantization via Rate-Distortion Theory
# RateQuant：基于率失真理论的最优混合精度 KV Cache 量化

Large language models cache all previously computed key-value (KV) pairs during generation, and this KV cache grows linearly with sequence length, making it a primary memory bottleneck for serving.
大型语言模型在生成过程中会缓存所有先前计算的键值（KV）对，且该 KV Cache 随序列长度线性增长，使其成为服务部署中的主要内存瓶颈。

Quantizing the KV cache to fewer bits reduces this cost, yet all current quantizers assign the same bit-width to every attention head, ignoring the large variation in head importance.
将 KV Cache 量化为更低的比特数可以降低这一成本，但目前所有的量化器都为每个注意力头分配相同的位宽，忽略了不同注意力头之间巨大的重要性差异。

A natural idea is to allocate more bits to important heads and fewer to the rest. We show, however, that such mixed-precision allocation has a hidden pitfall: each quantizer follows a different distortion curve D(b)=alpha*beta^{-b}, and the decay rate beta varies from 3.6 to 5.3 across quantizer designs.
一个自然的思路是为重要的注意力头分配更多的比特，而为其余部分分配较少的比特。然而，我们发现这种混合精度分配存在一个隐藏的陷阱：每个量化器都遵循不同的失真曲线 D(b)=alpha*beta^{-b}，且不同量化器设计之间的衰减率 beta 从 3.6 到 5.3 不等。

Applying one quantizer's distortion model to another inverts the allocation order and makes performance worse than uniform quantization. We call this failure mode distortion model mismatch and propose RateQuant to resolve it.
将一个量化器的失真模型应用于另一个量化器会导致分配顺序颠倒，使得性能甚至不如统一量化。我们将这种失效模式称为“失真模型不匹配”，并提出了 RateQuant 来解决这一问题。

RateQuant fits a per-quantizer distortion model from a small calibration set, then solves the resulting bit-allocation problem in closed form via reverse waterfilling from rate-distortion theory.
RateQuant 通过一个小型的校准集为每个量化器拟合失真模型，然后利用率失真理论中的反向注水法（reverse waterfilling）以闭式解形式解决比特分配问题。

On Qwen3-8B at 2.5 average bits, calibrated RateQuant reduces KIVI's perplexity from 49.3 to 14.9 (70% reduction) and improves QuaRot by 6.6 PPL. The entire calibration takes 1.6 s on a single GPU and adds zero overhead at inference time.
在 Qwen3-8B 模型上，以平均 2.5 比特进行校准后，RateQuant 将 KIVI 的困惑度（perplexity）从 49.3 降低至 14.9（降低了 70%），并将 QuaRot 的性能提升了 6.6 PPL。整个校准过程在单张 GPU 上仅需 1.6 秒，且在推理阶段不会增加任何额外开销。
