Working with data in Python often starts with tools like pandas. They are intuitive, powerful, and perfect for small to medium-sized datasets. But as soon as your data grows beyond what fits comfortably in memory, performance issues begin to surface. This is where PySpark comes in.
在 Python 中处理数据通常从 pandas 等工具开始。它们直观、强大，非常适合处理中小型数据集。但一旦数据量超过了内存的承载能力，性能问题就会随之而来。这时，PySpark 就派上用场了。

Note that in this article I’ll often use the terms Spark and PySpark interchangeably. For our purposes, it doesn’t matter, but you should remember that they are different. Spark is the overarching distributed computing framework (written in Scala), and PySpark is a dedicated Python API to Spark.
请注意，在本文中，我经常会交替使用 Spark 和 PySpark 这两个术语。就我们的目的而言，这并不重要，但你应该记住它们是不同的。Spark 是一个总体的分布式计算框架（用 Scala 编写），而 PySpark 是专门用于 Spark 的 Python API。

### What is PySpark?
### 什么是 PySpark？

PySpark is the Python API for Apache Spark, a distributed computing framework for efficiently processing large volumes of data. Instead of running all computations on a single machine, Spark spreads the work across multiple machines (a cluster), allowing you to process data at scale while writing code that still feels familiar to Python users.
PySpark 是 Apache Spark 的 Python API，Apache Spark 是一个用于高效处理海量数据的分布式计算框架。Spark 不会在单台机器上运行所有计算，而是将工作分散到多台机器（集群）上，让你能够在处理大规模数据的同时，编写出对 Python 用户来说依然熟悉的程序代码。

One of the key advantages of PySpark is that it abstracts away much of the complexity of distributed systems. You do not need to manually manage threads, memory, or network communication. Spark handles these concerns for you, while you focus on describing what you want to do with the data rather than how it should be executed.
PySpark 的主要优势之一是它屏蔽了分布式系统的大部分复杂性。你无需手动管理线程、内存或网络通信。Spark 会为你处理这些问题，而你只需专注于描述你想要对数据做什么，而无需关心它具体是如何执行的。

#### 1. Clusters
#### 1. 集群

When people hear that Spark runs on a “cluster,” it can sound intimidating. In practice, you do not need deep knowledge of distributed systems to get started. A cluster is simply a group of servers networked together that can collaborate. In a Spark application running on a cluster, one machine acts as the driver, coordinating work, while the others act as executors, performing computations on data chunks.
当人们听说 Spark 在“集群”上运行时，可能会感到畏惧。实际上，你不需要深入了解分布式系统就能上手。集群只是一组联网并可以协同工作的服务器。在运行于集群的 Spark 应用程序中，一台机器充当驱动器（Driver），负责协调工作，而其他机器则充当执行器（Executors），对数据块进行计算。

When the Executor nodes have finished their work, they signal back to the Driver node, and the Driver can then perform whatever is needed with the final result set.
当执行器节点完成工作后，它们会向驱动器节点发送信号，驱动器随后可以对最终结果集进行任何必要的操作。

Just remember, you do not need to run Spark on a physical compute cluster. When you run PySpark locally, Spark simulates a cluster on your laptop or PC using multiple cores. One of the strengths of PySpark is that the same code can later be deployed to a real cluster, whether in the cloud or on-premises, with only very minor changes. This separation of coordination and execution enables Spark to scale. As datasets grow, more executors can be added to process data in parallel, reducing runtime without requiring changes to your code.
请记住，你不需要在物理计算集群上运行 Spark。当你本地运行 PySpark 时，Spark 会利用你笔记本电脑或 PC 的多核处理器来模拟一个集群。PySpark 的优势之一是，同一套代码稍后只需极小的改动，即可部署到云端或本地的真实集群中。这种协调与执行的分离使得 Spark 具备了扩展性。随着数据集的增长，可以添加更多的执行器来并行处理数据，从而在无需更改代码的情况下缩短运行时间。

#### 2. The Spark dataframe
#### 2. Spark DataFrame

At the heart of PySpark is the DataFrame API, which is the main way you work with data in Spark. A DataFrame is simply a table of data, made up of rows and columns — very similar to a table in a database or a DataFrame in pandas. If you have used SQL or pandas before, the basic ideas will feel familiar.
PySpark 的核心是 DataFrame API，这是你在 Spark 中处理数据的主要方式。DataFrame 本质上就是一个由行和列组成的数据表——与数据库中的表或 pandas 中的 DataFrame 非常相似。如果你之前使用过 SQL 或 pandas，那么这些基本概念会让你感到很亲切。

With Spark DataFrames, you can perform common data tasks such as filtering rows, selecting columns, grouping data, joining tables, and calculating summaries like counts or averages. These operations are easy to read and write, allowing you to focus on what you want to do with the data rather than the technical details of how it runs.
使用 Spark DataFrame，你可以执行常见的数据任务，例如过滤行、选择列、分组数据、连接表以及计算计数或平均值等汇总信息。这些操作易于阅读和编写，让你能够专注于你想要对数据做什么，而不是它如何运行的技术细节。

What makes Spark special is what happens behind the scenes. Spark automatically determines the most efficient way to run your DataFrame operations and then executes them in parallel across multiple computers in a cluster. You don’t need to manage this yourself — Spark handles things like splitting the data, coordinating the work, and recovering from failures if something goes wrong.
Spark 的特别之处在于其幕后的工作机制。Spark 会自动确定运行 DataFrame 操作的最有效方式，然后在集群中的多台计算机上并行执行它们。你无需亲自管理这些——Spark 会处理诸如拆分数据、协调工作以及在出现故障时进行恢复等事务。

#### 3. Lazy vs eager evaluation
#### 3. 惰性求值与立即求值

Another strength of PySpark worth knowing is its approach to lazy versus eager execution. Most Python data libraries, like Pandas, use eager execution. This means that when you run an operation, it executes immediately, followed by the next operation, and so on. PySpark deals with this differently by using a technique called lazy execution.
PySpark 值得了解的另一个优势是它处理惰性执行与立即执行的方式。大多数 Python 数据库（如 Pandas）使用立即执行。这意味着当你运行一个操作时，它会立即执行，接着是下一个操作，依此类推。PySpark 则通过一种称为“惰性执行”的技术以不同的方式处理这个问题。

When you write data transformations, such as selecting columns or filtering rows, Spark does not execute them immediately. Instead, it builds an optimised execution plan and runs the computation only when an action (such as showing results or writing data to disk) is triggered. This allows Spark to optimise the workflow before execution, making your code more efficient without extra effort on your part.
当你编写数据转换操作（例如选择列或过滤行）时，Spark 不会立即执行它们。相反，它会构建一个优化的执行计划，并且仅在触发动作（例如显示结果或将数据写入磁盘）时才运行计算。这使得 Spark 能够在执行前优化工作流，从而在无需你额外努力的情况下提高代码效率。
