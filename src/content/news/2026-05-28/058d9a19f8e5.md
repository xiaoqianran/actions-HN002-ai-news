# Learning From Pairwise Preferences: An Introduction to the Bradley-Terry Model
# 从成对偏好中学习：Bradley-Terry 模型简介

How to turn simple head-to-head choices into probabilistic rankings.
如何将简单的两两选择转化为概率排名。

Much of statistical learning assumes the availability of absolute labels. For example, an instance belongs to a class, a document receives a score, an observation is assigned a probability, a product is rated on a fixed scale.
统计学习的很大一部分假设存在绝对标签。例如，一个实例属于某个类别，一份文档获得一个分数，一个观测值被分配一个概率，或者一个产品在固定的量表上被评分。

In practice, however, human judgment often appears in a more local and comparative form. People may not know whether an answer deserves 7.4 out of 10, but they can often say which of two answers is better. They may hesitate to assign an absolute quality score to a candidate, but they can say which of two candidates seems stronger.
然而在实践中，人类的判断往往以更局部和比较的形式出现。人们可能不知道一个答案是否应该得 7.4 分（满分 10 分），但他们通常能说出两个答案中哪一个更好。他们可能不愿给候选人打出绝对的质量分数，但他们能说出哪一个候选人看起来更强。

In many real systems, comparison is much easier than calibration. This is the setting in which the Bradley-Terry model becomes especially useful by offering a mathematically clean way to learn from pairwise preferences. Rather than asking for absolute judgments, it starts from simple head-to-head outcomes and uses them to infer a latent ordering over items to give a coherent probabilistic ranking.
在许多实际系统中，比较比校准要容易得多。在这种背景下，Bradley-Terry 模型通过提供一种从成对偏好中学习的数学简洁方法，变得尤为有用。它不要求绝对判断，而是从简单的两两对比结果出发，利用这些结果推断出项目的潜在顺序，从而给出连贯的概率排名。

### The Core Idea: Each Item Has a Latent Strength
### 核心思想：每个项目都有一个潜在强度

The model begins with a simple assumption. Each item $i$ is associated with an unobserved positive strength parameter, written as $\pi_i > 0$. When item $i$ is compared with item $j$, the probability that $i$ is preferred to $j$ is defined as:
该模型始于一个简单的假设。每个项目 $i$ 都关联一个未观测到的正强度参数，记为 $\pi_i > 0$。当项目 $i$ 与项目 $j$ 进行比较时，项目 $i$ 优于项目 $j$ 的概率定义为：

$$P(i \succ j) = \frac{\pi_i}{\pi_i + \pi_j}$$

and, symmetrically we can write:
对称地，我们可以写出：

$$P(j \succ i) = \frac{\pi_j}{\pi_i + \pi_j}$$

This form is quite attractive because it is both simple and interpretable. If the two items have equal strength, then each has probability 1/2 of winning. If $\pi_i$ is much larger than $\pi_j$, then $i$ becomes much more likely to win. The Bradley-Terry model translates hidden relative strengths into observable pairwise probabilities.
这种形式非常有吸引力，因为它既简单又具有可解释性。如果两个项目强度相等，那么每个项目获胜的概率均为 1/2。如果 $\pi_i$ 远大于 $\pi_j$，那么 $i$ 获胜的可能性就大得多。Bradley-Terry 模型将隐藏的相对强度转化为可观测的两两概率。

A second and often more convenient way to write the same model is to express each positive strength as the exponential of a real-valued score: $\pi_i = \exp(\beta_i)$. Substituting this into the probability expression yields:
编写同一模型的第二种且通常更方便的方法是将每个正强度表示为实数值分数的指数：$\pi_i = \exp(\beta_i)$。将其代入概率表达式可得：

$$P(i \succ j) = \frac{\exp(\beta_i)}{\exp(\beta_i) + \exp(\beta_j)}$$

which can also be written as:
这也可以写成：

$$P(i \succ j) = \frac{1}{1 + \exp(\beta_j - \beta_i)}$$

This makes an important fact visible. The probability that $i$ beats $j$ depends only on the difference $\beta_i - \beta_j$. Bradley-Terry is therefore closely related to logistic modeling. This is the same structural idea that appears in logistic regression. In logistic regression, a binary outcome is modeled by applying the logistic function to a linear score. In Bradley-Terry, the binary outcome is the result of a head-to-head comparison, and the relevant score is simply the difference between the two latent strengths.
这揭示了一个重要事实：$i$ 击败 $j$ 的概率仅取决于差值 $\beta_i - \beta_j$。因此，Bradley-Terry 与逻辑模型密切相关。这与逻辑回归中出现的结构思想相同。在逻辑回归中，二元结果是通过将逻辑函数应用于线性分数来建模的。在 Bradley-Terry 中，二元结果是两两比较的结果，而相关的分数仅仅是两个潜在强度之间的差值。

Equivalently, the log-odds that $i$ beats $j$ are linear in $\beta_i - \beta_j$, which makes Bradley-Terry a particularly natural model for pairwise preference data. More specifically, what matters is not the absolute level of an item’s score, but its position relative to the other item in the comparison.
等价地，$i$ 击败 $j$ 的对数几率（log-odds）是 $\beta_i - \beta_j$ 的线性函数，这使得 Bradley-Terry 成为处理成对偏好数据的一种特别自然模型。更具体地说，重要的不是项目分数的绝对水平，而是它在比较中相对于另一个项目的位置。

### A Simple Example
### 一个简单的例子

Consider three candidate answers generated by a language model: A, B, and C. Suppose human annotators produce the following preferences:
考虑由语言模型生成的三个候选答案：A、B 和 C。假设人类标注者产生了以下偏好：

* A is preferred to B
* A 优于 B
* A is preferred to C
* A 优于 C
* B is preferred to C
* B 优于 C

Even without any numeric ratings, a structure is already visible. A appears strongest, B next, and C weakest. The Bradley-Terry model formalizes this intuition by finding latent strengths that make these observed outcomes plausible under the model. This is the first conceptual step worth noticing. The model does not begin with global scores and then derive pairwise outcomes. It does the reverse. It begins with local comparisons and infers the latent scores that best explain them.
即使没有任何数字评分，结构也已经显而易见。A 看起来最强，B 次之，C 最弱。Bradley-Terry 模型通过寻找使这些观测结果在模型下合理的潜在强度，将这种直觉形式化。这是值得注意的第一个概念步骤。该模型不是从全局分数开始然后推导出两两结果，而是反其道而行之。它从局部比较开始，推断出最能解释这些比较的潜在分数。

### Fitting the Model From Data
### 从数据中拟合模型

Now suppose that comparisons are repeated many times across a larger collection of items. For each ordered pair $(i, j)$, let $w_{ij}$ denote the number of times that item $i$ beats item $j$, and let $w_{ji}$ denote the number of times that $j$ beats $i$. The Bradley-Terry model fits the parameters by choosing values of the strengths that make the observed comparison data as likely as possible. This is done through maximum likelihood estimation.
现在假设在更大的项目集合中重复进行多次比较。对于每一对有序对 $(i, j)$，令 $w_{ij}$ 表示项目 $i$ 击败项目 $j$ 的次数，$w_{ji}$ 表示 $j$ 击败 $i$ 的次数。Bradley-Terry 模型通过选择使观测到的比较数据可能性最大的强度值来拟合参数。这是通过最大似然估计完成的。

For a single pair of items $i$ and $j$, the likelihood contribution is:
对于单对项目 $i$ 和 $j$，其似然贡献为：

$$L_{ij} = P(i \succ j)^{w_{ij}} \cdot P(j \succ i)^{w_{ji}}$$

The interpretation is straightforward. If item $i$ beat item $j$ many times, then the fitted model should assign a high probability to $i$ beating $j$. If $j$ also won some comparisons, then the model should account for that as well. The likelihood rewards parameter settings that place high probability on the outcomes that were actually observed.
其解释非常直观。如果项目 $i$ 多次击败项目 $j$，那么拟合后的模型应该给 $i$ 击败 $j$ 分配一个高概率。如果 $j$ 也赢得了一些比较，那么模型也应该考虑到这一点。似然函数会奖励那些对实际观测到的结果赋予高概率的参数设置。

Across all item pairs, the full likelihood is obtained by multiplying these terms together. In practice, one works instead with the log-likelihood, because it is easier to optimise. The log-likelihood is:
在所有项目对中，通过将这些项相乘得到总似然。在实践中，人们通常使用对数似然，因为它更容易优化。对数似然为：

$$\ell = \sum_{i < j} (w_{ij} \log P(i \succ j) + w_{ji} \log P(j \succ i))$$

The fitting problem is then to find the parameter values that maximise this quantity.
拟合问题就是找到使该量最大化的参数值。

### A Deeper Look at Bradley-Terry Model Fitting
### 深入了解 Bradley-Terry 模型拟合

At an intuitive level, the optimization process adjusts the latent strengths so that the model’s predicted probabilities align with the empirical comparison outcomes. If an item wins frequently, its strength should rise. If it loses frequently, its strength should fall. If two items split their contests roughly evenly, their strengths should move closer together. These are the informal consequences.
从直观层面来看，优化过程会调整潜在强度，使模型预测的概率与经验比较结果保持一致。如果一个项目频繁获胜，其强度应该上升。如果它频繁失败，其强度应该下降。如果两个项目在比赛中平分秋色，它们的强度应该趋于接近。这些是非正式的结论。

The technical mechanism behind them is the gradient of the log-likelihood. Using the parameterisation $\pi_i = \exp(\beta_i)$, the gradient with respect to $\beta_i$ can be written as:
其背后的技术机制是对数似然的梯度。使用参数化 $\pi_i = \exp(\beta_i)$，关于 $\beta_i$ 的梯度可以写为：

$$\frac{\partial \ell}{\partial \beta_i} = \sum_{j \neq i} (w_{ij} - (w_{ij} + w_{ji}) P(i \succ j))$$

This expression is the central learning signal in the Bradley-Terry model, and it has a very clean interpretation. The first term, $w_{ij}$, is the number of wins item $i$ actually achieved against item $j$. The second term, $(w_{ij} + w_{ji}) P(i \succ j)$, is the number of wins the current model expects item $i$ to achieve against item $j$.
