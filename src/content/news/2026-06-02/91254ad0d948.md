# Arch Linux: Breaking changes for all users of `varnish`, which is renamed to `vinyl-cache`
# Arch Linux：`varnish` 更名为 `vinyl-cache`，所有用户需注意破坏性变更

Breaking changes for all users of `varnish`, which is renamed to `vinyl-cache` 2026-05-25 - Sven-Hendrik Haase
`varnish` 更名为 `vinyl-cache`，所有用户需注意破坏性变更 2026-05-25 - Sven-Hendrik Haase

The Varnish project has renamed itself to Vinyl Cache. We followed this rename with a new vinyl-cache package. This upgrade results in breaking changes and users are advised to study these changes and how it affects them before following the replacement.
Varnish 项目已更名为 Vinyl Cache。我们随之推出了新的 `vinyl-cache` 软件包。此次升级会导致破坏性变更，建议用户在进行替换操作前，仔细研究这些变更及其影响。

All references to "varnish" have been changed to "vinyl" in all binaries and directories. At minimum, users will have to:
所有二进制文件和目录中对 "varnish" 的引用均已更改为 "vinyl"。用户至少需要执行以下操作：

* rename /etc/varnish to /etc/vinyl-cache
* 将 /etc/varnish 重命名为 /etc/vinyl-cache
* rename /var/lib/varnish to /var/lib/vinyl-cache
* 将 /var/lib/varnish 重命名为 /var/lib/vinyl-cache
* fix up ownership of files inside /var/lib/varnish
* 修复 /var/lib/varnish 内部文件的所有权
* user varnish becomes vinyl
* 用户 varnish 变更为 vinyl
* group varnish becomes vinyl
* 用户组 varnish 变更为 vinyl
* user varnishlog becomes vinyllog
* 用户 varnishlog 变更为 vinyllog
* user vcache remains the same
* 用户 vcache 保持不变
* disable the old varnish.service and varnishncsa.service systemd units
* 禁用旧的 varnish.service 和 varnishncsa.service systemd 单元
* enable the new vinyl-cache.service and vinylncsa.service systemd units
* 启用新的 vinyl-cache.service 和 vinylncsa.service systemd 单元

Meanwhile, the varnish package has been dropped from [extra]. We're not currently planning to maintain a new varnish package as it's a different upstream project.
与此同时，`varnish` 软件包已从 [extra] 仓库中移除。由于这是一个不同的上游项目，我们目前没有计划维护新的 `varnish` 软件包。
