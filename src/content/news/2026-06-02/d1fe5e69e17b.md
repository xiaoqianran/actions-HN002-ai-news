# OmniMem: Scalable and Adaptive Memory Retrieval for Long Video Generation
# OmniMem：用于长视频生成的自适应可扩展内存检索框架

**Abstract:** Autoregressive (AR) video generation extends videos by producing latent chunks sequentially, but scaling to long videos requires repeated access to a growing historical KV cache. Existing methods reduce this cost by truncating the KV cache or compressing it into implicit memory, but both lose explicit access to query-relevant historical details. 

**摘要：** 自回归（AR）视频生成通过顺序生成潜在片段来延长视频，但扩展到长视频时，需要反复访问不断增长的历史 KV 缓存。现有方法通过截断 KV 缓存或将其压缩为隐式内存来降低成本，但这两种方式都会丢失对查询相关历史细节的显式访问能力。

We propose OmniMem, an explicit full-range memory retrieval framework that performs sparse KV retrieval over the historical cache. To make this practical for chunk-based AR video generation, OmniMem addresses two issues: (i) local bias in sparse KV selection and (ii) Union Explosion in memory access. 

我们提出了 OmniMem，这是一个显式的全范围内存检索框架，可在历史缓存上执行稀疏 KV 检索。为了使其适用于基于片段的 AR 视频生成，OmniMem 解决了两个问题：(i) 稀疏 KV 选择中的局部偏差，以及 (ii) 内存访问中的“联合爆炸”（Union Explosion）问题。

Adaptive Window Exclusion removes local-window blocks from the selection candidates when sufficient long-range history is available, preserving the sparse budget for informative long-range retrieval. Query-Shared KV Selection reduces cross-query diversity, while Per-Head Scattered KV Access avoids expanding head-specific selections into a large selected KV buffer. This allows each attention head to retrieve non-contiguous KV blocks according to its own selection pattern. 

“自适应窗口排除”（Adaptive Window Exclusion）机制在有足够的长期历史记录时，会从选择候选集中移除局部窗口块，从而为信息丰富的长期检索保留稀疏预算。“查询共享 KV 选择”（Query-Shared KV Selection）减少了跨查询的多样性，而“逐头分散 KV 访问”（Per-Head Scattered KV Access）则避免了将特定于头的选择扩展为庞大的已选 KV 缓冲区。这使得每个注意力头都能根据其自身的选择模式检索非连续的 KV 块。

Experiments on long-video generation show that OmniMem improves Dynamic Degree by 52.3% and preserves strong consistency over strong baselines, while maintaining comparable memory usage.

在长视频生成上的实验表明，OmniMem 将动态度（Dynamic Degree）提升了 52.3%，并在保持相当内存使用量的情况下，相比强基线模型展现出了更优的一致性。
