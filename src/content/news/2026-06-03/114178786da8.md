# From Local App to Public Website in Minutes
# 从本地应用到公共网站：只需几分钟

In this article, I’ll show you 3 completely free ways to quickly deploy a simple web app so it’s publicly available on the internet. By “simple,” I mean a static web app, i.e., one that has no database back end, user accounts, or server-side logic. While this might seem a little bit limiting, there’s still a lot you can do with just HTML, CSS and JavaScript.
在本文中，我将向你展示 3 种完全免费的方法，可以快速部署一个简单的 Web 应用，使其在互联网上公开可用。所谓的“简单”，是指静态 Web 应用，即没有数据库后端、用户账户或服务器端逻辑的应用。虽然这看起来有些局限，但仅使用 HTML、CSS 和 JavaScript 依然可以实现很多功能。

Since the focus is on different deployment methods, I won’t spend much time on the web app itself. I wanted a small project that was straightforward, but still real enough to deploy and share. So I had Codex build a browser game, a simple Space Invaders-style shooter which the agent decided to call Starline Defender.
由于重点在于不同的部署方法，我不会花太多时间介绍 Web 应用本身。我想要一个既简单直接，又足够真实以供部署和分享的小项目。因此，我让 Codex 构建了一个浏览器游戏，这是一款简单的“太空侵略者”风格射击游戏，AI 将其命名为“Starline Defender”。

The game is easy to play. You move a little ship left and right with the arrow keys, shoot incoming aliens using the space-bar, hide behind shields, lose lives when you get hit, and move through faster waves as the score climbs. It works with keyboard controls on desktop and touch controls on mobile. I’ll leave links to all three app URLs I deployed to at the end, so you can play with them yourself.
这款游戏很容易上手。你通过方向键左右移动小飞船，使用空格键射击来袭的外星人，躲在护盾后，被击中会损失生命值，随着分数提高，外星人的进攻波次会越来越快。它支持桌面端的键盘控制和移动端的触摸控制。我会在文末附上我部署的三个应用链接，方便你亲自体验。

### General Pre-requisites
### 通用前提条件

Most static hosts expect the homepage to be named index.html, so the first thing to do is rename our file from starline-defenders.html. For this project, that’s all we need, as all our assets and code are in this one file. Create an empty folder on your local system e.g starline-defenders/ and copy the index.html file there.
大多数静态托管服务都要求主页命名为 `index.html`，所以第一步是将我们的文件从 `starline-defenders.html` 重命名。对于本项目，这就是我们需要做的全部，因为所有的资源和代码都在这一个文件中。在本地系统创建一个空文件夹（例如 `starline-defenders/`），并将 `index.html` 文件复制到其中。

### Method 1: Deploy with GitHub Pages
### 方法一：使用 GitHub Pages 部署

GitHub Pages is a classic choice for small static sites and web apps like this. If your project already lives in GitHub, this is usually the simplest permanent deployment. GitHub Pages can serve HTML, CSS, JavaScript, images, and other static assets directly from a repository. For a one-file game, like this, it’s all the infrastructure you need.
GitHub Pages 是小型静态网站和此类 Web 应用的经典选择。如果你的项目已经托管在 GitHub 上，这通常是最简单的永久部署方式。GitHub Pages 可以直接从仓库中提供 HTML、CSS、JavaScript、图像和其他静态资源。对于像这样单文件的游戏，这就是你所需的全部基础设施。

**What You Need**
**你需要准备：**

*   A GitHub account (一个 GitHub 账户)
*   Git Bash installed locally (在本地安装 Git Bash)
*   A public GitHub repository. This is essential for free deployment. (一个公开的 GitHub 仓库。这对免费部署至关重要。)

**Step 1: Create a GitHub account**
**第一步：创建 GitHub 账户**
Go to https://github.com and click the Sign up button at the top right of the screen.
前往 https://github.com 并点击屏幕右上角的“Sign up”按钮。

**Step 2: Install the git bash app**
**第二步：安装 Git Bash 应用**
Git, via its Bash command-line interface, enables you to interact with GitHub, e.g., copy files to repositories. Get it by clicking: https://git-scm.com/book/en/v2/Getting-Started-Installing-Git and following the installation instructions appropriate to your local system.
Git 通过其 Bash 命令行界面使你能够与 GitHub 进行交互，例如将文件复制到仓库中。点击 https://git-scm.com/book/en/v2/Getting-Started-Installing-Git 并按照适合你本地系统的安装说明进行操作。

**Step 3: Create a New Repository**
**第三步：创建新仓库**
Log in to your GitHub account and create a new repository called, for example: starline-defenders. Make sure it’s a public repository.
登录你的 GitHub 账户并创建一个新仓库，例如命名为：`starline-defenders`。确保将其设置为公开仓库。

**Step 4: Put your index.html file into the public repository**
**第四步：将 index.html 文件放入公开仓库**
You do that by opening the Git Bash command-line terminal, then changing directory to the local folder containing your index.html file. Now type the following commands into Git Bash:
打开 Git Bash 命令行终端，然后切换到包含 `index.html` 文件的本地文件夹。现在在 Git Bash 中输入以下命令：

```bash
git init
git config --global user.email "[email protected]"
git config --global user.name "Your Name"
git add index.html
git commit -m "Add space invaders type game"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/starline-defenders.git
git push -u origin main
```
*(Replace `YOUR_USERNAME` with your GitHub username.)*
*（将 `YOUR_USERNAME` 替换为你的 GitHub 用户名。）*

**Step 5: Enable GitHub Pages**
**第五步：启用 GitHub Pages**
Open the new repository on GitHub, then:
在 GitHub 上打开新仓库，然后：
1. Go to **Settings**. (进入“Settings”)
2. Open **Pages** in the left sidebar. (在左侧边栏打开“Pages”)
3. Under **Build and deployment**, choose **Deploy from a branch**. (在“Build and deployment”下，选择“Deploy from a branch”)
4. Set the source branch to `main`. (将源分支设置为 `main`)
5. Set the folder to `/root`. (将文件夹设置为 `/root`)
6. Click **Save**. (点击“Save”)

GitHub will start publishing the site. It usually takes a minute or two, and your final URL will look like this: `https://YOUR_USERNAME.github.io/starline-defenders/`
GitHub 将开始发布网站。通常需要一两分钟，你的最终 URL 将如下所示：`https://YOUR_USERNAME.github.io/starline-defenders/`
