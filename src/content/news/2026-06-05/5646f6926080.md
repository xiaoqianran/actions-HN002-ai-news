在上一篇文章中，我讨论了 YOLOv3 [1]。该版本优于前代产品的原因之一，在于它采用了类似 FPN 的颈部（neck）结构，从而具备了检测小目标的能力。遗憾的是，由于当时我主要关注 YOLOv3 本身，对 FPN 的解释不够详尽。因此，在本文中，我决定专门根据其原始论文《用于目标检测的特征金字塔网络》（Feature Pyramid Networks for Object Detection）[2] 来撰写关于 FPN 的内容，以便让你更深入地理解它究竟是什么以及它是如何工作的。不仅如此，我还将演示如何从零开始实现 FPN，以及如何将其与 CNN 主干网络（backbone）和 RPN 头部（head）连接起来。

### Backbone, Neck, and Head
### 主干、颈部与头部

Before we get into FPN, we first need to know that the structure of an object detection model is different from that of the classification model, in which the main difference lies in the very last layer. In a typical classification model, the last layer comprises a number of neurons where each of them corresponds to every single class available in the dataset. Or, in the case of binary classification, the output layer only consists of a single neuron, which is responsible to predict whether a sample belongs to class 0 or 1. This kind of output layer is actually not suitable for detection task since it also requires neurons dedicated for predicting the location and the size of an object in addition to its class.
在深入了解 FPN 之前，我们首先需要知道目标检测模型的结构与分类模型不同，其主要区别在于最后一层。在典型的分类模型中，最后一层包含多个神经元，每个神经元对应数据集中的一个类别。或者在二分类的情况下，输出层仅由一个神经元组成，负责预测样本属于类别 0 还是 1。这种输出层实际上并不适合检测任务，因为检测任务除了需要预测类别外，还需要专门的神经元来预测目标的位置和大小。

So, in order for a model to be able to predict object location and size, we need to replace the output layer, i.e., the classification head, with the so-called detection head. The remaining layers themselves (everything except the head) are commonly called backbone. Some models that use this structure are YOLOv1 and YOLOv2, where they use a stack of convolution layers as the backbone and a specific head for predicting object location and size within an image as well as its class.
因此，为了使模型能够预测目标的位置和大小，我们需要用所谓的“检测头”（detection head）替换输出层（即分类头）。其余的层（除头部以外的所有部分）通常被称为“主干”（backbone）。采用这种结构的一些模型包括 YOLOv1 和 YOLOv2，它们使用堆叠的卷积层作为主干，并使用特定的头部来预测图像中目标的位置、大小及其类别。

Older object detection models like YOLOv1 and YOLOv2 mentioned above only consist of a backbone and a head. As time went on, researchers found that this structure is still not quite optimal, hence they finally came up with an idea by adding a new component called neck. As the name suggests, this is essentially something we place between the backbone and the head. And FPN, which we are going to talk about in this article, is one of the earliest necks proposed for object detection models. Look at the Figure 1 below to see the high-level architectural view of the older and modern object detection models.
上述提到的 YOLOv1 和 YOLOv2 等较早期的目标检测模型仅由主干和头部组成。随着时间的推移，研究人员发现这种结构并不完全理想，因此他们最终提出了增加一个名为“颈部”（neck）的新组件的想法。顾名思义，这本质上是我们放置在主干和头部之间的部分。而我们将在本文中讨论的 FPN，正是最早为目标检测模型提出的颈部结构之一。请看下图 1，了解早期和现代目标检测模型的高层架构视图。

*Figure 1. The architecture of an object detection model in general [3].*
*图 1. 目标检测模型的一般架构 [3]。*

The backbone of a model is mainly responsible for performing feature extraction, while the neck is useful for enhancing feature quality, and the head is for making predictions. Based on this notion, we can say that by applying FPN, a network can potentially achieve better accuracy thanks to the feature enhancement mechanism performed by the neck.
模型的主干主要负责执行特征提取，颈部用于增强特征质量，而头部则用于进行预测。基于这一概念，我们可以说，通过应用 FPN，网络可以利用颈部执行的特征增强机制，从而潜在地获得更高的准确率。

### The Evolution of Multi-Scale Detection Mechanism
### 多尺度检测机制的演进

Previously I mentioned that using backbone and detection head without neck is not quite optimal. This is especially regarding its capability in detecting small objects. Now let’s take a look at the Figure 2 below. The first two YOLO versions I mentioned earlier utilize the structure in image (b), where the bounding box and the object class predictions are made solely on top of the feature map produced by the deepest layer in the backbone.
之前我提到过，仅使用主干和检测头而不使用颈部并不完全理想，特别是在检测小目标的能力方面。现在让我们看看下图 2。我之前提到的前两个 YOLO 版本利用了图 (b) 中的结构，其中边界框和目标类别预测仅基于主干中最深层产生的特征图进行。

This method is valid, but is effective only for large objects. The reason is pretty straightforward: as an image gets deeper into a network, the spatial dimension shrinks, and more importantly, the pixel information contained in the deeper feature maps becomes a representation of several neighboring pixels in the shallower ones, which causes the spatial information to blend. By doing this, feature maps from deeper layers got a large receptive field, allowing large objects to be detected and recognized easily.
这种方法是有效的，但仅对大目标有效。原因很简单：当图像进入网络深层时，空间维度会缩小；更重要的是，深层特征图中包含的像素信息变成了浅层中多个相邻像素的表示，这导致了空间信息的融合。通过这种方式，深层的特征图获得了较大的感受野，从而可以轻松检测和识别大目标。

However, the degradation of spatial information as we get deeper prevents us from detecting small objects accurately because we do need a detailed pixel location in order to predict the exact coordinates of the objects. Additionally, the receptive field size of a feature map is positively correlated with the amount of semantic information it contains. In the figure below, a feature map of high semantic information is indicated by a thick blue outline. This is essentially why the deepest feature map in (b) has the thickest outline.
然而，随着深度增加，空间信息的退化阻碍了我们准确检测小目标，因为我们需要详细的像素位置来预测目标的精确坐标。此外，特征图的感受野大小与其包含的语义信息量呈正相关。在下图中，高语义信息的特征图用粗蓝色轮廓表示。这本质上就是为什么 (b) 中最深层的特征图具有最粗轮廓的原因。

*Figure 2. Comparison of different feature pyramid architectures [2].*
*图 2. 不同特征金字塔架构的比较 [2]。*

The most straightforward approach to allow a network to simultaneously detect large and small objects is by using featurized image pyramid (a). This method is able to achieve high accuracy because we can make predictions from different image resolutions. What’s essentially done here is that we rescale our input image into multiple scales, perform feature extraction independently on each scale, and make predictions on the resulting feature maps. The smaller feature map is responsible to detect large objects, whereas the larger one is specialized to detect small objects thanks to its detailed spatial information. However, this method is computationally expensive since we need to process multiple raw images of different scales at once.
使网络能够同时检测大目标和小目标的最直接方法是使用特征化图像金字塔 (a)。这种方法能够实现高精度，因为我们可以从不同的图像分辨率进行预测。其本质做法是将输入图像缩放到多个尺度，在每个尺度上独立执行特征提取，并对生成的特征图进行预测。较小的特征图负责检测大目标，而较大的特征图由于其详细的空间信息，专门用于检测小目标。然而，这种方法计算成本很高，因为我们需要同时处理多个不同尺度的原始图像。

Another solution was proposed by the authors of SSD (Single Shot Multibox Detector), which in Figure 2 above is the one referred to as pyramidal feature hierarchy (c). So, instead of feeding the network with the same image of different sizes, the authors of SSD attempted to use only the largest image and utilize the internal pyramidal structure of the CNN backbone to make predictions of varying scales.
SSD（Single Shot Multibox Detector）的作者提出了另一种解决方案，即上图 2 中提到的金字塔特征层级 (c)。因此，SSD 的作者没有向网络输入不同大小的同一图像，而是尝试仅使用最大的图像，并利用 CNN 主干的内部金字塔结构来进行不同尺度的预测。
