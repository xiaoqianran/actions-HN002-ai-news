# Automate Writing Your LLM Prompts
# 自动化编写你的 LLM 提示词

Large Language Models Automate Writing Your LLM Prompts Using DSPy to automatically create, evaluate, and optimize your prompts.
大型语言模型（LLM）通过使用 DSPy 自动创建、评估和优化你的提示词，从而实现提示词编写的自动化。

Working with LLMs, we’ve probably all had the experience of getting responses that weren’t quite what we wanted. Usually we’ll try rewording the prompts a few times until we get something reasonable. We sometimes have to be more clear, more precise, give examples, describe why we need the response, present a persona, or otherwise provide enough context and information that the LLM is able to provide a suitable response.
在使用大语言模型时，我们可能都有过这样的经历：得到的回复并不完全符合预期。通常，我们会尝试多次修改提示词，直到得到满意的结果。有时我们需要表达得更清晰、更精确，提供示例，说明我们需要该回复的原因，设定一个角色，或者提供足够的背景信息，以便 LLM 能够给出合适的回答。

This can be fine when we’re working directly with the LLM. However, it’s quite different when we’re writing an LLM-based application — software that will execute on its own, and that doing so will interact with one or more LLMs. Here, the software will work with predefined prompts and will pass these to the LLMs. If it doesn’t go well, we’re not there to reword the prompts and try again. Which means, they have to be written in a way that’s robust and reliable in the first place — we need prompts that we can be confident will work consistently well in production.
当我们直接与 LLM 交互时，这样做没问题。然而，当我们编写基于 LLM 的应用程序时，情况就大不相同了——软件需要自主运行，并与一个或多个 LLM 进行交互。在这种情况下，软件使用预定义的提示词并将其传递给 LLM。如果效果不佳，我们无法现场修改提示词并重试。这意味着，提示词必须从一开始就编写得稳健且可靠——我们需要确信这些提示词在生产环境中能够始终如一地发挥作用。

Creating such a prompt can be tricky. In this article, we’ll go over why that is, and also how a Python tool called DSPy can support creating prompts that will be reliable. DSPy not only generates prompts automatically for you, it also evaluates them thoroughly, so you can be confident of how well they’ll likely work in production.
创建这样的提示词可能很棘手。在本文中，我们将探讨其原因，以及一个名为 DSPy 的 Python 工具如何帮助创建可靠的提示词。DSPy 不仅能为你自动生成提示词，还能对其进行全面评估，让你确信它们在生产环境中的表现。

I’ll also provide an excerpt from my most recent book with Manning Publishing, *Building LLM Applications with DSPy*, co-authored with Serj Smorodinsky. That provides a complete description of DSPy and how to use it to create LLM-based applications.
我还将提供我与 Manning Publishing 合作的最新著作《Building LLM Applications with DSPy》（与 Serj Smorodinsky 合著）的节选。该书全面介绍了 DSPy 以及如何使用它来构建基于 LLM 的应用程序。

### The trick of creating a prompt that can work reliably in production
### 在生产环境中可靠运行提示词的诀窍

Part of what makes it difficult to create a reliable prompt is that we can’t fully predict the input we’ll have for the prompt. Say, for example, we’re creating a software application that will process documents. The documents may be found online, or possibly submitted by users of the software. As part of processing the documents, the application may ask an LLM to summarize them, translate them, extract key pieces of information, or to perform some other such task.
创建可靠提示词的难点之一在于，我们无法完全预测提示词将接收到的输入。例如，假设我们正在开发一个处理文档的软件应用程序。这些文档可能来自网络，也可能是由软件用户提交的。作为文档处理的一部分，应用程序可能会要求 LLM 对其进行总结、翻译、提取关键信息或执行其他类似任务。

For this example, let’s say the software will ask the LLM to critique how plausible the content in the documents appears to be. To do that we may write a prompt such as: `prompt_text = f"Assess how plausible the following text is: {document_text}"`
以这个例子为例，假设软件要求 LLM 评估文档内容的合理性。为此，我们可能会编写如下提示词：`prompt_text = f"Assess how plausible the following text is: {document_text}"`

That uses a Python f-string to form the prompt, with a slot for the text of the document. Other prompts may have multiple slots for the inputs, but for simplicity, we’ll assume here that each prompt has just one input — the piece of content you’ll want the LLM to process (which is the part that’s unpredictable).
这里使用了 Python 的 f-string 来构建提示词，并为文档内容预留了一个插槽。其他提示词可能包含多个输入插槽，但为了简化起见，我们假设每个提示词只有一个输入——即你希望 LLM 处理的内容（这部分内容是不可预测的）。

This prompt may work sufficiently well, but it also may not. There are any number of ways the LLM may respond in a way we don’t like, at least occasionally. We may find that the LLM picks up on irrelevant details in the documents. Or may have a different sense of ‘plausible’ than we intended. Or it may indicate almost every document is fully plausible (or the opposite, that almost none are). Or the responses may not be formatted as we wish. We may need to tweak the prompt to consistently get the responses we would expect.
这个提示词可能效果不错，但也可能不然。LLM 至少在某些时候可能会以我们不希望的方式做出回应。我们可能会发现 LLM 关注了文档中无关紧要的细节，或者对“合理性”的理解与我们的初衷不同。又或者，它可能认为几乎所有文档都是完全合理的（反之亦然，认为几乎都不合理）。或者回复的格式不符合我们的要求。我们可能需要调整提示词，以持续获得预期的回复。

To get started, we can try this and a few other simple prompts, but the final prompt may end up being considerably longer and more detailed that this. Usually, as we test with more inputs (in this case, more documents), we’ll find more cases where the current prompt doesn’t handle the input well, so we’ll tweak the prompt to handle these cases better. Sometimes we may reword the prompt to be more clear, and other times add some sentences to the prompt to handle these specific cases.
起初，我们可以尝试这个以及其他几个简单的提示词，但最终的提示词可能会比这长得多且详细得多。通常，随着我们测试的输入（在本例中为更多文档）增加，我们会发现当前提示词无法很好处理的情况，因此我们会调整提示词以更好地应对这些情况。有时我们可能会重写提示词以使其更清晰，有时则会在提示词中添加一些句子来处理这些特定情况。

For example, “If the document makes claims that are metaphorical, assess the general intent and not the literal meaning.” We can end up with any number of extra instructions like this in the prompt, which can help the prompt work well for these cases, but, of course, can also cause the prompt to work worse for other inputs. And, as the prompts get longer and more complicated, they can get harder to tweak. It can get less and less clear what the effect will be of adding, removing, re-ordering, or re-wording phrases in the prompt will be.
例如：“如果文档中的陈述是隐喻性的，请评估其总体意图而非字面意思。”我们最终可能会在提示词中添加许多此类额外指令，这有助于提示词在这些情况下表现良好，但当然，也可能导致它在处理其他输入时表现变差。而且，随着提示词变得越来越长、越来越复杂，它们会变得更难调整。添加、删除、重新排序或重写提示词中的短语会产生什么影响，将变得越来越难以预料。

Other LLM-based applications may work with other types of text data: text messages, emails, essays, journal articles, patent applications, and so on. Or may process image, audio, video, or other modalities. But, regardless of the type of input, for a non-trivial application, the specific input the application encounters (and passes on to the LLM) will be at least somewhat unpredictable. Which means, we’ll need a robust, well-specified prompt to handle a wide range of realistic input.
其他基于 LLM 的应用程序可能会处理不同类型的文本数据：短信、电子邮件、论文、期刊文章、专利申请等。或者可能处理图像、音频、视频或其他模态。但是，无论输入类型如何，对于非简单的应用程序，应用程序遇到的（并传递给 LLM 的）特定输入至少在一定程度上是不可预测的。这意味着，我们需要一个稳健、定义明确的提示词来处理各种现实输入。

The only way to test that your application will work reliably in production is to test with a large, diverse, and realistic set of inputs (in this case, a large, diverse collection of realistic emails). The LLM may classify the text in terms of identifying the language (English, French, etc.), the sentiment, toxicity, and so on.
