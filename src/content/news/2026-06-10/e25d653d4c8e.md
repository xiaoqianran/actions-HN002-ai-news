# Enabling KV Caching of Shared Prefix for Diffusion Language Models
# 为扩散语言模型启用共享前缀的 KV 缓存

**Abstract:** Key-value (KV) caching for shared prefixes is essential for high-throughput large language model (LLM) serving, but it faces critical challenges in emerging diffusion language models (DLMs). In DLMs, bidirectional attention means that updating any token dynamically alters the entire context and its corresponding KVs. Thus, existing caching techniques developed for LLMs, which assume that KVs remain invariant once computed, corrupt the shared prefix KVs. Our experiments show that applying these techniques to DLMs causes model accuracy to collapse to near zero.

**摘要：** 针对共享前缀的键值（KV）缓存对于大语言模型（LLM）的高吞吐量服务至关重要，但在新兴的扩散语言模型（DLM）中却面临严峻挑战。在 DLM 中，双向注意力机制意味着更新任何一个 token 都会动态改变整个上下文及其对应的 KV 值。因此，为 LLM 开发的现有缓存技术（假设 KV 一旦计算便保持不变）会破坏共享前缀的 KV。我们的实验表明，将这些技术应用于 DLM 会导致模型准确率几乎归零。

To unlock high-throughput DLM serving, we propose bidirectional prefix caching, bicache, the first KV caching technique for shared prefixes in DLMs. bicache is designed based on key observations from our comprehensive analysis: shared prefix KVs remain stable and reusable in shallow layers, while the depth of shallow layers depends on the fraction of shared prefix tokens in each request. Thus, bicache dynamically identifies a safe layer depth for reusing shared prefix KVs and eliminates redundant computation.

为了实现高吞吐量的 DLM 服务，我们提出了双向前缀缓存技术——bicache，这是首个针对 DLM 中共享前缀的 KV 缓存技术。bicache 的设计基于我们全面分析得出的关键观察结果：共享前缀的 KV 在浅层中保持稳定且可重用，而浅层的深度取决于每个请求中共享前缀 token 的比例。因此，bicache 能够动态识别重用共享前缀 KV 的安全层深度，从而消除冗余计算。

Evaluations demonstrate that bicache significantly improves serving throughput by 36.3%-98.3% compared to existing techniques without accuracy collapse (only 0-1.8% difference).

评估结果表明，与现有技术相比，bicache 在不导致准确率崩溃的情况下（仅有 0-1.8% 的差异），显著提升了 36.3%-98.3% 的服务吞吐量。
