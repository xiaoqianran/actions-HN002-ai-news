# Google Chrome is closing the loopholes that let old ad blockers keep working
# 谷歌 Chrome 正在封堵允许旧版广告拦截器继续运行的漏洞

Two Chrome updates coming later this summer will remove workarounds for running ad blockers like uBlock Origin.
今年夏天晚些时候推出的两次 Chrome 更新将移除运行 uBlock Origin 等广告拦截器的变通方法。

Google Chrome version 150 and 151, expected in late June and July, respectively, will cut off support for the last remaining workarounds for running older ad blockers, 9to5Google reports. Google phased out support for ad-blocking extensions built for Manifest V2, like uBlock Origin, in 2024. At that point, most Chrome users either switched to newer Manifest V3 ad blockers like uBlock Origin Lite or switched to a different browser.
据 9to5Google 报道，预计分别于 6 月下旬和 7 月发布的 Google Chrome 150 和 151 版本，将切断运行旧版广告拦截器最后剩余的变通方法。谷歌在 2024 年逐步淘汰了对基于 Manifest V2 构建的广告拦截扩展程序（如 uBlock Origin）的支持。当时，大多数 Chrome 用户要么切换到了较新的 Manifest V3 广告拦截器（如 uBlock Origin Lite），要么换用了其他浏览器。

Google is now removing the final bits of code referencing its previous Manifest V2 extensions platform, so only ad blockers that work on the Manifest V3 platform will work in Chrome version 151 onward.
谷歌目前正在移除引用其先前 Manifest V2 扩展平台的最后一部分代码，因此从 Chrome 151 版本开始，只有在 Manifest V3 平台上运行的广告拦截器才能正常工作。

Google developer Devlin Cronin commented on the Chromium commit calling for the removal of the last references to Manifest V2, explaining:
谷歌开发者 Devlin Cronin 在 Chromium 的提交记录中评论了移除 Manifest V2 最后相关引用的决定，并解释道：

MV2 extensions are no longer allowed in any supported version of Chrome, and we are removing support for them and the associated functionality. We won’t be able to provide / maintain this functionality indefinitely due to the complexity and tech debt, as well as the security risks it entails (we’ve actually found a number of bugs that are specific to MV2 lately). Of course, other browsers can continue supporting these if they so desire.
“MV2 扩展程序在任何受支持的 Chrome 版本中均不再被允许，我们正在移除对它们及相关功能的支持。由于其复杂性、技术债务以及所带来的安全风险（我们最近确实发现了许多 MV2 特有的漏洞），我们无法无限期地提供/维护此功能。当然，其他浏览器如果愿意，可以继续支持这些扩展。”
