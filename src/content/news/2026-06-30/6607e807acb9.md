# How Far Can Classical NLP Go? From Bag-of-Words to Stacking on Spooky Author Identification
# 经典自然语言处理能走多远？从词袋模型到堆叠模型在“幽灵作者识别”任务中的应用

An end-to-end classical NLP experiment on Kaggle’s Spooky Author Identification task.
这是一项关于 Kaggle“幽灵作者识别”任务的端到端经典自然语言处理实验。

Authorship attribution is a good way to test NLP models because it focuses not only on what a sentence says, but also on how it is written. Kaggle’s Spooky Author Identification competition is a compact version of this challenge: given a single sentence from gothic or horror fiction, the model has to predict whether it was written by Edgar Allan Poe (EAP), Mary Wollstonecraft Shelley (MWS), or H. P. Lovecraft (HPL).
作者归属分析是测试自然语言处理模型的好方法，因为它不仅关注句子的内容，还关注其写作方式。Kaggle 的“幽灵作者识别”竞赛是这一挑战的精简版：给定哥特式或恐怖小说中的一个句子，模型必须预测它是由埃德加·爱伦·坡 (EAP)、玛丽·雪莱 (MWS) 还是 H. P. 洛夫克拉夫特 (HPL) 所写。

At first, this seems like a typical three-class text classification problem. But in reality, it is more complex. The authors all write about similar themes: fear, mystery, death, atmosphere, and the supernatural. Simple keywords are not enough to tell them apart. Instead, the important clues are often stylistic: function words, punctuation, character patterns, short phrases, sentence rhythm, and the way each author builds a sentence.
起初，这看起来像是一个典型的三分类文本问题。但实际上，它要复杂得多。这些作者都围绕相似的主题写作：恐惧、神秘、死亡、氛围和超自然现象。简单的关键词不足以区分他们。相反，重要的线索往往是文体特征：功能词、标点符号、字符模式、短语、句子节奏以及每位作者构建句子的方式。

This made the project a good way to explore a specific question: How far can classical NLP go when we choose representations carefully and evaluate them honestly? I approached the task by building a sequence of increasingly capable classical models: a fast Vowpal Wabbit word baseline, a richer VW model with punctuation and character n-grams, a tuned TF-IDF ensemble, a stacked sparse-text ensemble using out-of-fold predictions, a small representation survey comparing sparse features, BM25, Word2Vec, and FastText.
这使得该项目成为探索特定问题的绝佳途径：当我们仔细选择特征表示并诚实地进行评估时，经典自然语言处理究竟能走多远？我通过构建一系列能力递增的经典模型来完成这项任务：一个快速的 Vowpal Wabbit (VW) 词基准模型、一个包含标点符号和字符 n-gram 的更丰富的 VW 模型、一个经过调优的 TF-IDF 集成模型、一个使用袋外预测（out-of-fold）的稀疏文本堆叠集成模型，以及一个比较稀疏特征、BM25、Word2Vec 和 FastText 的小型特征表示调研。

### Dataset and Evaluation Setup
### 数据集与评估设置

The dataset contains 19,579 labeled training sentences and 8,392 unlabeled test sentences. The class distribution is mildly imbalanced.
该数据集包含 19,579 个带标签的训练句子和 8,392 个无标签的测试句子。类别分布略有不平衡。

I encoded the labels as 1-based integers because Vowpal Wabbit’s One-Against-All multiclass mode expects labels starting at 1.
我将标签编码为从 1 开始的整数，因为 Vowpal Wabbit 的“一对多”（One-Against-All）多分类模式要求标签从 1 开始。

```python
train_texts = pd.read_csv(DATA_DIR / "train.csv", index_col="id")
test_texts = pd.read_csv(DATA_DIR / "test.csv", index_col="id")
AUTHOR_CODE = {"EAP": 1, "MWS": 2, "HPL": 3}
train_texts["author_code"] = train_texts["author"].map(AUTHOR_CODE)
```

To compare models locally, I used a single stratified 70/30 train-validation split with a fixed random seed. This kept the class proportions stable and ensured that every model was evaluated on the same held-out examples.
为了在本地比较模型，我使用了单一的分层 70/30 训练-验证集划分，并固定了随机种子。这保持了类别比例的稳定，并确保每个模型都在相同的留出样本上进行评估。

I focused on three main metrics:
我重点关注三个主要指标：
*   **Accuracy**: straightforward to understand, but it only measures the final top-class decision.
    **准确率 (Accuracy)**：易于理解，但仅衡量最终的最高概率类别决策。
*   **Macro-F1**: useful for checking whether performance is balanced across the three authors.
    **宏平均 F1 (Macro-F1)**：用于检查三位作者之间的表现是否平衡。
*   **Multiclass log loss**: the official Kaggle metric and the most important metric for this project, because it evaluates the quality of the predicted probabilities, not just the predicted class.
    **多分类对数损失 (Multiclass log loss)**：这是 Kaggle 的官方指标，也是本项目最重要的指标，因为它评估的是预测概率的质量，而不仅仅是预测的类别。

### 1. Word-only Vowpal Wabbit baseline
### 1. 仅使用词特征的 Vowpal Wabbit 基准模型

I started with Vowpal Wabbit because it is fast, handles sparse data well, and is well-suited to linear text models. VW trains online linear models, hashes features into a fixed feature space, and handles multiclass classification through One-Against-All. For the first baseline, I used only lowercased word features of length three or more.
我从 Vowpal Wabbit 开始，因为它速度快，能很好地处理稀疏数据，并且非常适合线性文本模型。VW 训练在线线性模型，将特征哈希到固定的特征空间，并通过“一对多”方式处理多分类。对于第一个基准模型，我仅使用了长度为 3 或以上的全小写词特征。

On the 70/30 holdout split, the word-only VW baseline reached a strong result for a fast linear model. It also established a useful baseline: any added representation or ensemble layer needed to clear this bar.
在 70/30 的留出集上，仅使用词特征的 VW 基准模型对于一个快速线性模型来说表现强劲。它也建立了一个有用的基准：任何后续添加的特征表示或集成层都必须超越这一水平。

### 2. 丰富的 VW 模型：添加文体感知特征
