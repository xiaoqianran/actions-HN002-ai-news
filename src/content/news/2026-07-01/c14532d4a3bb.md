# Position: RL Researchers Need to Distinguish Between Solving Simulators and Using Simulators as a Proxy
# 观点：强化学习研究者需要区分“解决模拟器”与“将模拟器作为代理”

**Abstract:** One goal in reinforcement learning (RL) research is to understand general-purpose sequential decision-making, using benchmark simulators as a proxy for learning in deployment settings. When running experiments, however, the goal of achieving high performance in the simulator can mutate into focusing exclusively on solving the simulator. To achieve high scores, researchers may adopt solutions exclusively meant for solving simulators, rather than learning while the agent is deployed outside a simulator. Solving simulators is also worthy of investigation, but it is a fundamentally different RL research question.

**摘要：** 强化学习（RL）研究的目标之一是理解通用序列决策，并利用基准模拟器作为在实际部署环境中学习的代理（Proxy）。然而，在进行实验时，追求模拟器内的高性能目标可能会演变成仅仅专注于“解决模拟器”本身。为了获得高分，研究人员可能会采用仅适用于解决模拟器的方案，而不是在智能体部署于模拟器之外的环境中进行学习。解决模拟器本身也值得研究，但这是一个本质上完全不同的强化学习研究课题。

In this paper, we argue that RL researchers need to distinguish between two use cases of simulators: solving simulators and using simulators as a proxy for learning in deployment. We first discuss how these two use-cases are importantly different, in terms of constraints on how the agent can use the simulator, which algorithms are appropriate, and which evaluation metrics are appropriate. We then highlight several issues and misleading conclusions that can occur by not making the distinction between these two settings clear, supported with examples and simple experiments.

在本文中，我们认为强化学习研究者需要区分模拟器的两种使用场景：解决模拟器，以及将模拟器作为部署学习的代理。我们首先讨论了这两种使用场景在智能体如何使用模拟器的约束条件、适用的算法以及适用的评估指标方面存在的重要差异。随后，我们通过示例和简单的实验，强调了若不明确区分这两种设置，可能会导致的问题及误导性结论。

This work is a call to the community to begin clearly distinguishing how they are using simulators in their work, hopefully sparking further discussion on which empirical practices work best in each setting.

这项工作旨在呼吁学术界开始明确区分其研究中如何使用模拟器，并希望以此激发关于在不同场景下哪些实证实践效果最佳的进一步讨论。
