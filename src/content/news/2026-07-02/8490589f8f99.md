# Build and Run Your Own AI Agent in the Cloud
# 在云端构建并运行你自己的 AI Agent

Using AI on AWS to perform useful work can be relatively straightforward. For a simple application, a few lines of Python using the boto3 library and the Bedrock API may be all that’s needed. You set up access to an LLM, send a prompt and some input, receive a response, and return it to the user.
在 AWS 上利用 AI 执行实用任务可以非常简单。对于一个简单的应用程序，使用 boto3 库和 Bedrock API 编写几行 Python 代码可能就足够了。你只需设置对大语言模型（LLM）的访问，发送提示词（Prompt）和输入，接收响应，然后将其返回给用户即可。

That approach becomes harder to manage if your application has to take on more responsibilities. Once the model needs to maintain conversation context, choose between tools, follow detailed instructions, or coordinate several steps, the surrounding application code begins to resemble an agent framework. That’s where the use of Strands and AgentCore comes into play.
如果你的应用程序需要承担更多职责，这种方法就会变得难以管理。一旦模型需要维护对话上下文、在不同工具间进行选择、遵循详细指令或协调多个步骤，周围的应用程序代码就开始变得像一个 Agent 框架。这时，Strands 和 AgentCore 的作用就体现出来了。

Strands provides that agent layer, but running an agent reliably introduces a different set of concerns. It needs somewhere to run, an invocation interface, session isolation, scaling, security, and potentially services such as long-term memory or managed access to tools. Amazon Bedrock AgentCore provides these operational capabilities without defining how the agent itself should behave.
Strands 提供了这一 Agent 层，但要可靠地运行一个 Agent，则会引入一系列不同的考量。它需要运行环境、调用接口、会话隔离、扩展性、安全性，以及可能需要的长期记忆或托管工具访问等服务。Amazon Bedrock AgentCore 提供了这些操作能力，而无需定义 Agent 本身应如何表现。

***

### Strands
### Strands

Strands is an open-source agent framework from AWS. It provides the application-level components needed to create an agent:
Strands 是 AWS 推出的开源 Agent 框架。它提供了创建 Agent 所需的应用级组件：

*   An LLM, such as Claude Sonnet, through Amazon Bedrock or local models via Ollama and other providers.
*   一个大语言模型（LLM），例如通过 Amazon Bedrock 使用的 Claude Sonnet，或通过 Ollama 等提供商使用的本地模型。
*   A system prompt that defines the agent’s role and behaviour.
*   定义 Agent 角色和行为的系统提示词（System Prompt）。
*   Tools the model can choose to call.
*   模型可以选择调用的工具。
*   Conversation messages and context.
*   对话消息和上下文。
*   An agent loop that sends requests to the model, executes requested tools, and returns tool results to the model.
*   一个 Agent 循环，负责向模型发送请求、执行所请求的工具，并将工具执行结果返回给模型。

Think of Strands as the AWS equivalent to LangChain or CrewAI. A basic Strands agent can be created with just a model and a system prompt:
你可以将 Strands 视为 AWS 版本的 LangChain 或 CrewAI。一个基础的 Strands Agent 只需一个模型和一个系统提示词即可创建：

```python
from strands import Agent
model = 'your_LLM_choice'
agent = Agent(
    model=model,
    system_prompt="You are an educational SME assistant.",
)
response = agent("Explain Newton's first law.")
```

Although I’m framing this article as a Strands/AgentCore double team, Strands itself doesn’t require AgentCore to run. A Strands agent is ordinary application code and can run locally or on other infrastructure. AgentCore comes in when the agent needs managed deployment on AWS infrastructure and production services, such as memory, scalability and observability.
尽管我将本文定位为 Strands 与 AgentCore 的“双剑合璧”，但 Strands 本身并不依赖 AgentCore 运行。Strands Agent 本质上是普通的应用程序代码，可以在本地或其他基础设施上运行。只有当 Agent 需要在 AWS 基础设施上进行托管部署，并使用内存、可扩展性和可观测性等生产级服务时，才需要用到 AgentCore。

### The agent we’re building
### 我们要构建的 Agent

We’ll build and deploy a Strands-based educational assistant before adding AgentCore Memory to preserve user preferences between conversations. Our agent will support questions about: Mathematics, Physics, Chemistry, Geography.
我们将构建并部署一个基于 Strands 的教育助手，然后添加 AgentCore Memory 以在不同对话之间保留用户偏好。我们的 Agent 将支持以下领域的问题：数学、物理、化学、地理。

The model decides which subject best fits each question, so there’s no keyword list or separate routing tool to maintain. It answers supported questions as the relevant subject expert and declines anything outside the four supported areas.
模型会自行决定每个问题最适合哪个学科，因此无需维护关键词列表或单独的路由工具。它会以相关学科专家的身份回答支持的问题，并拒绝回答这四个领域之外的任何内容。

### AgentCore
### AgentCore

Amazon Bedrock AgentCore is a set of managed AWS services for building, deploying, connecting, and operating agents on the AWS cloud. It’s framework-agnostic, so it can host agents built with Strands, LangChain, OpenAI Agents SDK, and other frameworks.
Amazon Bedrock AgentCore 是一套托管的 AWS 服务，用于在 AWS 云上构建、部署、连接和操作 Agent。它与框架无关，因此可以托管使用 Strands、LangChain、OpenAI Agents SDK 及其他框架构建的 Agent。

| Capability | Purpose |
| :--- | :--- |
| **功能** | **用途** |
| Runtime | Hosts and scales agents in session-isolated environments. Supports streaming and HTTP, MCP, and A2A protocols. |
| 运行时 | 在会话隔离的环境中托管和扩展 Agent。支持流式传输以及 HTTP、MCP 和 A2A 协议。 |
| Memory | Stores conversation events and extracts durable facts, preferences, summaries, or episodes for use across sessions. |
| 内存 | 存储对话事件，并提取持久的事实、偏好、摘要或片段，以供跨会话使用。 |
| Gateway | Exposes APIs, Lambda functions, and MCP servers as managed tools that agents can discover and call. |
| 网关 | 将 API、Lambda 函数和 MCP 服务器公开为托管工具，供 Agent 发现和调用。 |
| Identity | Manages inbound authentication and the credentials agents use to access external services. |
| 身份 | 管理入站身份验证以及 Agent 访问外部服务时使用的凭据。 |
| Policy | Applies Cedar authorisation rules to Gateway tool calls before they reach their targets. |
| 策略 | 在网关工具调用到达目标之前，应用 Cedar 授权规则。 |
| Browser | Provides managed browser sessions for agents that need to interact with websites. |
| 浏览器 | 为需要与网站交互的 Agent 提供托管浏览器会话。 |
| Code Interpreter | Runs Python, JavaScript, or TypeScript in isolated managed sandboxes. |
| 代码解释器 | 在隔离的托管沙箱中运行 Python、JavaScript 或 TypeScript。 |
| Observability | Sends agent logs, traces, and metrics to services such as CloudWatch and X-Ray. |
| 可观测性 | 将 Agent 的日志、追踪和指标发送到 CloudWatch 和 X-Ray 等服务。 |
