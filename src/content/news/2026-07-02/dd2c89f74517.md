### Key Takeaways / 关键要点

**The problem: in a multi-hop agent pipeline, every time the control shifts from agent A to agent B, the receiver throws A’s hidden state away and rebuilds the context from a prompt string. That is structurally the same “post-handover cold start” a user equipment (UE) suffers when it moves between two base stations (from source to target), where the target base station re-initialises the per-user recurrent state from scratch.**
问题所在：在多跳智能体流水线中，每当控制权从智能体 A 转移到智能体 B 时，接收方都会丢弃 A 的隐藏状态，并从提示词字符串中重建上下文。这在结构上与用户设备 (UE) 在两个基站之间移动（从源基站到目标基站）时所遭受的“切换后冷启动”相同，目标基站会从零开始重新初始化每个用户的循环状态。

**The fix: compress the sender’s recurrent state into a tiny latent payload, transport it across the hand-off, and let the receiver use it as a soft-prompt prefix instead of re-prefilling everything from text. The same “compute once, fan out shared state” lesson the series has been hammering since Part 1, applied across reasoning hops instead of within one pipeline.**
解决方案：将发送方的循环状态压缩为微小的潜在载荷，在切换过程中进行传输，并让接收方将其用作软提示前缀，而不是从文本中重新预填充所有内容。这与本系列自第一部分以来一直强调的“计算一次，分发共享状态”的经验相同，只不过这次是应用于推理跳跃之间，而非单一流水线内。

**The ‘un’usual receipts: the underlying method is Inductive Latent Context Persistence (ILCP), a peer-reviewed paper I co-authored recently, accepted at AI4NextG @ ICML 2026. On the Vienna 4G/5G drive-test, ILCP eliminates ping-pong handovers entirely (0.0% vs 6.5% no-transfer baseline, 22.6% Transformer baseline), recovers post-handover accuracy at +5.1 pp average / +13.3 pp peak, and runs end to end at 7.7 ms p99 per handover decision on the same GTX 1080 as the rest of this series.**
“不寻常”的数据：其底层方法是归纳潜在上下文持久化 (ILCP)，这是我最近合著的一篇同行评审论文，已被 AI4NextG @ ICML 2026 接收。在维也纳 4G/5G 路测中，ILCP 完全消除了乒乓切换（0.0% 对比无传输基准的 6.5% 和 Transformer 基准的 22.6%），切换后准确率平均提升 5.1 个百分点，峰值提升 13.3 个百分点，并且在与本系列其余部分相同的 GTX 1080 上，每次切换决策的端到端运行时间为 7.7 毫秒 (p99)。

**TL;DR: Multi-hop LLM agents currently hand off context as a string. Agent A finishes reasoning, summarises it into prompt text, and Agent B reads that string from scratch — the receiver’s KV cache, attention pattern, and any partial computation Agent A built up are all discarded.**
简而言之：多跳 LLM 智能体目前以字符串形式移交上下文。智能体 A 完成推理，将其总结为提示文本，然后智能体 B 从头开始读取该字符串——接收方的 KV 缓存、注意力模式以及智能体 A 建立的任何部分计算都被丢弃了。

**This is the agent version of the post-handover cold start that 5G/6G base stations suffer when a UE (mobile device) moves between two base stations: the target base station re-initialises the per-UE recurrent state and has to rebuild it from scratch. We solved that problem with a method called Inductive Latent Context Persistence (ILCP): a β-VAE compresses a 128-dim GRU state into a 128-byte latent payload, transports it over the standard 3GPP Xn interface, and then a gated MLP projects it into the target base station’s state space at handover.**
这是 5G/6G 基站在 UE（移动设备）在两个基站之间移动时所遭受的切换后冷启动的智能体版本：目标基站重新初始化每个 UE 的循环状态，并必须从头开始重建它。我们通过一种称为归纳潜在上下文持久化 (ILCP) 的方法解决了这个问题：β-VAE 将 128 维的 GRU 状态压缩为 128 字节的潜在载荷，通过标准的 3GPP Xn 接口进行传输，然后在切换时由门控 MLP 将其投影到目标基站的状态空间中。

**On the Vienna 4G/5G drive-test, ILCP eliminates ping-pong handovers (0.0% vs 6.5% no-transfer baseline), recovers post-handover next-cell accuracy by +5.1 pp average / +13.3 pp peak in the 50–250 ms post-handover window, and runs at 7.7 ms p99 per decision on a single GTX 1080. This part maps that exact same protocol onto LLM agent hand-offs: ilcp-for-agents learns to compress a pooled hidden summary, transport it across the hand-off, and project it back into a receiver-side soft-prompt prefix.**
在维也纳 4G/5G 路测中，ILCP 消除了乒乓切换（0.0% 对比无传输基准的 6.5%），在切换后 50-250 毫秒窗口内，切换后的下一小区准确率平均提升 5.1 个百分点，峰值提升 13.3 个百分点，并在单块 GTX 1080 上以 7.7 毫秒 (p99) 的速度运行。本部分将完全相同的协议映射到 LLM 智能体移交上：ilcp-for-agents 学习压缩池化的隐藏摘要，在移交过程中传输它，并将其投影回接收端的软提示前缀中。
