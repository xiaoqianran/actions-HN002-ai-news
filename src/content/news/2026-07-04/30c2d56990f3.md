# Fixed-Set Robustness in Programming by Example: Example Corruption and Semantic Partition Recovery
# 示例编程中的固定集鲁棒性：示例篡改与语义分区恢复

**Abstract:** Programming-by-example systems infer programs from a small set of input-output examples. Robust PBE work usually models wrong examples as samples from a stochastic noise process and then minimizes an expected or empirical loss. This paper studies a different failure mode: an adversary who sees the synthesizer and chooses the examples whose corruption most damages the returned program.

**摘要：** 示例编程（PBE）系统通过少量输入-输出示例来推断程序。鲁棒性 PBE 研究通常将错误示例建模为来自随机噪声过程的样本，进而最小化期望损失或经验损失。本文研究了一种不同的失效模式：攻击者在了解合成器机制的前提下，选择性地篡改示例，以最大程度地破坏合成出的程序。

We formalize fixed-set worst-case corruption for finite PBE version spaces, implement exact-within-bounded-pool and heuristic corruption searches for a string-transformation DSL, and introduce version-space partition aggregation (VPA), a defense that synthesizes on disjoint example groups and votes by semantic signatures.

我们对有限 PBE 版本空间中的“固定集最坏情况篡改”进行了形式化定义，针对字符串转换领域特定语言（DSL）实现了“有界池内精确篡改”和“启发式篡改”搜索，并引入了版本空间分区聚合（VPA）防御机制——该机制通过在不相交的示例组上进行合成，并根据语义签名进行投票来抵御攻击。

The central claim is deliberately bounded and partly negative: low-margin PBE tasks have an adversarial robustness dimension that random-typo and noisy-PBE evaluations miss, while semantic partition aggregation helps only when the clean semantics keep a partition vote margin, which often fails on realistic tasks.

本文的核心观点是有意限定且带有部分否定性质的：低容错率（low-margin）的 PBE 任务存在一种对抗性鲁棒性维度，而随机拼写错误和噪声 PBE 评估往往会忽略这一点；同时，语义分区聚合仅在干净语义保持分区投票余量时才有效，而这在现实任务中往往难以实现。

Evidence from curated/generated DSL tasks, accepted public SyGuS PBE_SLIA slices, SYNTRA Playgol v2, and noisy-PBE objective baselines supports that boundary. One curated edit edit flips all 8 spike tasks while 200-trial typo, DSL-pool, and distance-matched random controls succeed on 10.3%, 11.0%, and 16.7%; generated margin-1 rows flip under budget 1 yet VPA recovers them; on public SyGuS the vote margin is near one, so an adaptive attacker drives VPA accuracy to zero; accepted public SyGuS slices move across exact-within-pool budget boundaries; and Playgol shows positive paired-bootstrap gaps against typo and same-pool random controls on the 141 accepted rows.

来自精心策划/生成的 DSL 任务、已接受的公共 SyGuS PBE_SLIA 切片、SYNTRA Playgol v2 以及噪声 PBE 目标基准的证据支持了这一边界。一次精心策划的编辑即可翻转全部 8 个尖峰任务，而 200 次试验的拼写错误、DSL 池和距离匹配随机对照组的成功率仅为 10.3%、11.0% 和 16.7%；生成的余量为 1 的行在预算 1 下会被翻转，但 VPA 可以恢复它们；在公共 SyGuS 上，投票余量接近 1，因此自适应攻击者可将 VPA 的准确率降至零；已接受的公共 SyGuS 切片跨越了池内精确预算边界；Playgol 在 141 个已接受行上，对比拼写错误和同池随机对照组，显示出正向的配对自助法（paired-bootstrap）差距。

A small exact-output prompt harness over 20 controlled margin-1 tasks shows the same qualitative clean-to-attacked pattern across local and API models, while it is treated as a scope check, not a broad LLM benchmark.

针对 20 个受控的余量为 1 的任务进行的小型精确输出提示测试显示，在本地模型和 API 模型中，从“干净”到“被攻击”的定性模式是一致的；该测试被视为一种范围检查，而非广泛的大语言模型（LLM）基准测试。
