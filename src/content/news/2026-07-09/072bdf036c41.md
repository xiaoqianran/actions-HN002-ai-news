# The Real Challenge Limiting AI Models Today
# 当今限制人工智能模型的真正挑战

Artificial Intelligence: The Real Challenge Limiting AI Models Today. Hint: it is not GPU speed!
人工智能：当今限制人工智能模型的真正挑战。提示：这与 GPU 速度无关！

When we talk about some of the problems we face in implementing AI algorithms, we usually focus on the processors’ ability to handle them. But now, NVIDIA’s GPUs power the latest large language models, and companies compete to build faster AI accelerators. These new chips promise more computational power, more cores, and more operations per second. So, one would assume that the future of AI depends on building increasingly powerful processors!
当我们谈论在实现人工智能算法时所面临的问题时，通常会关注处理器处理这些算法的能力。但如今，NVIDIA 的 GPU 为最新的大型语言模型提供了动力，各家公司也在竞相制造更快的 AI 加速器。这些新芯片承诺提供更强的计算能力、更多的核心以及每秒更多的运算次数。因此，人们可能会认为人工智能的未来取决于制造越来越强大的处理器！

But there is a problem. Many of today’s AI systems are not limited by how quickly they can perform calculations. They are limited by how quickly they can access data. Data that is essential for them to perform their calculations. In other words, the future of AI may depend less on computation and more on memory.
但问题在于，当今许多人工智能系统受限的并非计算速度，而是数据访问速度——即执行计算所必需的数据的获取速度。换句话说，人工智能的未来可能不再仅仅取决于计算，而更多地取决于内存。

I know that may not seem very intuitive, but let’s take a step back and imagine something: imagine hiring the world’s most efficient and fastest chef. This chef can prepare meals at incredible speed. However, every ingredient is stored in a warehouse several miles away. Before the chef can cook, someone needs to grab the ingredients and deliver them to the kitchen. No matter how talented the chef is, there will be periods when they simply stand around, waiting for the ingredients they need to start cooking to arrive.
我知道这看起来可能不太直观，但让我们退一步想象一下：假设你雇佣了世界上最高效、最快速的厨师。这位厨师能以惊人的速度准备餐点。然而，所有的食材都存放在几英里外的仓库里。在厨师开始烹饪之前，必须有人去取食材并送到厨房。无论厨师多么有天赋，总会有那么一段时间，他们只能站在那里，等待开始烹饪所需的食材送达。

Modern AI systems face a similar challenge. The processors they use can perform calculations, but they cannot operate on data that has not yet arrived. If the processor can compute faster than memory can deliver information, performance becomes limited by data movement rather than computation. This is what is known in computer science as the memory bottleneck. It is one of the most important and least discussed challenges in modern AI.
现代人工智能系统面临着类似的挑战。它们所使用的处理器虽然能进行计算，但无法处理尚未送达的数据。如果处理器的计算速度超过了内存传输信息的速度，性能就会受限于数据移动而非计算本身。这就是计算机科学中所说的“内存瓶颈”。这是现代人工智能中最重要却最少被讨论的挑战之一。

### The Scale of Modern Models
### 现代模型的规模

To better understand why memory has become such a significant issue, we need to consider the size of today’s AI models. Early machine learning models might have contained thousands or millions of parameters, while modern foundation models contain billions or even trillions. Each one of these parameters represents a numerical value that must be stored in memory and repeatedly accessed during training and inference.
为了更好地理解为什么内存会成为如此重大的问题，我们需要考虑当今人工智能模型的规模。早期的机器学习模型可能包含数千或数百万个参数，而现代基础模型则包含数十亿甚至数万亿个参数。每一个参数都代表一个数值，必须存储在内存中，并在训练和推理过程中被反复访问。

Okay, that sounds logical, but let’s solidify it with some numbers! Suppose we have a model that contains 70 billion parameters. Before even performing a single calculation, the system needs a place to store all those parameters. Now, I want you to imagine thousands of users interacting with the model simultaneously. The hardware must continuously move enormous amounts of information between memory and processors. As you may conclude, the challenge is no longer simply performing calculations; rather, it is feeding data to the hardware quickly enough.
好吧，这听起来很合逻辑，但让我们用一些数字来具体说明！假设我们有一个包含 700 亿个参数的模型。在进行任何计算之前，系统首先需要一个地方来存储所有这些参数。现在，请想象一下成千上万的用户同时与该模型进行交互。硬件必须在内存和处理器之间持续移动海量信息。正如你所推断的那样，挑战不再仅仅是执行计算，而是如何足够快地将数据输送给硬件。

Moving data can be more expensive than computing on it, which is one of the most counterintuitive realities in computing. Over the decades, processor performance has improved dramatically, as engineers have become exceptionally good at designing chips that can perform calculations faster and faster. On the other hand, though, improvements of the memory system moved at a slower pace. This created a growing imbalance that became more apparent as AI systems grew bigger.
移动数据的成本可能比计算数据本身还要高，这是计算领域中最反直觉的现实之一。几十年来，处理器性能得到了显著提升，因为工程师们在设计计算速度越来越快的芯片方面已经变得非常出色。然而另一方面，内存系统的改进速度却相对缓慢。这造成了一种日益严重的不平衡，随着人工智能系统规模的扩大，这种不平衡变得愈发明显。

Modern processors can execute trillions of operations per second, but they often spend a lot of time waiting for data to arrive. This data bottleneck appears in different ways within AI systems. We can see it while moving data between memory and processors, between GPUs, across servers, or between data centers. These models will only continue to grow, and as they do, the data movement issue will play a major role in the system’s overall performance!
现代处理器每秒可以执行数万亿次运算，但它们往往花费大量时间等待数据送达。这种数据瓶颈在人工智能系统内部以不同方式表现出来。无论是在内存与处理器之间、GPU 之间、服务器之间，还是数据中心之间移动数据时，我们都能看到这一问题。这些模型只会持续增长，随着它们的增长，数据移动问题将在系统的整体性能中扮演关键角色！

### Understanding AI Memory
### 理解人工智能内存

So far, I think I’ve written the word “memory” over 20 times (I didn’t count!). You might have wondered, what kind of memory is she talking about? Most people are familiar with RAM, the memory installed in laptops and desktop computers. AI systems use different types of memory for different purposes.
到目前为止，我想我已经写了超过 20 次“内存”这个词（我没数过！）。你可能在想，她到底在说哪种内存？大多数人熟悉的是 RAM，即安装在笔记本电脑和台式机中的内存。人工智能系统会根据不同的用途使用不同类型的内存。

1. **RAM:** Random Access Memory stores data used by the CPU. It is relatively large but not particularly fast compared to specialized AI memory.
1. **RAM（随机存取存储器）：** 存储 CPU 使用的数据。与专门的 AI 内存相比，它的容量相对较大，但速度并不特别快。

2. **VRAM:** Graphics Processing Units contain dedicated memory known as Video Random Access Memory (VRAM). Which is used during training and inference to store model parameters, training batches, activations, and intermediate calculations. The amount of available VRAM often determines whether a model can fit on a GPU.
2. **VRAM（显存）：** 图形处理器包含专用的内存，即视频随机存取存储器（VRAM）。它在训练和推理过程中用于存储模型参数、训练批次、激活值和中间计算结果。可用 VRAM 的大小通常决定了模型能否在 GPU 上运行。

3. **High-Bandwidth Memory (HBM):** Modern AI accelerators increasingly rely on High-Bandwidth Memory (HBM). HBM is designed specifically to move large amounts of data extremely quickly. Rather than simply increasing memory capacity, HBM focuses on increasing memory bandwidth, which is the rate at which information can be transferred.
3. **HBM（高带宽内存）：** 现代 AI 加速器越来越依赖高带宽内存（HBM）。HBM 专为极速移动海量数据而设计。HBM 不仅仅是增加内存容量，更侧重于提高内存带宽，即信息传输的速率。

Though the size of the memory (its capacity) is important, its bandwidth plays a bigger role. One way to think about those two concepts is a highway (stay with me here for a second). Capacity is the number of cars that can exist on the highway, while bandwidth is the number of lanes available. You can have a huge parking lot, but if all vehicles must leave through a single lane, traffic becomes the limiting factor.
虽然内存的大小（容量）很重要，但其带宽所起的作用更大。我们可以用高速公路来理解这两个概念（请听我解释一下）。容量是高速公路上可以容纳的车辆总数，而带宽则是可用的车道数量。你可以拥有一个巨大的停车场，但如果所有车辆都必须通过单车道离开，那么交通流量就会成为限制因素。

Now, as we said before, AI systems use different types of memory in different ways. The memory challenge appears differently during training and inference.
正如我们之前所说，人工智能系统以不同的方式使用不同类型的内存。内存挑战在训练和推理过程中表现出的形式也各不相同。

**Training:** Training requires storing model parameters, gradients, activations, and optimizer states. So, as a result, memory requirements become enormous. This would require distributing memory across many GPUs.
**训练：** 训练需要存储模型参数、梯度、激活值和优化器状态。因此，内存需求变得极其巨大。这需要将内存分布在多个 GPU 上。

**Inference:** Inference generally requires less memory than training, but it introduces a different challenge.
**推理：** 推理通常比训练需要更少的内存，但它带来了另一个挑战。
