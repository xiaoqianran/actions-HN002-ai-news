# Learning to Control LLM Agent Harnesses with Offline Reinforcement Learning
# 利用离线强化学习控制大语言模型代理的执行框架

**Abstract:** Large language model (LLM) agents are usually improved by changing prompts, models, or hand-written workflows, while the execution harness around the model is treated as fixed infrastructure. We argue that this harness is itself a learnable control layer. 

**摘要：** 大语言模型（LLM）代理的改进通常依赖于调整提示词（prompts）、模型本身或人工编写的工作流，而模型周围的执行框架（execution harness）往往被视为固定的基础设施。我们认为，这一执行框架本身就是一个可学习的控制层。

We formalize harness operation as a finite-horizon Harness MDP, where a lightweight controller selects structural execution actions while the LLM executor remains frozen. The controller is trained from offline rollouts using advantage-weighted regression with only terminal task-rubric rewards. 

我们将框架的运行形式化为有限时域的“框架马尔可夫决策过程”（Harness MDP），其中一个轻量级控制器负责选择结构化的执行动作，而 LLM 执行器保持冻结状态。该控制器通过离线回放数据进行训练，采用优势加权回归（advantage-weighted regression），且仅使用最终任务评分作为奖励。

We also separate final task quality from a post-hoc Harness Maturity Score, which measures whether the harness follows reliable execution patterns rather than only whether the final answer is correct. This separation gives a finite-buffer view of harness learning: final-quality gains require high-return support in the offline buffer, while process behavior can shift whenever it aligns with advantage-weighted actions. 

我们还将最终任务质量与事后评估的“框架成熟度评分”（Harness Maturity Score）区分开来，后者用于衡量框架是否遵循了可靠的执行模式，而不仅仅是判断最终答案是否正确。这种分离为框架学习提供了一种有限缓冲区视角：最终质量的提升需要离线缓冲区中高回报数据的支持，而过程行为则可以在与优势加权动作对齐时随时进行调整。

Across six controlled domains and two public-benchmark adapters, the learned controller consistently improves verification behavior and selectively improves final task quality, with the largest gains on adapted tau-bench retail, adapted AgentBench DB-Bench, and coding with a calibrated structural verifier. 

在六个受控领域和两个公共基准适配器上的实验表明，学习到的控制器能够持续改善验证行为，并有选择地提升最终任务质量。其中，在适配后的 tau-bench 零售场景、适配后的 AgentBench DB-Bench 以及结合校准结构化验证器的编码任务中，性能提升最为显著。

Ablations against behavior cloning and Forced CHECK show that the gains are not explained by imitation or by simply adding checks. These results identify harness control as a learnable layer for frozen LLM agents, while showing that offline support limits when better process control becomes better final answers.

通过与行为克隆（behavior cloning）和强制检查（Forced CHECK）进行的消融实验表明，这些性能增益并非源于简单的模仿或单纯的检查添加。这些结果证实了框架控制可以作为冻结 LLM 代理的一个可学习层，同时也揭示了离线数据的支持限制了更好的过程控制转化为更好最终答案的边界。
