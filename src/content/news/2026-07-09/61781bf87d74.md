# AdaStop: Cost-Aware Early Stopping for DNN Test Selection
# AdaStop：面向深度神经网络测试选择的成本感知提前停止策略

Existing methods for testing deep neural networks (DNNs) primarily prioritize test inputs likely to reveal model faults under a fixed labeling budget. In practice, choosing that budget is difficult: too little testing misses failures, while too much incurs unnecessary labeling costs.
现有的深度神经网络（DNN）测试方法主要是在固定的标注预算下，优先选择最有可能揭示模型缺陷的测试输入。在实际应用中，确定这一预算非常困难：测试过少会导致漏检故障，而测试过多则会产生不必要的标注成本。

This work studies the stopping problem in DNN testing. We formulate testing as a cost--benefit decision process in which labeling an input incurs cost $c$ and discovering a fault yields value $v$. Based on this formulation, we introduce \textit{AdaStop}, a framework that estimates the marginal fault discovery rate during testing and stops labeling when the estimated rate falls below the threshold $\tau = c/v$.
本研究探讨了 DNN 测试中的停止问题。我们将测试过程建模为一个成本-收益决策过程，其中标注一个输入会产生成本 $c$，而发现一个故障则带来价值 $v$。基于此模型，我们提出了 AdaStop 框架。该框架能够在测试过程中估算边际故障发现率，并在估算出的发现率低于阈值 $\tau = c/v$ 时停止标注。

Experiments across multiple datasets, architectures, and selection strategies show that $65$--$84\%$ of faults can be discovered using only $9$--$31\%$ of the labeling budget.
在多个数据集、模型架构和选择策略上的实验表明，仅需使用 $9\%$--$31\%$ 的标注预算，即可发现 $65\%$--$84\%$ 的模型故障。
