# OpenClaw Just Shipped an Animated Welcome & Android Image Previews
# OpenClaw 发布了动画欢迎界面与 Android 图片预览功能

OpenClaw pushed 10 commits in the last 3 hours — headlining with a redesigned chat welcome screen and native Android image previews, alongside a wave of UTF-16 safety fixes and CI stability work.
OpenClaw 在过去 3 小时内推送了 10 次提交，主要更新包括重新设计的聊天欢迎界面和原生的 Android 图片预览功能，同时还包含了一系列 UTF-16 安全修复和 CI（持续集成）稳定性改进。

**What changed**
**更新内容**

Animated welcome replaces canned suggestions. The chat landing page now shows an animated Clawd mascot alongside the user’s recent conversations instead of static suggestion chips. Makes the start of every session feel alive and personalized.
动画欢迎界面取代了预设建议。聊天着陆页现在会显示一个动画版的 Clawd 吉祥物以及用户的近期对话，而不是静态的建议卡片。这让每次会话的开始都显得生动且个性化。

Android gets chat image previews. Long a gap on mobile — images shared in threads now render inline as previews instead of showing raw file names or download links. Brings the Android experience closer to parity with desktop.
Android 端获得了聊天图片预览功能。这长期以来是移动端的短板——现在对话中分享的图片会直接以内联预览的形式呈现，而不是显示原始文件名或下载链接。这使得 Android 端的体验更接近桌面版。

Sidebar navigation got cleaned up. Activity moved to Settings, Dreams folded into the Agents Memory tab, and MCP mutation ownership became single-source. Reduces context-switching for power users.
侧边栏导航经过了精简。Activity（活动）被移至 Settings（设置），Dreams（梦想）被整合进 Agents Memory（智能体记忆）标签页，MCP 变更所有权也实现了单一来源管理。这减少了高级用户的上下文切换频率。

Beyond features, a coordinated batch of fixes improved UTF-16 byte-count reporting across set command output, summary text truncation, Telegram QA progress details, worker SSH tunnel stderr tails, and file write success messages. The browser also gained a guard on act hooks against missing current tab URLs.
除了功能更新外，一系列协同修复改进了 UTF-16 字节计数报告，涵盖了 set 命令输出、摘要文本截断、Telegram QA 进度详情、工作节点 SSH 隧道 stderr 尾部日志以及文件写入成功消息。浏览器端还为 act 钩子增加了针对缺失当前标签页 URL 的保护机制。

**Why it matters**
**重要意义**

The animated welcome signal matters psychologically — a blank chat box with “try asking...” is cold; showing past sessions plus a live character turns the agent from a tool into a collaborator.
动画欢迎界面在心理层面意义重大——一个写着“试着问问……”的空白聊天框显得冷冰冰；而展示过往会话并配上一个活灵活现的角色，能将智能体从一个工具转变为协作伙伴。

Android image previews unblock mobile workflows that previously required switching to desktop to see shared visuals. The UTF-16 fixes, while less glamorous, prevent silent data corruption when working with non-ASCII file names and messages — critical for international teams.
Android 图片预览功能打通了移动端工作流，此前用户必须切换到桌面端才能查看分享的视觉内容。UTF-16 的修复虽然不那么显眼，但能防止在处理非 ASCII 文件名和消息时出现静默数据损坏，这对国际化团队至关重要。

**The takeaway**
**总结**

OpenClaw is investing heavily in both first-impression UX (animated welcome) and mobile parity (image previews). If you use OpenClaw on Android or manage multi-language projects, update to the latest — these changes directly improve daily flow.
OpenClaw 正在大力投入于第一印象的用户体验（动画欢迎）和移动端对齐（图片预览）。如果您在 Android 上使用 OpenClaw 或管理多语言项目，请更新至最新版本——这些改进将直接提升您的日常工作效率。
