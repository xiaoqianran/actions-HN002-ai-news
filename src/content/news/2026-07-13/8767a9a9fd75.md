### How to orchestrate a lot of agents: Starting headless sessions
### 如何编排大量智能体：启动无头（Headless）会话

The short answer to the question of how to orchestrate a lot of agents is to use headless mode. Headless mode in both Claude Code and Codex is basically a mode where you can spin up a session using the CLI and have the agent work on a particular task and only come back to you with the answer.
关于如何编排大量智能体的简短回答是：使用“无头模式”（headless mode）。Claude Code 和 Codex 中的无头模式本质上是一种通过命令行（CLI）启动会话的模式，让智能体处理特定任务，并在完成后仅向你返回结果。

You can activate headless mode with Claude Code using: `claude -p "your prompt here"`
And you can activate headless mode with Codex using: `codex exec "your prompt here"`
你可以通过以下命令激活 Claude Code 的无头模式：`claude -p "your prompt here"`
你也可以通过以下命令激活 Codex 的无头模式：`codex exec "your prompt here"`

What’s gonna happen once you run this is that you’re gonna spin up a completely separate Codex or Claude Code session, and that session is gonna work on the command you provided until it’s completed the task. The orchestrator agent, which you are talking to, will then only see the logs if it chooses to, and the final result of the command will be the output of the agent, for example, if work was completed.
一旦运行这些命令，你就会启动一个完全独立的 Codex 或 Claude Code 会话，该会话将处理你提供的指令，直到任务完成。你所对话的编排智能体（orchestrator agent）可以选择查看日志，而命令的最终结果将是智能体的输出，例如任务是否已完成。

### Being effective with headless mode
### 如何高效使用无头模式

I’ll cover three main techniques to be effective with headless mode:
1. Providing a way for the agent to verify its own work.
2. Finding suitable tasks for headless modes, such as refactoring.
3. Give the agent all the tools it needs.
我将介绍三种高效使用无头模式的主要技术：
1. 为智能体提供验证自身工作的方法。
2. 为无头模式寻找合适的任务，例如重构。
3. 为智能体提供它所需的所有工具。

**Providing a way for the agent to verify its own work**
One of the challenges is that you can’t be directly involved in all of the decision-making. Since you’re moving one abstraction layer up, you have to make the agents perform more decisions. Which gives you less control in a way. Thus, you have to change the way you prompt the model to be more high-level, and you have to tell it how to verify its work.
**为智能体提供验证自身工作的方法**
挑战之一是你无法直接参与所有的决策。由于你提升了一个抽象层级，你必须让智能体做出更多的决策，这在某种程度上减少了你的控制权。因此，你必须改变提示词（prompt）的方式，使其更具宏观性，并且必须告诉它如何验证自己的工作。

**Finding suitable tasks for headless modes, such as refactoring**
Another important thing when you use agents like this is to apply them to tasks that are well suited to headless mode. More complicated tasks in themselves aren’t suited for headless mode and are probably better off with more of your input. One task that is particularly well suited for headless mode is refactoring.
**为无头模式寻找合适的任务，例如重构**
使用此类智能体时，另一个重点是将它们应用于适合无头模式的任务。过于复杂的任务本身并不适合无头模式，通常需要你更多的参与。而重构（refactoring）是一个特别适合无头模式的任务。
