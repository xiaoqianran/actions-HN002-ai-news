# MawForge: Memory-Bounded Expert Materialization for Local Mixture-of-Experts Inference
# MawForge：用于本地混合专家模型推理的内存受限专家实例化技术

Sparse Mixture-of-Experts (MoE) language models separate total parameter count from per-token active computation, but local inference systems often still require the full model, key-value cache, runtime buffers, and operating system headroom to fit in fast memory.
稀疏混合专家（MoE）语言模型将总参数量与每个 Token 的实际计算量分离开来，但本地推理系统通常仍要求将完整模型、键值（KV）缓存、运行时缓冲区以及操作系统预留空间全部装入高速内存中。

MawForge tests a different systems hypothesis: local MoE serving can be made practical on constrained unified-memory machines by storing the full model on disk, keeping common tensors resident, and materializing routed expert tensors into a bounded execution cache on demand.
MawForge 验证了一个不同的系统假设：通过将完整模型存储在磁盘上，仅保留常用张量在内存中，并按需将路由后的专家张量实例化到受限的执行缓存中，可以在内存受限的统一内存架构机器上实现实用的本地 MoE 服务。

The central finding is that MawForge is effective as a bounded execution mechanism and measurement substrate for local MoE inference, but not as a cache-maximization policy. Performance depends on balancing expert reuse against resident footprint, KV-cache size, quantization, route locality, and macOS memory pressure.
核心研究发现表明，MawForge 作为一种本地 MoE 推理的受限执行机制和测量基底是有效的，但它并非一种缓存最大化策略。其性能表现取决于专家重用率与常驻内存占用、KV 缓存大小、量化程度、路由局部性以及 macOS 内存压力之间的平衡。
