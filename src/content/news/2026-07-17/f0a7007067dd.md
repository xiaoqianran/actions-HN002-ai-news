# Introducing RegionCheck: Test Endpoints from AWS, Azure, and Google Cloud Regions
# 介绍 RegionCheck：从 AWS、Azure 和 Google Cloud 区域测试端点

Today I'm excited to launch RegionCheck, a tool for testing, monitoring, and debugging endpoints from cloud regions around the world. The idea is simple - test an endpoint's DNS and HTTP connectivity and response from a defined set of cloud regions. Whether you're troubleshooting an API, validating a deployment, checking DNS propagation, or investigating latency, seeing the results from multiple cloud regions can quickly reveal issues that aren't obvious from your own machine.
今天，我很高兴推出 RegionCheck，这是一个用于从全球云区域测试、监控和调试端点的工具。其理念很简单——从一组指定的云区域测试端点的 DNS 和 HTTP 连接性及响应。无论您是在排查 API 故障、验证部署、检查 DNS 解析传播，还是在调查延迟问题，查看来自多个云区域的结果都能快速揭示在您本地机器上无法察觉的问题。

What is RegionCheck? RegionCheck lets you run endpoint checks from AWS, Azure, and Google Cloud regions without provisioning infrastructure or maintaining test instances. Current capabilities include: HTTP endpoint testing, DNS lookups, TLS certificate validation, Continuous monitoring with alerts, Side-by-side comparison across cloud providers and regions, Shareable result pages for collaboration, and API/MCP access for automation and agents.
什么是 RegionCheck？RegionCheck 让您无需配置基础设施或维护测试实例，即可从 AWS、Azure 和 Google Cloud 区域运行端点检查。目前的功能包括：HTTP 端点测试、DNS 查询、TLS 证书验证、带警报的持续监控、跨云服务商和区域的并排对比、可共享的协作结果页面，以及用于自动化和代理的 API/MCP 访问。

Why I built it: When debugging production issues, I often wanted to answer questions like: Is DNS returning the same result everywhere? Or is geo-DNS returning the results intended? Is TLS certificate propagation for my CDN working as intended? Is one region significantly slower than another? Is my CDN caching working as expected? Are my geo-HTTP redirects working as intended? (For some interesting examples try www.yahoo.com and www.cnn.com in non-US regions). There are many tools that exist that provide these answers, but nothing that answers all of these questions in one place. That's what RegionCheck aims to provide.
我为何开发它：在调试生产环境问题时，我经常需要回答诸如以下的问题：DNS 在各地返回的结果是否一致？地理位置 DNS（geo-DNS）返回的结果是否符合预期？CDN 的 TLS 证书传播是否按预期工作？某个区域是否明显比其他区域慢？CDN 缓存是否按预期工作？地理位置 HTTP 重定向是否按预期工作？（若想查看一些有趣的示例，请尝试在非美国地区测试 www.yahoo.com 和 www.cnn.com）。市面上有很多工具可以提供这些答案，但没有一个能在一个地方解决所有这些问题。这正是 RegionCheck 旨在实现的目标。

Who it's for: RegionCheck is designed for engineers who work with cloud infrastructure, including: DevOps engineers, Site Reliability Engineers (SREs), Platform engineers, Backend developers, and anyone who likes to take a peek at backend infrastructure.
适用人群：RegionCheck 专为从事云基础设施工作的工程师设计，包括：DevOps 工程师、站点可靠性工程师 (SRE)、平台工程师、后端开发人员，以及任何喜欢窥探后端基础设施的人。

Try it out: RegionCheck is available at https://regioncheck.io. You can run free checks directly from the website; or create an account to access monitoring, alerting, the API, and additional features.
立即试用：RegionCheck 现已上线 https://regioncheck.io。您可以直接在网站上运行免费检查；或者创建一个账户以使用监控、警报、API 及更多附加功能。

I'd love your feedback: RegionCheck is just getting started, and I'd really appreciate feedback from the engineering community.
期待您的反馈：RegionCheck 才刚刚起步，我非常希望能听到工程社区的反馈。
