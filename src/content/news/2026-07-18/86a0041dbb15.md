# AWS: Inaccurate Estimated Billing Data – $1.7 billion
# AWS：账单预估数据错误——高达 17 亿美元

I've got an estimated bill for $1.7 BILLION over this month. Normal usage is < $5. Obvs have created an urgent AWS support ticket. Anyone else seeing something like this?
我这个月的账单预估金额竟然高达 17 亿美元，而我平时的正常使用费用不到 5 美元。显然，我已经提交了紧急 AWS 工单。还有其他人遇到这种情况吗？

**donavanm:** I've dealt with this error at AWS. It’s a unit error. In my case we _meant_ to charge like 5¢/GB, but missed the unit (GB), and then the billing system defaults to bytes. 5¢ per Byte of data transferred meant some customers were seeing MM bills within hours. Got paged by support around 2am, had it fixed and amendments issues by 3-4am, apology emails shortly after.
**donavanm：** 我在 AWS 处理过这种错误。这是一个单位错误。在我遇到的案例中，我们本意是按每 GB 5 美分收费，但漏掉了单位（GB），导致计费系统默认按字节（Byte）计算。每传输 1 字节收费 5 美分，意味着一些客户在几小时内就收到了数百万美元的账单。凌晨 2 点左右收到支持团队的传呼，3 到 4 点修复并发布了修正案，随后很快发出了道歉邮件。

Services emit metering values that aren't directly tied to prices. Every SKU/line item is defined in a “pricing plan”, with a unit type, regions, and price per unit. The metering records are joined to a pricing plan based on account id, region, sku, etc. Mess up the unit type in the pricing plan and the metering data conversion doesn't work, and you get crazy bills.
各项服务发出的计量值并不直接与价格挂钩。每个 SKU/行项目都在“定价计划”中定义，包含单位类型、区域和单价。计量记录会根据账户 ID、区域、SKU 等与定价计划进行关联。如果在定价计划中弄错了单位类型，计量数据转换就会失效，从而产生离谱的账单。
