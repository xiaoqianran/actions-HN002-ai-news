# Introducing TypeYAML: A More Expressive, Type-Safe Way to Write YAML

介绍 TypeYAML：一种更具表现力、类型安全的 YAML 编写方式

Today, we are releasing TypeYAML, an open-source language that extends YAML with stronger structure, reusable types, validation, and developer-focused tooling. TypeYAML is designed for developers who like YAML’s readability but need more confidence when working with large configuration files, schemas, deployment definitions, automation pipelines, application settings, and structured data. The project is now available through:  
npm: @magnexis/typeyaml  
Visual Studio Code Marketplace: magnificent-language/typeyaml  
GitHub: magnexis/typeyaml-lang

今天，我们发布了 TypeYAML，这是一种开源语言，通过更强的结构、可重用类型、验证和面向开发者的工具来扩展 YAML。TypeYAML 专为喜欢 YAML 可读性但需要在处理大型配置文件、架构、部署定义、自动化流水线、应用设置和结构化数据时更有信心的开发者设计。项目现已通过以下方式提供：  
npm: @magnexis/typeyaml  
Visual Studio Code Marketplace: magnificent-language/typeyaml  
GitHub: magnexis/typeyaml-lang

TypeYAML is still evolving, but this release establishes the foundation for a language intended to make configuration safer, clearer, and easier to maintain.

TypeYAML 仍在不断发展，但此版本为一种旨在使配置更安全、更清晰、更易维护的语言奠定了基础。

YAML is everywhere. It is used for:  
CI/CD pipelines  
Docker Compose files  
Kubernetes resources  
GitHub Actions  
Application configuration  
Infrastructure definitions  
API specifications  
Static-site configuration  
Package and project metadata

为什么还要构建另一种基于 YAML 的语言？  
YAML 无处不在。它用于：  
CI/CD 流水线  
Docker Compose 文件  
Kubernetes 资源  
GitHub Actions  
应用配置  
基础设施定义  
API 规范  
静态站点配置  
包和项目元数据

Its popularity makes sense. YAML is compact, readable, and approachable. A configuration file can often be understood without extensive documentation.

它的流行是有道理的。YAML 紧凑、可读且易于接近。配置文件通常无需大量文档即可理解。

However, YAML becomes more difficult to manage as projects grow. Large YAML files can contain deeply nested structures, duplicated values, inconsistent field names, invalid data types, undocumented conventions, and subtle indentation errors.

然而，随着项目增长，YAML 变得更难管理。大型 YAML 文件可能包含深度嵌套的结构、重复的值、不一致的字段名、无效的数据类型、未记录的约定和微妙的缩进错误。

In many cases, the file is only validated after another tool attempts to consume it. That creates a frustrating workflow: Write configuration. Run a command. Receive an unclear validation error. Search through a large YAML file. Fix the issue. Repeat.

在许多情况下，文件只有在另一个工具尝试使用它时才会被验证。这造成了一个令人沮丧的工作流：编写配置。运行命令。收到不明确的验证错误。在大型 YAML 文件中搜索。修复问题。重复。

TypeYAML is an attempt to improve that experience without abandoning what makes YAML useful. The goal is not to replace YAML’s readability. The goal is to build a stronger language around it.

TypeYAML 试图在不放弃 YAML 有用特性的情况下改善这种体验。目标不是取代 YAML 的可读性。目标是围绕它构建一种更强大的语言。

What Is TypeYAML?  
TypeYAML is a typed superset of YAML. It introduces a language layer for defining and validating structured configuration while keeping the syntax familiar to developers who already use YAML.

什么是 TypeYAML？  
TypeYAML 是 YAML 的类型化超集。它引入了一个语言层来定义和验证结构化配置，同时保持对已使用 YAML 的开发者熟悉的语法。

A normal YAML file might look like this:  
```
user:
  name: person
  age: 48
  active: true
```  
This is easy to read, but the file does not **independently explain** its expected structure. Should age always be a number? Is name required? Can active be omitted? Are additional fields allowed?

一个普通的 YAML 文件可能如下所示：  
```
user:
  name: person
  age: 48
  active: true
```  
这很容易阅读，但该文件没有**独立解释**其预期结构。age 是否应始终是数字？name 是否必填？active 是否可以省略？是否允许额外字段？

TypeYAML is designed to make those expectations explicit. A TypeYAML definition could describe the intended structure before data is used by the rest of an application:  
```
type User:
  name: string
  age: number
  active: boolean
```

TypeYAML 旨在使这些期望显式化。TypeYAML 定义可以在数据被应用程序其余部分使用之前描述预期结构：  
```
type User:
  name: string
  age: number
  active: boolean
```

The exact syntax and feature set will continue to develop, but the central idea is simple: configuration should be able to describe and verify itself. Instead of treating configuration as loosely structured text, TypeYAML treats it as a language with rules, types, tooling, and predictable behavior.

确切的语法和功能集将继续发展，但核心思想很简单：配置应该能够描述和验证自身。TypeYAML 不是将配置视为松散结构的文本，而是将其视为具有规则、类型、工具和可预测行为的语言。

The Main Goals of TypeYAML  
TypeYAML is being developed around several core principles.

TypeYAML 的主要目标  
TypeYAML 正在围绕几个核心原则进行开发。

1. Preserve YAML’s Readability  
TypeYAML should remain recognizable to anyone familiar with YAML. A developer should not need to learn an entirely unfamiliar syntax simply to gain validation and stronger structure. The language should continue to feel lightweight and configuration-oriented rather than becoming a verbose general-purpose programming language.

1. 保持 YAML 的可读性  
TypeYAML 应保持对熟悉 YAML 的人可识别。开发者不应仅仅为了获得验证和更强的结构而学习一种完全不熟悉的语法。该语言应继续感觉轻量级且面向配置，而不是变成冗长的通用编程语言。

2. Detect Errors Earlier  
Configuration problems should be caught while the file is being written, not only after deployment or execution. TypeYAML aims to support earlier feedback for issues such as:  
Incorrect value types  
Missing required properties  
Unknown fields  
Invalid nested structures  
Duplicate declarations  
Incorrect references  
Malformed language syntax  
Earlier validation means fewer failed builds, fewer broken deployments, and less time spent debugging configuration.

2. 更早检测错误  
配置问题应在文件编写过程中被捕获，而不是仅在部署或执行之后。TypeYAML 旨在支持对以下问题提供更早的反馈：  
不正确值类型  
缺失必填属性  
未知字段  
无效嵌套结构  
重复声明  
不正确引用  
格式错误的语言语法  
更早的验证意味着更少的构建失败、更少的部署中断和更少的配置调试时间。

3. Make Large Configurations Easier to Maintain  
As configuration grows, repeated structures become difficult to keep consistent. TypeYAML is intended to support reusable definitions so developers can describe a structure once and use it throughout a project. This can reduce duplicated schemas and make changes safer.

3. 使大型配置更易维护  
随着配置增长，重复结构变得难以保持一致性。TypeYAML 旨在支持可重用定义，以便开发者可以描述一次结构并在整个项目中重复使用。这可以减少重复的架构并使更改更安全。

4. Provide First-Class Developer Tooling  
A language is more than its parser. TypeYAML is being released with tooling intended to support a practical development workflow, including an npm package and a Visual Studio Code extension. The long-term goal is to provide a complete language experience with:  
Syntax highlighting  
Diagnostics  
Validation  
Autocomplete  
Hover information  
Formatting  
Navigation  
Language-aware refactoring  
Command-line tooling  
Programmatic APIs

4. 提供一流的开发者工具  
语言不仅仅是其解析器。TypeYAML 正在发布旨在支持实用开发工作流的工具，包括 npm 包和 Visual Studio Code 扩展。长期目标是提供完整的语言体验，包括：  
语法高亮  
诊断  
验证  
自动完成  
悬停信息  
格式化  
导航  
语言感知重构  
命令行工具  
编程 API

5. Remain Compatible With Existing Ecosystems  
YAML already has a massive ecosystem. TypeYAML should work alongside existing YAML-based tools rather than requiring developers to immediately replace their entire configuration stack. A major goal is to make it possible to author safer TypeYAML files and transform or compile them into standard YAML or other widely supported formats. This approach would allow TypeYAML to add stronger guarantees during development while preserving compatibility with tools that already understand YAML.

5. 与现有生态系统保持兼容  
YAML 已经拥有庞大的生态系统。TypeYAML 应与现有的基于 YAML 的工具协同工作，而不是要求开发者立即替换整个配置堆栈。一个主要目标是使其能够编写更安全的 TypeYAML 文件，并将它们转换或编译为标准 YAML 或其他广泛支持的格式。这种方法允许 TypeYAML 在开发过程中添加更强的保证，同时保留与已理解 YAML 的工具的兼容性。

Installing TypeYAML  
The TypeYAML package is available on npm under the Magnexis organization. Install it globally for command-line usage:  
`npm install -g @magnexis/typeyaml`  
Or install it inside a project:  
`npm install --save-dev @magnexis/typeyaml`  
Using it as a project dependency makes it easier to keep builds reproducible because the project can define the exact TypeYAML version it expects. After installation, the package can be integrated into scripts, build systems, validation workflows, or other developer tooling supported by the current release.

安装 TypeYAML  
TypeYAML 包在 npm 上的 Magnexis 组织下提供。全局安装以用于命令行：  
`npm install -g @magnexis/typeyaml`  
或在项目中安装：  
`npm install --save-dev @magnexis/typeyaml`  
将其作为项目依赖项使用可以更容易地保持构建可重现，因为项目可以定义其期望的精确 TypeYAML 版本。安装后，该包可以集成到脚本、构建系统、验证工作流或当前版本支持的其他开发者工具中。
