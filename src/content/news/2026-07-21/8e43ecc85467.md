# A document QC tool — and how to check it  
# 一份文档质检工具——以及如何检查它

---

A check that cannot be determined from the file says "not evaluable" — never a silent pass.  
无法从文件本身判断的检查会显示“不可评估”——绝不会静默通过。

The problem: It looks fine right up until it doesn't — in a viewer, on screen, in the happy path — and the failure only shows at the point where it's expensive to fix.  
问题在于：它看起来一切正常，直到突然出现问题——在查看器中、屏幕上、顺利流程中——而失败只在修复成本高昂时才显现。

What actually happens: This is a determinable property of the file: you can check it deterministically, without running the document and without guessing. Preflight does exactly that as part of its correctness-first pipeline.  
实际情况：这是文件的一个可确定性属性：你可以无需运行文档、无需猜测，就以确定性方式检查它。Preflight 正是这样做的，作为其以正确性为先的流程的一部分。

The honest caveat: `not_evaluable` is a first-class verdict; it never counts as a pass. That distinction is the whole point of the tool — a check that can't be determined from the file says `not evaluable`, never a silent pass.  
诚实的说明：`not_evaluable` 是一个首要判定结果；它绝不算作通过。这种区分正是该工具的核心——无法从文件判断的检查会显示 `not evaluable`，绝不会静默通过。

A QC tool that passes everything hasn't checked your document; it has flattered it.  
一个让所有内容都通过的质检工具，并未真正检查你的文档；它只是在奉承它。

Try it: Preflight parses the print formats most tools can't open (PDF, AFP/MO:DCA, PCL 5, PostScript, IPDS) and runs deterministic checks like this one — free, in your browser, nothing uploads.  
试试看：Preflight 能解析大多数工具无法打开的打印格式（PDF、AFP/MO:DCA、PCL 5、PostScript、IPDS），并运行此类确定性检查——免费，在浏览器中，无需上传任何文件。
