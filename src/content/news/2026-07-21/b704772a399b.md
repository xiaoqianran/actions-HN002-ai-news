# Gitolite
Gitolite

## Hosting Git Repositories
Gitolite allows you to setup git hosting on a central server, with fine-grained access control and many more powerful features.
Gitolite 允许你在中央服务器上设置 git 托管，具有细粒度访问控制还有许多其他强大功能。

## install/setup
If you're installing via your package manager, make sure you get the right one; it's often called gitolite3.
如果你通过包管理器安装，请确保获取正确的版本；它通常被称为 gitolite3。

Source code is at https://codeberg.org/sitaramc/gitolite.
源代码位于 https://codeberg.org/sitaramc/gitolite。

Note Wherever the documentation mentions the above URL, you can also use the alternative URL https://github.com/sitaramc/gitolite. If that ever changes this section will be updated.
注意： wherever the documentation mentions the above URL, you can also use the alternative URL https://github.com/sitaramc/gitolite。如果此信息有变，本部分将会更新。

If your Unix-fu and ssh-fu are good, take a look at the quick install page. Otherwise follow the documentation flow (using the "next"/"previous" buttons at the bottom of each page), skipping whatever you think you don't need.
如果你的 Unix 和 ssh 操作技巧熟练，请查看快速安装页面。否则，请遵循文档流程（使用每页底部的“下一步”/“上一步”按钮），跳过你认为不需要的部分。

## TROUBLESHOOTING
The emergencies page helps with all sorts of install/setup issues, recovering from lost keys and other self-inflicted wounds, lists some common (and some uncommon!) errors and non-standard configs that may trip you up.
紧急情况页面帮助解决各种安装/设置问题，包括恢复丢失的密钥和其他自己造成的麻烦，列出一些常见（和不常见！）错误以及可能让你困惑的非标准配置。

## contact/support
security issues# If you think you found a possible security issue, please email me (sitaramc@gmail.com) directly.
安全问题# 如果你认为发现了潜在的安全问题，请直接给我发邮件（sitaramc@gmail.com）。

mailing list(s)# For all other communications, please use the mailing list. This has some advantages over sending direct to me (if I'm not available, someone else might respond; someone else may have additional info that I may not have thought of, or experiences to share; the issue becomes searchable for future reference; and finally, it keeps me honest!)
邮件列表# 对于所有其他通信，请使用邮件列表。这比直接发给我有一些优势（如果我不在，其他人可能会回复；其他人可能有我没想到的额外信息或经验分享；问题变得可搜索以供将来参考；最后，这让我保持诚实！）

NOTE: The first email from a new member is held until I can clear it; subsequent emails from the same email address do not get held up.
注意：新成员的第一封邮件会被 held 直到我清除；同一邮箱地址的后续邮件不会被 held。

Mailing list for support and general discussion: Subscribe: gitolite+subscribe@googlegroups.com Post: gitolite@googlegroups.com (If you're unwilling to join the mailing list, you can still send me direct mail if it's something simple, obvious, or quick. But if the issue needs multiple mails and/or some discussion, I prefer the mailing list.)
支持和一般讨论的邮件列表：订阅：gitolite+subscribe@googlegroups.com 发布：gitolite@googlegroups.com（如果你不愿意加入邮件列表，如果是简单、明显或快速的事情，仍然可以直接给我发邮件。但如果问题需要多封邮件和/或一些讨论，我更喜欢使用邮件列表。）

还有一个低流量、单向的发布和安全公告邮件列表：订阅：gitolite-announce+subscribe@googlegroups.com
