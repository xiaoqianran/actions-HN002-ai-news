# Building an AmigaOS Development Environment in 2026  
# 在2026年构建AmigaOS开发环境

---

Apropos of nothing, I decided to set up an Amiga development environment. Since figuring things out wasn't straightforward, I wrote down instructions for Linux about how to compile the first program and run it in an emulator. Enjoy!  
顺便一提，我决定搭建一个Amiga开发环境。由于整个过程并不直观，我整理了在Linux上编译第一个程序并在模拟器中运行的指南。Enjoy！

---

## Prerequisites  
## 先决条件

In this post I'm assuming that the reader is a proficient Linux user who can build software from source code and figure out missing dependencies. Standard build tools should be sufficient to get started. Basic understanding of shell commands is also expected.  
本文假设读者是熟练的Linux用户，能够从源代码构建软件并解决缺失的依赖项。标准构建工具足以入门。同时也需要基本的shell命令知识。

More detailed instructions on how to build the toolchain are available here: https://franke.ms/amiga/amiga-gcc.wiki.  
关于如何构建工具链的更详细说明，请参阅：https://franke.ms/amiga/amiga-gcc.wiki。

Another rather problematic requirement is that to run AmigaOS, you need ROM images (firmware) and workbench disks (installation media). They may be bought from legal sources or dumped from owned hardware. Alternatively one could use the AROS ROM that is shipped with the emulator, although it is known to have worse compatibility with standard Amiga software.  
另一个相当棘手的要求是，要运行AmigaOS，你需要ROM镜像（固件）和工作台磁盘（安装介质）。它们可以从合法来源购买，或从拥有的硬件中转储。或者，你也可以使用模拟器自带的AROS ROM，尽管众所周知它与标准Amiga软件的兼容性较差。

---

## Toolchain  
## 工具链

To build programs for AmigaOS we need the compiler. A popular tool is amiga-gcc.  
要构建AmigaOS程序，我们需要编译器。一个流行的工具是amiga-gcc。

First I tried to use the default (the most battle-tested) GCC branch, but while the produced executable worked, loading the binary in m68k-amigaos-gdb caused a segmentation fault. After some trial and error I reported the issue and the toolchain maintainer, Stefan 'bebbo' Franke, kindly explained that the toolchain now produces DWARF2 debug info and then fixed a bug in embedding debug sections in the final executable. I've finally settled on a branch amiga16.1 for GCC and on a branch amiga-2.46 for binutils (it contains the abovementioned fix).  
首先我尝试使用默认（最经过实战检验）的GCC分支，但虽然生成的可执行文件能运行，在m68k-amigaos-gdb中加载二进制文件时却导致了段错误。经过一番尝试，我报告了这个问题，工具链维护者Stefan 'bebbo' Franke kindly解释称工具链现在生成DWARF2调试信息，并修复了在最终可执行文件中嵌入调试节的bug。我最终为GCC选择了amiga16.1分支，为binutils选择了amiga-2.46分支（它包含了上述修复）。

Note that `make update` may fail due to connection errors. In that case rerun it until everything is downloaded. Both `make update` and `make all` may take some time - on my machine everything takes about 1 hour.  
注意`make update`可能会因连接错误而失败。在这种情况下，请重新运行它直到所有内容下载完成。`make update`和`make all`都可能需要一些时间——在我的机器上整个过程大约需要1小时。

```bash
export WORKSPACE=${HOME}/Workshop/AmigaOS
export TOOLCHAIN=${WORKSPACE}/amiga-gcc-toolchain
export PREFIX=${TOOLCHAIN}
mkdir -p ${WORKSPACE}
cd ${WORKSPACE}
git clone https://franke.ms/git/bebbo/amiga-gcc.git
pushd amiga-gcc
make branch branch=amiga16.1 mod=gcc
make branch branch=amiga-2.46 mod=binutils
make update
make all
# Check whether the compiler works. SDL1.2 build failed on my host, but it is
# not needed.
${TOOLCHAIN}/bin/m68k-amigaos-gcc -v
popd
```

Now we have a working AmigaOS toolchain. It supports a few different runtime/libc versions. When nothing is specified it defaults to newlib.  
现在我们有了一个可用的AmigaOS工具链。它支持几种不同的运行时/libc版本。如果不指定，默认使用newlib。

```bash
export WORKSPACE=${HOME}/Workshop/AmigaOS
export TOOLCHAIN=${WORKSPACE}/amiga-gcc-toolchain
export PATH=${TOOLCHAIN}/bin:${PATH}
mkdir Shared
pushd Shared
cat > woosh.c <<EOF
#include <stdio.h>
void hi(void) {
    printf("jd was here \\\o/\n");
}
int main() {
    hi();
    return 0;
}
EOF
m68k-amigaos-gcc -g woosh.c -o woosh.out
file woosh.out
# woosh.out: AmigaOS loadseg()ble executable/binary
popd
```

Congratulations! We've just built our first AmigaOS program. It seems that `file` has a bit of a problem parsing AmigaOS executables, but the binary works.  
恭喜！我们刚刚构建了第一个AmigaOS程序。`file`命令似乎有点问题，无法正确解析AmigaOS可执行文件，但二进制文件本身是能运行的。

## Emulator  
## 模拟器

Now how to run this file? We need two pieces: the emulator AmigaOS ROMs  
现在如何运行这个文件？我们需要两样东西：模拟器和AmigaOS ROMs

Obtaining the emulator is straightforward. I picked Amiberry, because it provides a good developer experience - built-in bsdsocket.library, virtual hard disks and shared directories, and it works well on Linux. Another good contender is FS-UAE. Both should work well. They have binary releases or we can build them from source.  
获取模拟器很简单。我选择了Amiberry，因为它提供了良好的开发体验——内置bsdsocket.library、虚拟硬盘和共享目录，并且在Linux上运行良好。另一个不错的选择是FS-UAE。两者都应该能很好地工作。它们有二进制发布版本，也可以从源代码构建。

We'll install Amiberry from Flathub:  
我们将从Flathub安装Amiberry：

```bash
flatpak install flathub com.blitterstudio.amiberry
```

Obtaining ROMs is a bit harder. If you own the original hardware you may already have a legal dump. Otherwise you may buy them from https://www.amigaforever.com/ or https://www.hyperion-entertainment.com/.  
获取ROMs则稍微困难一些。如果你拥有原版硬件，可能已经有了合法的转储。否则，你可以从https://www.amigaforever.com/或https://www.hyperion-entertainment.com/购买。

Assuming that you have ROMs, we may start the emulator and configure our target system. We need the following files:  
假设你已经有了ROMs，我们可以启动模拟器并配置目标系统。我们需要以下文件：

- ROM file: `kick.a1200.46.143`
- floppies: `Install314.adf`, `Workbench314.adf`, etc.

```bash
mkdir -p Amiberry/{ROMs,Floppies}
# Amiberry expects .a1200 file extension (it doesn't recognize .46.143)
cp Nero/ROMs/kick.a1200.46.143 Amiberry/ROMs/kick.a1200
cp Nero/Floppies/*.adf Amiberry/Floppies
flatpak run com.blitterstudio.amiberry
```

Now navigate to the "Paths" tab and configure them to point to the directory "Amiberry" that we just created. Confirm when prompted to create subdirectories. Then click the "Rescan Paths" button and go back to the "Quickstart" tab.  
现在导航到"Paths"选项卡，将它们配置指向我们刚创建的"Amiberry"目录。当提示创建子目录时确认。然后点击"Rescan Paths"按钮，返回"Quickstart"选项卡。

Amiga hardware has a few distinct flavors with different hardware configurations and running variations of M68K CPU. RAM is divided between the "chip memory" that is accessible directly by custom chips (graphics, audio and DMA) and "fast memory" - general-purpose RAM. The first one is usually 1-2 MB, while the latter can typically be expanded to something like 4 MB of Fast RAM. That's plenty!  
Amiga硬件有几个不同的型号，具有不同的硬件配置和M68K CPU的运行变体。内存分为由自定义芯片（图形、音频和DMA）直接访问的"chip memory"和通用RAM的"fast memory"。前者通常是1-2 MB，而后者通常可以扩展到大约4 MB的Fast RAM。这足够了！

We will pick Amiga 1200, because it has AGA (Amiga Advanced Graphics Architecture) chipset and 4 MB Fast RAM expanded configuration.  
我们将选择Amiga 1200，因为它有AGA（Amiga高级图形架构）芯片组和4 MB Fast RAM扩展配置。

- Model: Amiga 1200
- Configuration: 4 MB Fast RAM expanded configuration
- Emulated Drives (DF0): Install314.adf

While installing, the floppy will need to be replaced by files requested by the installer. The emulated drive DF0 represents the floppy drive. To get back to the configuration menu while the virtual machine is running, press F12. Then navigate to the "ROM" tab and select `kick.a1200` ROM as the "Main ROM File". Then go to "Expansions" tab and enable `bsdsocket.library`. We'll need TCP/IP connection later to run `bgdbserver`. Finally add a virtual hard drive and a shared directory:  
安装过程中，软盘需要被安装程序请求的文件替换。模拟驱动器DF0代表软盘驱动器。在虚拟机运行时，按F12可返回配置菜单。然后导航到"ROM"选项卡，选择`kick.a1200` ROM作为"Main ROM File"。接着转到"Expansions"选项卡并启用`bsdsocket.library`。我们稍后需要TCP/IP连接来运行`bgdbserver`。最后添加一个虚拟硬盘和一个共享目录：

- Navigate to "Hard drives/CD" tab
- Select "Create Hardfile" :: 1024MB, RDB Mode enabled
- Select "Add Hardfile" and select the newly created file
- Select "Add Directory"

These drives will be visible in Amiga as DH0 and DH1, respectively. RDB is Amiga's native partition table format.  
这些驱动器在Amiga中将分别显示为DH0和DH1。RDB是Amiga的原生分区表格式。
