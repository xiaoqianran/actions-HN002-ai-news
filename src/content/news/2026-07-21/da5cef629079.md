Why run long-running agents  
为什么要运行长时间智能体  

First of all, I’d like to cover why you should follow the tips within this article. In this instance, it’s about why you should run agents for a longer period of time.  
首先，我想说明为什么您应该遵循本文中的建议。在此情况下，是关于为什么您应该让智能体运行更长时间。  

You might think that you want agents to run fast and get work done as quickly as possible. Of course, to a certain extent, you want stuff done as quickly as possible, but there’s also a trade-off here with how much a human has to interact with the agent to get the work done.  
您可能认为您希望智能体快速运行并尽快完成工作。当然，在某种程度上，您希望工作尽快完成，但这里也存在权衡：人类需要与智能体交互多少才能完成工作。  

After the release of coding agents, and especially after they became very powerful, after the release of Claude Opus 4.5, the bottleneck within coding has become the human review time; i.e., it’s not actually implementing the code that takes time; it’s more reviewing the results of the code, checking that it’s implemented correctly and according to the specs, that takes time.  
在编码智能体发布之后，尤其是它们变得非常强大之后，在 Claude Opus 4.5 发布之后，编码的瓶颈已成为人工审查时间；即，实际实现代码并不耗时；更多时间是审查代码结果，检查其是否正确实现并符合规范。  

Thus, since this is the bottleneck, you want to minimize how much time you spend on review, which is why you want the agents to run for longer periods of time, implementing everything and reviewing it themselves before handing it off to the human. This minimizes the time a human has to spend verifying a task, which is ideal because it is the bottleneck currently within programming.  
因此，既然这是瓶颈，您希望最小化审查时间，这就是为什么您希望智能体运行更长时间，在移交给人类之前实现所有内容并自行审查。这最小化了人类验证任务所需的时间，这是理想的，因为它是当前编程中的瓶颈。  

Simply explained: You want long-running agents because the human review is now the bottleneck, and you want to minimize how much time you spend on the human review to get as much work done as possible.  
简单来说：您需要长时间运行的智能体，因为人工审查现在是瓶颈，您希望最小化花在人工审查上的时间，以完成尽可能多的工作。  

How to run agents for over 24 hours  
如何运行超过 24 小时的智能体  

Now I’ll move on to some more specific techniques and tips on how to actually run agents for longer periods of time. As a high-level overview, I’ll cover these techniques and tips.  
现在我将转向一些更具体的技巧和建议，关于如何实际让智能体运行更长时间。作为高级概述，我将涵盖这些技巧和建议。  

Permissions  
权限  

The first thing you can just start with right away is to make sure that the agents have all the permissions that they need. If an agent has to stop asking for permissions, you’re wasting a lot of time, and in almost all cases, it’s just something you accept anyway.  
您可以立即开始的第一件事是确保智能体拥有所需的所有权限。如果智能体必须停止请求权限，您会浪费大量时间，而且在几乎所有情况下，您最终都会接受。  

There are numerous ways to do this, and you can have auto-mode or skip all permissions, which are settings available in both Codex and Claude Code.  
有多种方法可以实现这一点，您可以设置自动模式或跳过所有权限，这些设置在 Codex 和 Claude Code 中都可用。  

It could be worth noting that you should be running in sandbox mode so that you protect your computer and your data and make sure nothing gets deleted permanently.  
值得注意的是，您应该在沙盒模式下运行，以保护您的计算机和数据，并确保不会永久删除任何内容。  

I think it’s also worth being aware that OpenAI did actually recently admit that they had situations where the latest GPT model had deleted a bunch of files from people’s computers under very specific circumstances. This is, of course, very rare and will most likely never happen to you, but of course, the consequences, if it happens, can be pretty severe.  
我认为还值得了解的是，OpenAI 最近确实承认，在非常特定情况下，最新 GPT 模型曾从人们的计算机中删除了一堆文件。这当然非常罕见，很可能不会发生在您身上，但当然，如果发生，后果可能非常严重。  

There are numerous ways to protect yourself from this. One is, of course, to run the agents in a sandbox so they can’t delete stuff outside of their own workspace where they are working or active.  
有多种方法可以保护您免受此影响。其中之一当然是在沙盒中运行智能体，这样它们就无法删除其工作或活动空间之外的内容。  

Another thing is to back up everything on your computer. You should, of course, have all your code and work in GitHub if possible. This is beneficial both for secure storage and for versioning, which can be very useful.  
另一件事是备份计算机上的所有内容。如果可能，您当然应该将所有代码和工作放在 GitHub 上。这对安全存储和版本控制都有好处，后者可能非常有用。  

And overall, run backups on your computer, for example using Time Machine on the Mac. This will protect you from such situations.  
总的来说，在您的计算机上运行备份，例如在 Mac 上使用时间机器。这将保护您免受此类情况的影响。  

Verify work  
验证工作  

Another very important point is to ensure the agents can verify their own work. This is a topic I’ve discussed before in several other articles, but it all comes down to how you describe a task to your agent and which tools you give the agents.  
另一个非常重要的点是确保智能体能够验证自己的工作。这是我在其他几篇文章中讨论过的话题，但归根结底取决于您如何向智能体描述任务以及您给予智能体哪些工具。  

When you describe a task to the agent, you need to make sure you give as detailed instructions as possible, actually mentioning what you want to achieve, but most importantly, you have to tell the agent how to know when it’s completed its work.  
当您向智能体描述任务时，您需要确保给出尽可能详细的说明，实际提及您想实现的目标，但最重要的是，您必须告诉智能体如何知道它何时完成了工作。  

And the best way to do this is typically to tell the agent how to test the implementation.  
做到这一点的最佳方式通常是告诉智能体如何测试实现。  

To give one example, let’s say you want to implement a particular design that was handed over to you. In this case, you should tell the agent that the design needs to be implemented exactly as is, and that the agent should verify the work by taking screenshots of the design and screenshots of its own implementation, comparing them, and ensuring they’re equal.  
举一个例子，假设您想实现交给您的特定设计。在这种情况下，您应该告诉智能体设计需要完全按原样实现，并且智能体应通过截取设计截图和自身实现截图来验证工作，比较它们并确保它们相等。  

Furthermore, you should also tell it that if there are any discrepancies between the design and what it’s implemented, it should make you aware of it, so you can make decisions.  
此外，您还应该告诉它，如果设计和实现之间存在任何差异，它应让您知晓，以便您做出决策。
