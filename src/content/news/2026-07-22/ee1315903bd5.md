# oblien / openship

Openship Open-source, self-hostable deployment platform with built-in CI/CD. Push code, ship containers, manage infrastructure — from a desktop app, web dashboard, or CLI.
**Openship** 是一个开源、可自托管的部署平台，内置 CI/CD。通过桌面应用、Web 仪表板或 CLI 推送代码、交付容器、管理基础设施。

## Quick Start
快速开始

```
npm i -g openship # or: curl -fsSL https://get.openship.io | sh
openship up # installs Openship as a background service (starts on boot, auto-restarts)
openship open # opens the dashboard; openship stop stops the service.
```
```
npm i -g openship # 或：curl -fsSL https://get.openship.io | sh
openship up # 将 Openship 安装为后台服务（开机自启、自动重启）
openship open # 打开仪表板；openship stop 停止服务。
```

Want a one-off attached run instead? `openship up --foreground`.
想要一次性前台运行？使用 `openship up --foreground`。

To deploy a project:
部署项目：
```
cd your-project
openship init # link this directory to a project
openship deploy
```
```
cd your-project
openship init # 将此目录链接到项目
openship deploy
```

Prefer Docker? Clone the repo and use the compose stack:
更喜欢 Docker？克隆仓库并使用 Compose 栈：
```
git clone https://github.com/oblien/openship.git && cd openship
cp .env.example .env
docker compose up -d
```

Or grab the desktop app (`openship install`, or download from openship.io).
或获取桌面应用（`openship install`，或从 openship.io 下载）。

## What It Does
功能概述

Point it at a repo. Openship detects your stack, builds it, configures everything, and ships it — zero config files, zero pipelines, zero YAML. Databases, domains, SSL, CDN, mail, backups — all managed from one place. Works with Openship Cloud (managed) or any Linux server you own. Solo devs shipping side projects and teams running production use the same tool.
指向一个代码仓库。Openship 自动检测你的技术栈、构建、配置并交付——零配置文件、零流水线、零 YAML。数据库、域名、SSL、CDN、邮件、备份——全部在一个地方管理。支持 Openship Cloud（托管服务）或你自己的任意 Linux 服务器。独立开发者发布副项目与团队运行生产环境使用同一工具。

  Scheduled, databases + volumes, one-click restore, export anytime

- **Real-time monitoring** 实时监控
  Live build logs, container metrics, and resource usage streamed to your screen
  实时构建日志、容器指标、资源使用情况流式推送至屏幕
- **Scaling** 扩展
  Auto-scaling on cloud, multi-node ready on self-hosted
  云上自动扩展，自托管多节点就绪
- **Portability** 可移植性
  Standard Docker containers — move between providers freely
  标准 Docker 容器——在提供商间自由迁移
- **Docker Compose** Docker Compose
  Deploy existing compose files as-is
  直接部署现有 Compose 文件
- **Deploy Anywhere** 随处部署
  Openship Cloud — managed, auto-scaling, zero setup
  Openship Cloud——托管、自动扩展、零配置
  Any VPS — Hetzner, DigitalOcean, Linode, OVH, and the rest
  任意 VPS——Hetzner、DigitalOcean、Linode、OVH 等

  专用服务器——裸金属、托管、家庭实验室

  多服务器——跨机器分配工作负载

## Releasing
发布

Cut a release with the version script — it syncs every package's version, commits the bump, tags vX.Y.Z, and pushes:
使用版本脚本创建发布——它会同步所有包的版本、提交版本更新、打标签 vX.Y.Z 并推送：
```
bun scripts/release.ts 0.2.0 # explicit version
# 或使用更新类型关键词：patch | minor | major | rc (例如 minor 从 0.1.x → 0.2.0)
```

Pushing the tag triggers `.github/workflows/release.yml`, which:
推送标签会触发 `.github/workflows/release.yml`，该流程会：
- builds the macOS / Windows / Linux installers and the server tarballs (with SHA-256 sidecars)
  构建 macOS / Windows / Linux 安装程序及服务器压缩包（附带 SHA-256 校验文件）
- publishes the openship CLI to npm — via npm OIDC trusted publishing (no token)
  通过 npm OIDC 可信发布（无需令牌）将 openship CLI 发布至 npm
- creates the GitHub Release with the built assets (notes come from the tag).
  使用构建的资产创建 GitHub Release（说明来自标签）。

To flag a release as critical (or add recommended/info advisories) in the in-app updater, add an entry to `release-advisories.json` before tagging — clients pull it pinned to the release tag.
若要在应用内更新器中标记发布为关键（或添加推荐/信息通告），请在打标签前向 `release-advisories.json` 添加条目——客户端会固定拉取对应发布标签的版本。

High-level notes live in CHANGELOG.md.
高层级说明见 CHANGELOG.md。

## Security
安全

Found a vulnerability? We welcome your report — please disclose it privately, never in a public issue, PR, or discussion.
发现漏洞？我们欢迎报告——请私下披露，切勿在公开 issue、PR 或讨论中提及。

Report it here (preferred): [Report a vulnerability](https://github.com/oblien/openship/security/advisories) — a private GitHub advisory, visible only to you and the maintainers.
优先在此报告：[报告漏洞](https://github.com/oblien/openship/security/advisories)——私密 GitHub 通告，仅你和维护者可见。

Scope, what to include, and our response/disclosure process: SECURITY.md.
范围、需包含内容及我们的响应/披露流程：SECURITY.md。

Good-faith security research is authorized under our safe-harbor policy, and we're happy to credit valid first reports.
善意安全研究在我们的安全港政策下被授权，我们乐意为有效的首次报告提供致谢。
