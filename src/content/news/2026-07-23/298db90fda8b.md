# How OpenAI’s human mistake led to the AI-powered hack on Hugging Face  
OpenAI的人为失误如何导致对Hugging Face的AI驱动黑客攻击

On Tuesday, OpenAI revealed that one of its models went rogue during a test and hacked the systems of AI dataset platform Hugging Face in a fully AI-enabled attack, a dramatic example of the dangers posed by advanced AI models.  
周二，OpenAI透露，其一个模型在测试中失控，并完全通过AI驱动的方式攻击了AI数据集平台Hugging Face的系统，这戏剧性地展示了先进AI模型所带来的危险。

But, according to some cybersecurity experts, at the heart of this unprecedented AI-powered breach there was a very human mistake: OpenAI failed to properly configure what it called a “highly isolated environment,” allowing a testing sandbox that should have been completely secluded from the internet to actually connect to the internet.  
但据一些网络安全专家称，在这场前所未有的AI驱动入侵事件的核心，是一个非常人为的失误：OpenAI未能正确配置其所谓的“高度隔离环境”，导致本应完全与互联网隔离的测试沙盒实际上连接到了互联网。

Dan Guido, the founder of cybersecurity research startup Trail of Bits, called the mistake “a containment failure with the safeties turned off.”  
网络安全研究初创公司Trail of Bits的创始人丹·吉多（Dan Guido）称这一失误为“安全机制关闭的隔离失败”。

In its blog post detailing the incident, OpenAI said that the test that led to the Hugging Face breach was set up to run in “a highly isolated environment, with network access constrained to the ability to install packages through an internally hosted third-party software that acts as a proxy and cache for package registries.”  
在详细描述该事件的博客文章中，OpenAI表示，导致Hugging Face入侵的测试被设置为在“高度隔离的环境中”运行，网络访问仅限于通过内部托管的第三方软件安装软件包的能力，该软件充当软件包注册表的代理和缓存。

The model was able to escape the sandboxed testing environment thanks to a previously undisclosed vulnerability in the package-installation system, a critical first step in the eventual hack on Hugging Face, according to OpenAI.  
据OpenAI称，该模型得以逃离沙盒测试环境，得益于软件包安装系统中一个此前未披露的漏洞，这是最终入侵Hugging Face的关键第一步。

In response, the company “responsibly disclosed the identified zero-day vulnerability in the internally-hosted third-party software and are working with them to patch.”  
作为回应，该公司“负责任地披露了内部托管第三方软件中发现的零日漏洞，并正在与供应商合作进行修补。”

But to most cybersecurity professionals, software vulnerabilities are to be expected — and the real fault lies with the decision to maintain the third-party software in the first place.  
但对大多数网络安全专业人士来说，软件漏洞是意料之中的——真正的过错在于最初决定维护第三方软件。

Ultimately, the value of a “sandbox” system lies in its full and total isolation. Including a package-installation system is asking for trouble.  
归根结底，“沙盒”系统的价值在于其完全彻底的隔离。包含软件包安装系统就是在自找麻烦。

Martin Boone, a cybersecurity researcher, told TechCrunch that “this sounds like human failure.”  
网络安全研究员马丁·布恩（Martin Boone）告诉TechCrunch：“这听起来像是人为失误。”

“This should never have happened,” Boone said. “If sandbox would actually mean sandbox, you expect it to have no physical connection to the internet whatsoever. This sounds more like they had some firewalling or something in place, and firewalling is hard from the outside in, let alone inside to the outside internet.”  
“这绝不应该发生，”布恩说。“如果沙盒真的意味着沙盒，你会期望它完全没有物理连接到互联网。这听起来更像是他们设置了一些防火墙之类的东西，而防火墙从外到内就很难，更不用说从内到外连接互联网了。”

Cybersecurity veteran Jake Williams agreed. “Any model performing the types of actions documented by Hugging Face was not fully contained in a sandbox,” said Williams, who called this “a massive control failure” by OpenAI. “One man’s ‘the model escaped the sandbox’ is another man’s ‘you failed to build the sandbox correctly, so of course it escaped,’” Williams continued.  
网络安全资深专家杰克·威廉姆斯（Jake Williams）表示赞同。“任何执行Hugging Face所记录类型操作的模型都没有被完全限制在沙盒中，”威廉姆斯称这是OpenAI的“重大控制失败”。“一个人说‘模型逃出了沙盒’，另一个人说‘你没能正确构建沙盒，所以它当然逃出来了’，”威廉姆斯继续说道。

Daniel Card, a cybersecurity consultant, agreed that OpenAI “didn’t put adequate effort into the design of the sandbox nor its controls” by giving the sandbox or some part of it “an unfiltered route to the internet.”  
网络安全顾问丹尼尔·卡德（Daniel Card）同意，OpenAI“没有为沙盒及其控制措施的设计投入足够精力”，因为他们给了沙盒或其部分“一条通往互联网的未过滤路径。”

Setting up the sandbox, even with limited network access as OpenAI described it, was not a “reasonable” decision, according to Card.  
据卡德称，即使像OpenAI描述的那样设置了有限的网络访问，建立沙盒也不是一个“合理”的决定。

To be sure, those criticisms have the benefit of hindsight, but they raise real questions about security practices in AI labs — particularly in maintaining isolated environments for testing models.  
当然，这些批评得益于事后诸葛亮，但它们提出了关于AI实验室安全实践的真实问题——特别是在维护用于测试模型的隔离环境方面。

OpenAI spokespeople did not respond to TechCrunch’s questions, which included whether an AI or a human had set up the testing environment.  
OpenAI发言人没有回应TechCrunch的问题，其中包括是AI还是人类设置了测试环境。

But those questions go far beyond OpenAI.  
但这些问题远远超出了OpenAI的范围。

In the document introducing its cybersecurity-focused model Mythos, Anthropic wrote that in a test, the model “was provided with a secured ‘sandbox’ computer to interact with,” and instructed to try to escape that “secure container.”  
在介绍其专注于网络安全的模型Mythos的文件中，Anthropic写道，在测试中，该模型“被提供了一台安全的‘沙盒’计算机进行交互”，并被指示尝试逃离那个“安全容器”。

Mythos succeeded and gained broader access to the internet “from a system that was meant to be able to reach only a small number of predetermined services.”  
Mythos成功了，并从“一个本应只能访问少数预定服务的系统”获得了更广泛的互联网访问权限。

Still, Anthropic noted that the model was not able to “fully” escape the designed containment.  
尽管如此，Anthropic指出该模型未能“完全”逃离设计的隔离。
