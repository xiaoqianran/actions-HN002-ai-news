# Making  
# 关于“创造”

Beej's Bit Bucket ⚡ Tech and Programming Fun 2026-03-12 On Making  
Beej's Bit Bucket ⚡ 科技与编程乐趣 2026-03-12 关于“创造”

I made this!  
这是我做的！

TLDR: I gain a lot of fulfillment by making things. I don't consider things built by others at my request to be made by me, and are therefore much less fulfilling. And then I feel sad.  
 TL;DR：我通过亲手创造事物获得巨大的满足感。我不认为请他人代劳的东西是我做的，因此满足感大打折扣，然后我会感到难过。

This article starts strong and then heads off into the weeds.  
这篇文章开头有力，后来却钻进了牛角尖。

There have been a lot of pieces written about what I'll call "the AI dev schism"  
已经有很多文章讨论我称之为“AI开发者分裂”的现象

And I think there's a lot of truth to those:  
我认为其中有很多真相：

Loss of the craft, coding things by hand  
手工艺的丧失——亲手编写代码

Loss of low-level problem-solving  
底层问题解决能力的丧失

Loss of fun  
乐趣的丧失

Gain of high-level problem-solving  
高级问题解决能力的获得

Getting through back-burnered projects  
推进被搁置的项目

Gain of fun  
乐趣的获得

We'll just grant those as being correct for various developers.  
我们就姑且认为这些对不同开发者来说都是正确的。

But there's something else that troubles me.  
但还有别的事困扰着我。

在开始前先讲讲背景，以便你更好地理解我的视角：

我是X世代黑客；我在80年代微型计算机时代入门。

我拥有计算机科学的学士和硕士学位。

我有20年行业经验（惠普、初创公司、联合创始人、动视等）。

CS instructor for the last 9 years, now at Oregon State University-Cascades.  
过去9年担任计算机科学讲师，现在在俄勒冈州立大学卡斯卡德分校。

在AI乌托邦/末日论光谱上，我持65%的悲观立场。

```rust
fn try_move(&mut self, dx: i32, dy: i32) {
    let nx = self.player.x + dx;
    let ny = self.player.y + dy;
    // Check for monster combat
    if let Some(idx) = self.world.monster_at(nx, ny) {
        let result = {
            let monster = &mut self.world.monsters[idx];
            resolve_combat(&mut self.player, monster, &mut self.rng)
        };
        self.messages.push_many(result.messages);
        if result.monster_defeated {
            let monster = &self.world.monsters[idx];
            let xp = monster.xp_reward;
            let gold = monster.gold_reward;
            self.player.xp += xp;
            self.player.gold += gold;
            if gold > 0 {
                self.messages.push(format!("You find {} gold!", gold));
            }
            if self.player.try_level_up() {
                self.messages.push(format!(
                    "Level up! You are now level {}!",
                    self.player.level
                ));
            }
        }
        if !self.player.is_alive() {
            self.messages.push("You have been slain! Rest in peace...");
        }
        self.advance_turn();
        return;
    }
    // Check terrain passability
    if self.world.is_passable(nx, ny) {
        self.player.x = nx;
        self.player.y = ny;
        self.advance_turn();
    } else {
        let terrain = self.world.terrain_at(nx, ny);
        self.messages.push(format!("The {} blocks your path.", terrain.name()));
    }
}
```

I Am Uncomfortable  
我感到不适

I don't like lying. And yet I feel, dear reader, I have misled you.  
我不喜欢撒谎。然而，亲爱的读者，我觉得我误导了你。

Yes, all that has been created (including my deck) and I was the initiator of all that creation.  
是的，所有被创造出来的东西（包括我的甲板）都是我发起的。

But I don't really feel like I made any of it. I'm uncomfortable claiming that I did so.  
但我并不觉得我真的做了其中任何一件。声称我做了这些让我很不适。

Since you are certainly aware by now that all of the above is AI-generated (except my deck, which was created by skilled, paid craftsmen), perhaps you feel a little bit of discomfort with me claiming credit for doing those things, too.  
既然你现在肯定知道以上所有东西都是AI生成的（除了我的甲板，它是由熟练的、受雇的工匠制作的），也许你对我声称做了这些事也感到有点不适。

However, I don't think everyone feels this way.  
然而，我认为不是每个人都这样感觉。

I know many people who ask contractors to build things and they phrase it like they built it. "I put in a new front deck," they'd say, even though other people did all the work.  
我认识很多人，他们请承包商建造东西，却说得像自己建的一样。“我装了个新前廊，”他们会说，尽管所有工作都是别人做的。

Personally, I feel that's misleading. I'm more of a "I had a new front deck put in" kind of person.  
我个人认为那是误导性的。我更像是个“我让人装了个新前廊”的人。

And when I do have Claude create something for me, I just can't say that I made it. Other people can, but I just can't.  
而且当我有Claude为我创造东西时，我就是不能说是我做的。别人可以，但我就是不能。

Again, I'm more prone to say, "I had this code built for me."  
同样，我更容易说：“我让人为我写了这段代码。”

I don't even feel comfortable MIT-licensing that (not-for-hire) work, if that's even legally possible. I just Unlicense it all.  
我甚至对MIT许可那份（非雇佣）工作感到不适，如果这在法律上可能的话。我就直接把它都Unlicense了。

As a manager, I'd never say that I built a product. "My team built this," I'd say.  
作为经理，我永远不会说我打造了一个产品。“我的团队打造了这个，”我会说。
