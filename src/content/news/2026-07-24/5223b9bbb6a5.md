# Title: Software rendering in 500 lines of bare C++  
# 标题：仅用500行裸C++实现软件渲染

---

In this series of articles, I aim to demonstrate how OpenGL, Vulkan, Metal, and DirectX work by writing a simplified clone from scratch.  
在本系列文章中，我旨在通过从零开始编写一个简化版克隆，来演示 OpenGL、Vulkan、Metal 和 DirectX 的工作原理。

Surprisingly, many people struggle with the initial hurdle of learning a 3D graphics API.  
令人惊讶的是，许多人在学习 3D 图形 API 的初始阶段就遇到了困难。

To help with this, I have prepared a short series of lectures, after which my students are able to produce quite capable renderers.  
为了解决这个问题，我准备了一系列简短的讲座，学完后我的学生们能够制作出相当强大的渲染器。

The task is as follows: using no third-party libraries (especially graphics-related ones), we will generate an image like this:  
任务如下：不使用任何第三方库（尤其是图形相关的库），我们将生成如下图像：

Warning: This is a training material that loosely follows the structure of modern 3D graphics libraries. It is a software renderer. I do not intend to show how to write GPU applications — I want to show how they work.  
警告：这是一份训练材料， loosely 遵循现代 3D 图形库的结构。它是一个软件渲染器。我无意展示如何编写 GPU 应用程序——我想展示它们是如何工作的。

I firmly believe that understanding this is essential for writing efficient applications using 3D libraries.  
我坚信，理解这一点对于使用 3D 库编写高效应用程序至关重要。

---

The final code consists of about 500 lines.  
最终代码大约由 500 行组成。

My students typically require 10 to 20 hours of programming to start producing such renderers.  
我的学生通常需要 10 到 20 小时的编程才能开始制作这样的渲染器。

The input is a 3D model composed of a triangulated mesh and textures.  
输入是由三角网格和纹理组成的 3D 模型。

The output is a rendered image.  
输出是一张渲染后的图像。

There is no graphical interface, the program simply generates an image.  
没有图形界面，程序只是生成一张图像。

To minimize external dependencies, I provide my students with a single class for handling TGA files — one of the simplest formats supporting RGB, RGBA, and grayscale images.  
为了最小化外部依赖，我向学生提供一个用于处理 TGA 文件的单一类——这是支持 RGB、RGBA 和灰度图像的最简单格式之一。

This serves as our foundation for image manipulation.  
这作为我们图像操作的基础。

At the beginning, the only available functionality (besides loading and saving images) is the ability to set the color of a single pixel.  
起初，唯一可用的功能（除了加载和保存图像）是设置单个像素颜色的能力。

There are no built-in functions for drawing line segments or triangles — we will implement all of this manually.  
没有内置的绘制线段或三角形的函数——我们将手动实现所有这些。

While I provide my own source code, written alongside my students, I do not recommend using it directly, as doing the work yourself is essential to understanding the concepts.  
虽然我提供了与学生一起编写的源代码，但不建议直接使用它，因为亲自动手工作对于理解概念至关重要。

The complete code is available on github, and you can find the initial source code I provide to my students here.  
完整代码可在 github 上获取，你可以在这里找到我提供给学生的初始源代码。

看，这就是起点：

**main.cpp**  
```cpp
#include "tgaimage.h"

constexpr TGAColor white = {255, 255, 255, 255}; // attention, BGRA order
constexpr TGAColor green = { 0, 255, 0, 255};
constexpr TGAColor red = { 0, 0, 255, 255};
constexpr TGAColor blue = {255, 128, 64, 255};
constexpr TGAColor yellow = { 0, 200, 255, 255};

int main(int argc, char** argv) {
    constexpr int width = 64;
    constexpr int height = 64;
    TGAImage framebuffer(width, height, TGAImage::RGB);

    int ax = 7, ay = 3;
    int bx = 12, by = 37;
    int cx = 62, cy = 53;

    framebuffer.set(ax, ay, white);
    framebuffer.set(bx, by, white);
    framebuffer.set(cx, cy, white);

    framebuffer.write_tga_file("framebuffer.tga");
    return 0;
}
```

**Compilation**  
**编译：**
```bash
git clone https://github.com/ssloy/tinyrenderer.git && cd tinyrenderer && cmake -Bbuild && cmake --build build -j && build/tinyrenderer obj/diablo3_pose/diablo3_pose.obj obj/floor.obj
```

The rendered image is saved to `framebuffer.tga`.  
渲染后的图像将保存到 `framebuffer.tga`。
