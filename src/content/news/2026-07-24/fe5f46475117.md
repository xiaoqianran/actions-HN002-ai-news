agegr / pi-web

Pi Web 中文文档  
Local web UI for the pi coding agent. Pi Web reads your local pi session files and gives you a browser workspace for session browsing, real-time chat, model configuration, skill management, and project file preview. The same pi session in CLI and Pi Web: structured tool calls, readable Markdown, session browsing, and cleaner results.  
Pi Web 中文文档  
pi 编码代理的本地 Web 界面。Pi Web 读取本地 pi 会话文件，为您提供浏览器工作空间，用于会话浏览、实时聊天、模型配置、技能管理和项目文件预览。CLI 和 Pi Web 中的相同 pi 会话：结构化工具调用、可读的 Markdown、会话浏览和更清晰的结果。  

## Quick Start  
Quick Start  

Run without installing:  
无需安装即可运行：  
```bash  
npx @agegr/pi-web@latest  
```  
Or install globally:  
或全局安装：  
```bash  
npm install -g @agegr/pi-web pi-web  
```  
Then open http://localhost:30141. The CLI will try to open the browser automatically after the server is ready.  
然后打开 http://localhost:30141。服务器准备就绪后，CLI 将尝试自动打开浏览器。  

## Options  
Options  

```bash  
pi-web --port 8080          # custom port  
pi-web --hostname 127.0.0.1 # local access only  
pi-web -p 8080 -H 127.0.0.1 # combine options  
pi-web --no-open            # do not open the browser automatically  
```  
```bash  
PORT=8080 pi-web            # environment variable is also supported  
PI_WEB_NO_OPEN=1 pi-web     # useful when running as a background service  
```  
```bash  
pi-web --port 8080          # 自定义端口  
pi-web --hostname 127.0.0.1 # 仅本地访问  
pi-web -p 8080 -H 127.0.0.1 # 组合选项  
pi-web --no-open            # 不自动打开浏览器  
```  
```bash  
PORT=8080 pi-web            # 也支持环境变量  
PI_WEB_NO_OPEN=1 pi-web     # 作为后台服务运行时有用  
```  

## HTTP Proxy  
HTTP 代理  

Pi Web reads the standard HTTP_PROXY, HTTPS_PROXY, and NO_PROXY environment variables for server-side model and API requests.  
Pi Web 读取标准 HTTP_PROXY、HTTPS_PROXY 和 NO_PROXY 环境变量，用于服务器端模型和 API 请求。  

On macOS or Linux:  
在 macOS 或 Linux 上：  
```bash  
HTTP_PROXY=http://127.0.0.1:7890 \  
HTTPS_PROXY=http://127.0.0.1:7890 \  
NO_PROXY=localhost,127.0.0.1 \  
npx @agegr/pi-web@latest  
```  
On Windows PowerShell:  
在 Windows PowerShell 上：  
```powershell  
$env:HTTP_PROXY = "http://127.0.0.1:7890"  
$env:HTTPS_PROXY = "http://127.0.0.1:7890"  
$env:NO_PROXY = "localhost,127.0.0.1"  
npx @agegr/pi-web@latest  
```  

## Features  
功能  

- **Pick work back up**: browse previous pi conversations by project without digging through terminal history or session paths.  
  **继续工作**：按项目浏览之前的 pi 对话，无需翻找终端历史记录或会话路径。  
- **Try different directions safely**: continue from an earlier message or fork a session into a separate route.  
  **安全尝试不同方向**：从较早的消息继续，或将会话分支到独立路线。  
- **Work across branches**: switch Git worktrees from the sidebar so new sessions and the Explorer follow the checkout you choose.  
  **跨分支工作**：从侧边栏切换 Git 工作树，使新会话和资源管理器跟随您选择的分支。  
- **Chat beside the project**: browse files on the left and preview source, docs, images, audio, and PDFs on the right while the agent works.  
  **项目旁聊天**：在左侧浏览文件，在右侧预览源代码、文档、图像、音频和 PDF，同时代理工作。  
- **See session state clearly**: context usage, cost, compaction state, and system prompt details are visible from the top bar.  
  **清晰查看会话状态**：上下文使用情况、成本、压缩状态和系统提示详情在顶部栏可见。  
- **Configure less from the terminal**: manage models, login/API keys, model tests, and skill switches from the web UI.  
  **减少终端配置**：通过 Web UI 管理模型、登录/API 密钥、模型测试和技能切换。  

## Notes  
注意事项  

- **Data directory**: Pi Web reads `~/.pi/agent/sessions` by default. Set `PI_CODING_AGENT_DIR` to point at another pi agent directory.  
  **数据目录**：Pi Web 默认读取 `~/.pi/agent/sessions`。设置 `PI_CODING_AGENT_DIR` 指向其他 pi 代理目录。  
- **Session files**: files are stored as `~/.pi/agent/sessions/<encoded-cwd>/<timestamp>_<uuid>.jsonl`.  
  **会话文件**：文件存储为 `~/.pi/agent/sessions/<encoded-cwd>/<timestamp>_<uuid>.jsonl`。  
- **Model config**: the Models panel reads and writes `models.json` in the pi agent directory. Model lists and defaults come from pi's config.  
  **模型配置**：模型面板读取和写入 pi 代理目录中的 `models.json`。模型列表和默认值来自 pi 的配置。  
- **File access**: file browsing and preview are scoped to the selected project directory and working directories that appear in sessions.  
  **文件访问**：文件浏览和预览范围限于选定的项目目录和会话中出现的工作目录。  
- **Git worktrees**: see Worktrees in Pi Web for when the switcher appears, how new worktrees are created, and what removal does.  
  **Git 工作树**：参阅 Pi Web 中的工作树，了解切换器何时出现、如何创建新工作树以及移除操作的影响。  
- **Forks vs in-session branches**: Fork creates a new `.jsonl` file. "Edit from here" creates another branch inside the same session file.  
  **分支与会话内分支**：Fork 创建新的 `.jsonl` 文件。“从此编辑”在同一会话文件中创建另一个分支。  

## Development  
开发  

```bash  
npm install  
npm run dev  
```  
The local dev server runs at http://localhost:30141.  
本地开发服务器运行在 http://localhost:30141。  

Common checks:  
常见检查：  
```bash  
node_modules/.bin/tsc --noEmit  
npm run lint  
```  
Avoid running `next build` / `npm run build` during local development. It writes to `.next/` and can interfere with the dev server; leave builds for release work.  
本地开发期间避免运行 `next build` / `npm run build`。它会写入 `.next/` 并可能干扰开发服务器；将构建留给发布工作。
